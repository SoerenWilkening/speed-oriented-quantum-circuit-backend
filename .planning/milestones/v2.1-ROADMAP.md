# Milestone v2.1: Compile Enhancements

**Status:** ✅ SHIPPED 2026-02-05
**Phases:** 52-54
**Total Plans:** 6

## Overview

Make `@ql.compile` inverse reuse physical ancilla qubits from forward call (uncompute + deallocate), and support `ql.qarray` as compiled function arguments.

## Phases

### Phase 52: Ancilla Tracking & Inverse Qubit Reuse

**Goal**: Users can call `f(x).inverse()` or `f.inverse(x)` to target the same physical ancilla qubits allocated during `f(x)`, uncomputing them to |0> and deallocating them
**Depends on**: v2.0 compilation infrastructure (Phase 51)
**Requirements**: INV-01, INV-02, INV-03, INV-04, INV-05, INV-06
**Plans**: 2 plans

Plans:
- [x] 52-01: Infrastructure + f.inverse(x) + f.adjoint(x) implementation
- [x] 52-02: Comprehensive tests for INV-01 through INV-06

**Details:**
- _deallocate_qubits exposed from _core.pyx wrapping allocator_free
- Forward call tracking with AncillaRecord storing ancilla qubits, virtual-to-real mapping, block, return qint
- f.inverse(x) via _AncillaInverseProxy: replays adjoint gates on original ancilla qubits, deallocates them
- f.adjoint(x) via _InverseCompiledFunc: standalone adjoint with fresh ancillas, no tracking
- Error handling for double-forward-without-inverse, inverse-without-forward, double-inverse
- 20 comprehensive tests covering all INV requirements

### Phase 53: Qubit-Saving Auto-Uncompute

**Goal**: When qubit-saving mode is active, compiled functions that return a qint automatically uncompute all ancillas except the return value's qubits after the forward call
**Depends on**: Phase 52 (ancilla tracking infrastructure)
**Requirements**: INV-07
**Plans**: 2 plans

Plans:
- [x] 53-01: Auto-uncompute implementation in compile.py
- [x] 53-02: Comprehensive tests for INV-07

**Details:**
- Cache key includes qubit_saving mode for recompilation on mode change
- _function_modifies_inputs helper detects input side-effects (skip auto-uncompute)
- _partition_ancillas separates return vs temp qubits
- _auto_uncompute method on CompiledFunc for automatic temp ancilla cleanup
- Modified _AncillaInverseProxy with reduced gate set after auto-uncompute
- 10 INV-07 tests covering all auto-uncompute scenarios

### Phase 54: qarray Support in @ql.compile

**Goal**: Users can pass `ql.qarray` objects as arguments to `@ql.compile`-decorated functions with correct capture, caching, and replay
**Depends on**: v2.0 compilation infrastructure (Phase 51); independent of Phases 52-53
**Requirements**: ARR-01, ARR-02, ARR-03, ARR-04
**Plans**: 2 plans

Plans:
- [x] 54-01: qarray support implementation in compile.py
- [x] 54-02: Comprehensive tests for ARR-01 through ARR-04

**Details:**
- qarray arguments recognized and validated in _classify_args()
- Unified qubit extraction functions handle both qint and qarray
- Cache key uses ('arr', len(arr)) tuple for qarray differentiation
- qarray return values correctly constructed during replay
- Replay validates total qubit count matches cached block
- 14 qarray-specific tests covering all ARR requirements

## Milestone Summary

**Key Decisions:**
- Only track forward calls when ancilla qubits exist (in-place functions skip tracking)
- f.inverse is @property returning _AncillaInverseProxy (enables f.inverse(x) syntax)
- f.adjoint is @property returning _InverseCompiledFunc (standalone, no tracking)
- Auto-uncompute triggers after both replay and capture paths
- Cache key includes qubit_saving mode for recompilation
- Use iteration protocol for qarray cdef access (not _elements directly)
- Cache key uses ('arr', length) tuple to distinguish qarray from qint

**Issues Resolved:**
- Fixed double-forward detection for in-place functions without ancillas
- Fixed adjoint-triggered capture polluting forward call registry
- Fixed cdef attribute access for qarray (use iteration protocol)

**Issues Deferred:**
None

**Technical Debt Incurred:**
None
