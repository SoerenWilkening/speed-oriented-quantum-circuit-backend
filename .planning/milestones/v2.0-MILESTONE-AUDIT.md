## Milestone Definition of Done

## Tech Debt Summary

### Non-Critical Items (5 total)

**Phase 49:**
- OPT-01 uses Python-level optimizer instead of C-level `circuit_optimize()` — functionally equivalent, could be optimized to C for performance in the future
- Real QFT sequences don't benefit from peephole optimizer (C backend already handles QFT/iQFT pairing) — optimizer benefits user-constructed sequences

**Phase 50:**
- True nested `with` blocks (`with q1: with q2:`) unsupported — pre-existing limitation (controlled AND not implemented)
- First call inside `with` runs uncontrolled body — accepted trade-off, both variants eagerly cached

**Pre-existing (unrelated to v2.0):**
- 4 uncomputation test failures (2 comparison ancilla bugs, 2 OOM) — existed before v2.0 work
