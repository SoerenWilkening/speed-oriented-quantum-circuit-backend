# Milestone v1.2: Automatic Uncomputation

**Status:** SHIPPED 2026-01-28
**Phases:** 16-20
**Total Plans:** 10

## Overview

This milestone delivers automatic uncomputation of intermediate qubits in boolean expressions. Phase 16 establishes dependency tracking infrastructure to record which intermediates were created. Phase 17 implements C-level reverse gate generation for creating adjoint circuits. Phase 18 integrates uncomputation with Python object lifetimes, enabling automatic cleanup when qbool variables go out of scope. Phase 19 extends this to quantum conditionals (the `with` statement). Phase 20 adds mode control (lazy vs eager uncomputation) and explicit user control methods.

## Milestone Summary

**Key Decisions:**

- Weak references for dependency storage — prevents circular reference memory leaks
- Creation order cycle prevention — assert parent._creation_order < self._creation_order at add-time
- Track only quantum operands — classical int operands don't need uncomputation tracking
- Empty range (start == end) is no-op — returns silently, correct behavior
- Reversed gates appended to circuit — not replaced, matches uncomputation model
- Refcount threshold of 2 (self + getrefcount) — for explicit uncompute validation
- Uncompute BEFORE restoring control state — for quantum correctness in `with` blocks
- Mode captured at creation time — prevents retroactive mode changes
- keep() is permanent — flag never cleared automatically

**Issues Resolved:**

- Dependency tracking enables safe automatic uncomputation
- LIFO cascade order verified correct for quantum state preservation
- Scope-based cleanup works correctly with Python garbage collection
- Clear error messages guide users to correct usage

**Issues Deferred:**

- Nested quantum conditionals require quantum-quantum AND (tracked in STATE.md)

**Technical Debt Incurred:**

- None — all phases clean with no TODOs or deferred items
