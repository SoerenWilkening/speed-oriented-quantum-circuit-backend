# Requirements Archive: Quantum Assembly v3.0 — Fault-Tolerant Arithmetic

**Archived:** 2026-02-18
**Defined:** 2026-02-14
**Core Value:** Write quantum algorithms in natural programming style that compiles to efficient, memory-optimized quantum circuits.

## v3.0 Requirements — Final Status

### Toffoli Addition

- [x] **ADD-01**: Ripple-carry adder (QQ) adds two quantum registers using only Toffoli/CNOT/X gates with 1 ancilla qubit — Phase 66
- [x] **ADD-02**: Ripple-carry adder (CQ) adds a classical value to a quantum register using only Toffoli/CNOT/X gates — Phase 66, Phase 73 (inline CQ with classical-bit simplification)
- [x] **ADD-03**: Controlled ripple-carry adder (cQQ) performs QQ addition conditioned on a control qubit — Phase 67
- [x] **ADD-04**: Controlled ripple-carry adder (cCQ) performs CQ addition conditioned on a control qubit — Phase 67, Phase 73 (inline cCQ with classical-bit simplification)
- [x] **ADD-05**: Subtraction works via inverse of Toffoli adder (reversed gate sequence) for all 4 variants — Phase 66, Phase 73
- [x] **ADD-06**: Carry look-ahead adder (QQ) achieves O(log n) depth using 2n-2 ancilla qubits (Draper et al. 2004) — Phase 71 (Brent-Kung variant, ~50% depth reduction)
- [x] **ADD-07**: Mixed-width Toffoli addition handles operands of different bit widths via zero-extension — Phase 66

### Toffoli Multiplication

- [x] **MUL-01**: Schoolbook multiplication (QQ) computes quantum*quantum product using Toffoli-based adders — Phase 68
- [x] **MUL-02**: Classical-quantum multiplication (CQ) computes quantum*classical using shift-and-add with Toffoli adders — Phase 68
- [x] **MUL-03**: Controlled multiplication (cQQ) performs QQ multiply conditioned on a control qubit — Phase 69
- [x] **MUL-04**: Controlled multiplication (cCQ) performs CQ multiply conditioned on a control qubit — Phase 69
- [x] **MUL-05**: Controlled add-subtract optimization reduces Toffoli count by ~50% in multiplication subroutine — Phase 72 (AND-ancilla MCX decomposition)

### Toffoli Division

- [x] **DIV-01**: Restoring division with classical divisor computes quotient and remainder using Toffoli add/sub — Phase 69
- [x] **DIV-02**: Restoring division with quantum divisor computes quotient and remainder using Toffoli add/sub — Phase 69

### Backend Dispatch

- [x] **DSP-01**: `ql.option('fault_tolerant', True)` switches all arithmetic operations to Toffoli-based implementations — Phase 67
- [x] **DSP-02**: Hot-path C functions dispatch to Toffoli or QFT sequence generators based on fault_tolerant flag — Phase 67
- [x] **DSP-03**: Toffoli-based arithmetic is the default; QFT arithmetic remains available via `ql.option('fault_tolerant', False)` — Phase 67

### Infrastructure

- [x] **INF-01**: Ancilla qubits are allocated, used, and returned to |0> state (uncomputed) correctly for all Toffoli operations — Phase 65
- [x] **INF-02**: Verification suite confirms Toffoli and QFT backends produce identical results for widths 1-8 across all operations — Phase 70
- [x] **INF-03**: Hardcoded Toffoli gate sequences for common widths eliminate generation overhead — Phase 72, Phase 73, Phase 74, Phase 75
- [x] **INF-04**: T-count reporting in circuit statistics (each Toffoli = 7 T gates) — Phase 72, Phase 74 (exact T-count with Clifford+T decomposition)
