## End-to-End Flows

### Flow 1: Lazy Mode with Context Manager ✓

```python
ql.option('qubit_saving', False)  # Default lazy mode
condition = ql.qbool(True)
with condition:
    temp = ql.qbool(False)
    # temp created with creation_scope=1, _uncompute_mode=False
# Block exit → temp automatically uncomputed
# condition survives (creation_scope=0, not registered in scope)
```

**Verified:** All steps work correctly

### Flow 2: Eager Mode Automatic Cleanup ✓

```python
ql.option('qubit_saving', True)  # Enable eager mode
a = ql.qbool(True)  # Mode captured: _uncompute_mode=True
b = ql.qbool(False)
result = a & b  # Dependencies tracked, layers captured
# GC triggers → immediate cleanup (no scope wait)
```

**Verified:** All steps work correctly

### Flow 3: Keep Method Prevents Cleanup ✓

```python
result = x & y
result.keep()  # Sets _keep_flag=True
# GC runs → no cleanup (__del__ checks flag)
result.uncompute()  # Manual cleanup still works (ignores flag)
```

**Verified:** All steps work correctly

### Flow 4: Error Flow (Use After Uncompute) ✓

```python
z = ql.qbool(True)
z.uncompute()  # Mark as uncomputed
result = z & w  # Raises ValueError
# Error: "Cannot use qbool: already uncomputed. Create a new qbool or call .keep()..."
```

**Verified:** Clear error with actionable suggestion
