# Milestone v1.6: Array & Comparison Fixes

**Status:** ✅ SHIPPED 2026-02-02
**Phases:** 34-36
**Total Plans:** 5

## Overview

v1.6 fixes three categories of bugs discovered during v1.5 exhaustive verification: array constructor and element-wise operation bugs, comparison operator inversion and boundary errors, and circuit size explosion for ordering comparisons. Phases proceed from array fixes (independent of comparison code) through comparison fixes (independent of array code) to a final verification phase that converts xfail tests to passing and confirms no regressions.

### Phase 35: Comparison Bug Fixes

**Goal**: All six comparison operators produce correct results for all value pairs and reasonable circuit sizes

## Milestone Summary

**Key Decisions:**

- Positional value + keyword width pattern for qint constructor calls in arrays (34-01)
- Comparison results persist without auto-uncompute, matching Phase 29-16 pattern (35-01)
- MSB-first qubit ordering for C backend comparison operations (35-01)
- Widened (n+1)-bit comparison applied to __lt__ (35-02)
- Right-aligned storage means index [63] is always MSB (35-02)
- LSB-aligned CNOT bit copies for widened comparisons (35-03)
- Target index formula: 64 - comp_width + i_bit for proper zero-extension (35-03)

**Issues Resolved:**

- BUG-ARRAY-INIT: Array constructor parameter swap (width passed as value)
- BUG-CMP-01: Equality/inequality inversion (dual-bug: GC gate reversal + bit-order reversal)
- BUG-CMP-02: Ordering comparison errors at MSB boundary (XOR bit-copy misalignment)
- BUG-CMP-03: Confirmed as non-issue (linear circuit growth, not exponential)

**Issues Deferred:**

- BUG-DIV-01: Division overflow for divisor >= 2^(w-1)
- BUG-MOD-REDUCE: _reduce_mod result corruption
- BUG-COND-MUL-01: Controlled multiplication corruption
- Dirty ancilla from widened comparisons (gt/le) — known limitation

**Technical Debt Incurred:**

- Stale 35-VERIFICATION.md shows gaps_found from before 35-03 resolved all issues (cosmetic only)
- Dirty ancilla from widened comparisons (2 xfail preserved) — known limitation, not correctness issue
