# Milestone v1.8: Quantum Copy, Array Mutability & Uncomputation Fix

**Status:** ✅ SHIPPED 2026-02-03
**Phases:** 41-44
**Total Plans:** 7

## Overview

Enable quantum state copying for correct binary operations, make arrays mutable via augmented assignment, and fix the uncomputation regression.

## Phases

### Phase 41: Uncomputation Fix

**Goal**: Automatic uncomputation works correctly -- expressions uncompute on scope exit without corrupting state
**Depends on**: Nothing (regression fix, highest priority)
**Plans**: 2 plans

Plans:

- [x] 41-01: Integrate .pxi files, add layer tracking, fix uncomputation regression
- [x] 41-02: Gap closure: add layer tracking to __lt__/__gt__ widened-temp comparisons

**Details:**
- Added `_start_layer`/`_end_layer`/`operation_type`/`add_dependency` tracking to all qint operations
- Fixed LAZY mode scope condition from `<=` to `<` (prevents scope-0 auto-uncompute)
- Reversed D41-01-3: layer tracking IS set on lt/gt results (no double-reversal risk)
- Identified architectural limitation: circuit optimizer parallelizes gates into shared layers, making used_layer unreliable for gate boundary tracking
- 5711 tests pass, 0 regressions

### Phase 42: Quantum Copy Foundation

**Goal**: Users can create quantum copies of qint values using CNOT-based entanglement
**Depends on**: Phase 41
**Plans**: 1 plan

Plans:

- [x] 42-01: Implement copy()/copy_onto() on qint, qbool.copy() override, and Qiskit-verified tests

**Details:**
- `qint.copy()` allocates fresh |0⟩ qubits, applies CNOT gates via Q_xor, sets layer tracking metadata
- `qint.copy_onto(target)` XOR-copies source bits onto target with width validation
- `qbool.copy()` wraps qint.copy() result preserving type
- 70 exhaustive Qiskit-verified tests (widths 1-4, all values)

### Phase 43: Copy-Aware Binary Operations

**Goal**: Binary operations on qint and qarray preserve quantum state by using quantum copy instead of classical value initialization
**Depends on**: Phase 42
**Plans**: 2 plans

Plans:

- [x] 43-01: Fix __add__, __radd__, __sub__ to use XOR-copy instead of classical init + tests
- [x] 43-02: Add missing qint ops (neg, rsub, lshift, rshift) + missing qarray ops + tests

**Details:**
- Replaced `qint(value=self.value, width=W)` with zero-init + XOR-copy pattern in __add__, __radd__, __sub__
- Added 4 new qint operations: __neg__, __rsub__, __lshift__, __rshift__
- Added 6 new qarray operations: __floordiv__, __mod__, __invert__, __neg__, __lshift__, __rshift__
- Shift=0 short-circuit optimization to avoid unnecessary mul/div circuits
- 542 tests pass, 8 xfail (pre-existing bugs)

### Phase 44: Array Mutability

**Goal**: Users can modify qarray elements in-place using augmented assignment operators
**Depends on**: Phase 41
**Plans**: 2 plans

Plans:

- [x] 44-01: Add missing qint in-place ops + qarray __setitem__ + qarray in-place wrappers
- [x] 44-02: Comprehensive mutability tests (all operators, multi-dim, operand types)

**Details:**
- Added 3 missing qint in-place operators: __ilshift__, __irshift__, __ifloordiv__
- Replaced TypeError-raising __setitem__ with working implementation (int, slice, tuple indexing)
- Added 3 missing qarray in-place wrappers: __ilshift__, __irshift__, __ifloordiv__
- 33 tests covering all 9 augmented assignment operators, 3 operand types, multi-dimensional indexing

## Milestone Summary

**Key Decisions:**

- D41-01-1: Keep inline implementations in qint.pyx (Cython 3.0.11 disallows include inside cdef class)
- D41-01-2: Use strict < for LAZY scope comparison (prevents scope-0 auto-uncompute)
- D41-02-1: Revise D41-01-3: layer tracking IS set on lt/gt results
- D41-02-2: Keep 4 pre-existing failures as-is (architectural: layer counter vs instruction counter)
- D42-01-1: Place copy methods after __invert__ (grouped with bitwise ops)
- D42-01-2: copy_onto has no layer tracking (raw CNOT op)
- D42-01-3: qbool.copy() uses cdef cast for qubit access
- D43-01-1: Mark mixed-width add/sub as xfail (pre-existing QFT off-by-one)
- D43-02-1: Mark rshift shift>0 as xfail (pre-existing BUG-DIV-02)
- D43-02-2: Add shift=0 short-circuit in lshift/rshift

**Issues Resolved:**

- Uncomputation regression: expressions not uncomputing on scope exit (fixed via layer tracking)
- Classical value reinitialization in binary ops (replaced with quantum copy)
- Missing qint operations (neg, rsub, lshift, rshift, ilshift, irshift, ifloordiv)
- Missing qarray operations (floordiv, mod, invert, neg, lshift, rshift, ilshift, irshift, ifloordiv)
- Immutable qarray elements (now support all augmented assignments)

**Issues Deferred:**

- BUG-WIDTH-ADD: Mixed-width QFT addition off-by-one (discovered in Phase 43)
- BUG-DIV-02: Floordiv incorrect for small widths (affects rshift)
- BUG-MOD-REDUCE: _reduce_mod result corruption (carry-forward)
- BUG-COND-MUL-01: Controlled multiplication corruption (carry-forward)
- Architectural limitation: layer counter vs instruction counter for uncomputation tracking

**Technical Debt Incurred:**

- 4 pre-existing uncomputation test failures remain (need instruction-counter-based tracking)
- Pre-existing *= segfault in C backend (1 skipped test in array mutability)
- Dirty ancilla in gt/le comparisons (architectural, not correctness bug)
