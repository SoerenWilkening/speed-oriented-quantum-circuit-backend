# Milestone v2.3: Hardcoding Right-Sizing

**Status:** SHIPPED 2026-02-08
**Phases:** 62-64
**Total Plans:** 4

## Overview

Benchmark-driven evaluation and right-sizing of hardcoded gate sequence generation. Measured import time, first-call generation cost, and cached dispatch overhead for all 9 operation types. Kept all addition widths 1-16 hardcoded based on data, factored out shared QFT/IQFT sub-sequences reducing generated C by 32.9%, and confirmed zero performance regression.

## Phases

### Phase 62: Measurement
**Goal**: Comprehensive benchmarks quantify the cost/benefit of hardcoded sequences for all operation types, producing a data-driven recommendation
**Depends on**: Nothing (first phase of v2.3)
**Plans**: 2 plans

Plans:
- [x] 62-01: Benchmark measurement engine (import time, first-call generation, cached dispatch)
- [x] 62-02: Evaluation and comparison report (mul/bitwise/div recommendations, amortization analysis)

**Details:**
- Import time: 192ms median (20 subprocess iterations)
- First-call generation cost for all 9 operations x 16 widths (144 data points)
- Cached dispatch: hardcoded 2-6x faster than dynamic
- Multiplication: "investigate" (48x addition cost, CQ_mul not cached)
- Bitwise: "skip" (max 288us, trivial generation)
- Division: "skip" (Python-level loop cost, not C sequence generation)
- Break-even: 3,533 cached calls or 550 first calls to recoup 192ms import

### Phase 63: Right-Sizing Implementation
**Goal**: Addition hardcoded sequences are right-sized based on Phase 62 measurements
**Depends on**: Phase 62
**Plans**: 1 plan

Plans:
- [x] 63-01: Decision document + shared QFT/IQFT factoring

**Details:**
- Decision: KEEP all addition widths 1-16 hardcoded (2-6x dispatch speedup)
- Refactored generate_seq_all.py with segmented optimization (QFT/middle/IQFT independently)
- Shared static const arrays for QQ/cQQ, init helpers for CQ/cCQ
- Reduced generated C from 79,867 to 53,598 lines (32.9% reduction)
- All 165 hardcoded sequence tests pass

### Phase 64: Regression Verification
**Goal**: End-to-end circuit generation performance is verified to have no regression
**Depends on**: Phase 63
**Plans**: 1 plan

Plans:
- [x] 64-01: Benchmark comparison and regression report

**Details:**

## Milestone Summary

**Key Decisions:**
- KEEP all addition widths 1-16 hardcoded (data-driven from Phase 62 benchmarks)
- Segmented optimization (QFT/middle/IQFT independently) for shared layer arrays
- Packed QFT layout from _generate_qft_layers() for shared static const arrays
- Const vs mutable separation: static const for QQ/cQQ, init helpers for CQ/cCQ
- 15% regression tolerance (accounts for ~8% stdev observed in Phase 62)
- Multiplication: "investigate" for future hardcoding (48x addition cost)
- Bitwise: "skip" hardcoding (trivial generation cost)
- Division: "skip" hardcoding (Python-level loop cost)

**Issues Resolved:**
- Determined break-even point for hardcoded sequence import overhead (550 first calls)
- Identified system noise patterns in benchmarking (contradictory Q_and/Q_or results)
- Reduced binary size 11.1% through shared QFT/IQFT arrays

**Issues Deferred:**
- Multiplication hardcoding implementation (evaluated as "investigate" — deferred to future milestone)
- Pre-existing test segfaults (32-bit multiplication buffer overflow)

**Technical Debt Incurred:**
- None — this milestone reduced technical debt (32.9% less generated C code)
