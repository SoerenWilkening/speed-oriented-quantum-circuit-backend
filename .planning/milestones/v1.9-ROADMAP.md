# Milestone v1.9: Pixel-Art Circuit Visualization

**Status:** SHIPPED 2026-02-03
**Phases:** 45-47
**Total Plans:** 7

## Overview

Add compact pixel-art circuit visualization that scales to 200+ qubits where existing tools fail. Provides two zoom levels (overview and detail) with automatic selection and a clean public API.

## Phases

### Phase 45: Data Extraction Bridge

**Goal**: Python code can access structured per-gate circuit data with compact qubit indexing
**Depends on**: Nothing (first phase of v1.9)
**Plans**: 2 plans

Plans:
- [x] 45-01: C extraction function (draw_data_t) and Cython bridge (draw_data() method)
- [x] 45-02: Integration tests: compaction, controls, scale, gate types

**Details:**
- Added `draw_data_t` struct in `circuit_output.h` with parallel arrays for gate data
- Implemented `circuit_to_draw_data()` in C with qubit compaction (sparse→dense remapping)
- Two-pass approach: count controls first, then fill arrays (single allocation)
- Cython `draw_data()` method returning Python dict with `num_layers`, `num_qubits`, `gates`, `qubit_map`
- 8 integration tests covering empty circuits, structure, gate types, compaction, controls, angles, 200+ qubit scale

### Phase 46: Core Renderer

**Goal**: Users can render any quantum circuit as a compact pixel-art PNG showing all gates, wires, and control connections at overview scale
**Depends on**: Phase 45
**Plans**: 3 plans

Plans:
- [x] 46-01: draw.py renderer with layout engine, wires, single-qubit gate rendering, Pillow dep, and basic tests
- [x] 46-02: Multi-qubit control lines, control dots, rendering order, and multi-qubit gate tests
- [x] 46-03: Scale testing at 200+ qubits / 10K+ gates and real circuit integration test

**Details:**
- NumPy-based renderer with 3x3 pixel cells (2px gate + 1px gap)
- 10 color-coded gate types with measurement checkerboard pattern
- Layered rendering order: wires -> control lines -> gate blocks -> control dots
- Scale-tested at 200 qubits x 10,000 gates (30000x600 px, 54MB, <5s)
- 21 tests covering dimensions, colors, wires, gates, controls, scale, memory, and integration

### Phase 47: Detail Mode & Public API

**Goal**: Users can visualize circuits at two zoom levels with automatic selection and a clean public API
**Depends on**: Phase 46
**Plans**: 2 plans

Plans:
- [x] 47-01: Detail mode rendering with gate labels (render_detail function + tests)
- [x] 47-02: Auto-zoom selection and public API (ql.draw_circuit, lazy Pillow import)

**Details:**
- 12px cell detail mode with PIL ImageDraw text labels (H, X, Rx, etc.)
- 40px left margin for qubit labels, gate borders, measurement checkerboard at detail scale
- Wire termination at measurement positions
- Auto-zoom: detail mode unless BOTH qubits > 30 AND layers > 200
- `ql.draw_circuit()` with mode override, save parameter, lazy Pillow import
- 39 cumulative tests with zero failures

## Milestone Summary

**Key Decisions:**
- Pure Python (PIL) renderer first; optimize to C only if needed
- Pixel-art over ASCII for large circuits (scales to 200+ qubits)
- NumPy array-based bulk rendering (not per-pixel ImageDraw)
- 3x3 pixel cells for overview, 12px cells for detail mode
- Rendering order: wires -> control lines -> gate blocks -> control dots (VIZ-CTRL-ORDER)
- Synthetic draw_data dicts for scale tests to avoid Cython build dependency (VIZ-SCALE-SYNTH)
- Overview only when BOTH qubits > 30 AND layers > 200 exceeded (VIZ-AUTOZOOM-BOTH)

**Issues Resolved:**
- Auto-zoom AND logic bug caught and fixed by tests during 47-02
- Ruff B904 lint error in lazy import fixed during 47-02

**Issues Deferred:**
- Angle field unused in rendering (present in draw_data for future use)
- qubit_map unused by renderers (provided for external callers)

**Technical Debt Incurred:**
- None significant — clean implementation with full test coverage
