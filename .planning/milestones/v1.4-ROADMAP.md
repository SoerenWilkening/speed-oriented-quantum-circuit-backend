# Milestone v1.4: OpenQASM Export & Verification

**Status:** SHIPPED 2026-01-30
**Phases:** 25-27
**Total Plans:** 5

## Overview

Production-quality OpenQASM 3.0 string export from the C backend and a standalone Qiskit-based verification script. The roadmap progressed bottom-up: fix and extend the C export function, add Cython/Python bindings, then build the verification script that exercises the full pipeline.

## Phases

### Phase 25: C Backend OpenQASM Export

**Goal**: Production-quality OpenQASM 3.0 string export from C backend with all gate types, multi-controlled gates, measurements, and error handling
**Depends on**: v1.3 completion (Phase 24)
**Plans**: 2 plans

Plans:
- [x] 25-01: Implement circuit_to_qasm_string() with all gate types, controls, measurements, error handling
- [x] 25-02: Fix circuit_to_opanqasm() and add circuit_to_openqasm() file-based wrapper

**Details:**
- New `circuit_to_qasm_string()` returns heap-allocated OpenQASM 3.0 string
- All 10 gate types (X, Y, Z, H, P, Rx, Ry, Rz, M, R) with correct syntax
- Multi-controlled gates: cx (1 ctrl), ccx (2 ctrl), ctrl(n) @ gate (>2 ctrl)
- Large_control array access for gates with >2 controls
- Measurement export with classical register declaration and assignment
- Dynamic buffer reallocation (512 + gates*100, doubles on overflow)
- Delegation pattern: old circuit_to_opanqasm() delegates to new implementation
- File export composes string export + file write (single source of truth)

### Phase 26: Python API Bindings

**Goal**: Users can call `ql.to_openqasm()` to get an OpenQASM 3.0 string from their circuit
**Depends on**: Phase 25
**Plans**: 2 plans

Plans:
- [x] 26-01: Create openqasm.pxd/pyx Cython wrapper, update __init__.py and setup.py
- [x] 26-02: Build package, create tests, verify full pipeline

**Details:**
- Cython .pxd/.pyx split for C function wrapping
- try-finally pattern for C memory management (guaranteed free())
- Pointer casting: <circuit_t*><unsigned long long> for circuit pointer
- NULL check after C function call to detect failures
- Module-level ql.to_openqasm() convenience function
- extras_require for optional Qiskit verification dependencies
- 6 comprehensive unit tests covering structure, content, and API exposure

### Phase 27: Verification Script

**Goal**: Standalone script that exports circuits to OpenQASM, simulates via Qiskit, and verifies outcomes match expected values
**Depends on**: Phase 26
**Plans**: 1 plan

Plans:
- [x] 27-01: Create verification script with framework, CLI, test cases, and end-to-end validation

**Details:**
- Subprocess isolation per test (ql.circuit() doesn't deallocate qubits in-process)
- 18 test cases: arithmetic (3), comparison (11), bitwise (4) — 3 skipped for C backend bugs
- MSB-first bit extraction (Qiskit convention: leftmost = highest qubit index)
- Pytest-style output with ANSI color auto-detection
- CLI with --fail-fast, --category, --show-qasm, --log flags
- Deterministic 1-shot simulation with exact match verification

## Milestone Summary

**Key Decisions:**

- Use %.17g precision for rotation angle export (preserves full double precision)
- Normalize angles to [0, 2π) before export for consistent OpenQASM output
- Delegation pattern for fixing legacy functions without breaking backward compatibility
- try-finally for C memory management in Cython (guaranteed free())
- Subprocess isolation for verification tests (C backend doesn't deallocate qubits)
- MSB-first bit extraction for Qiskit bitstring convention

**Issues Resolved:**

- All 14 bugs in legacy circuit_to_opanqasm() fixed via delegation to new implementation
- Memory-safe C string handling prevents leaks in all code paths
- Subprocess isolation prevents OOM from qubit accumulation

**Issues Deferred:**

- 3 C backend bugs documented but not fixed (subtraction underflow, multiplication segfault, less-equal comparison)
- PYTHONPATH requirement for in-place development builds
- Subprocess overhead (~30s for full suite)

**Technical Debt Incurred:**

- Subprocess isolation adds ~1.5s per test due to Python startup overhead
- 3 test cases skipped for pre-existing C backend bugs
- Source code injection via inspect.getsource() is fragile for complex generators
