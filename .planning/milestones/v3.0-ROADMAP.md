# Milestone v3.0: Fault-Tolerant Arithmetic

**Status:** SHIPPED 2026-02-18
**Phases:** 65-75
**Total Plans:** 35

## Overview

Implement Toffoli-based arithmetic (no phase/rotation gates) as an alternative backend, enabling fault-tolerant circuit generation. CDKM ripple-carry adder as foundation, schoolbook multiplication, restoring division, CLA depth optimization, all wired through `ql.option('fault_tolerant')` dispatch. MCX/CCX gate decomposition and Clifford+T hardcoded sequences complete the fault-tolerant gate pipeline.

## Milestone Summary

**Key Decisions:**
- Toffoli arithmetic as default mode (ARITH_TOFFOLI, DSP-03)
- CDKM ripple-carry before CLA (simpler, 1 ancilla)
- Division via existing Python-level composition with Toffoli add/sub
- BK CLA algorithm with 6-phase compute-copy-uncompute pattern
- AND-ancilla MCX decomposition (MCX(3)->3 CCX) instead of V-chain
- CCX->Clifford+T: exact 15-gate sequence (2H+4T+3Tdg+6CX per CCX)
- Inline CQ/cCQ generators for classical-bit gate elimination
- Separate ToffoliAdditionCDKM.c / ToffoliAdditionCLA.c split

**Issues Resolved:**
- reverse_circuit_range() self-inverse gate negation (65-01)
- Allocator block reuse for contiguous ancilla (65-02)
- CQ Toffoli MAJ/UMA simplification bug (66-03, temp-register approach)
- MCX use-after-free in run_instruction (67-02, ownership transfer)
- BK CLA ancilla uncomputation (71-05, compute-copy-uncompute pattern)

**Issues Deferred:**
- BUG-DIV-02: MSB comparison leak in division
- BUG-MOD-REDUCE: _reduce_mod result corruption
- BUG-COND-MUL-01: Controlled multiplication scope uncomputation
- BUG-CQQ-QFT: QFT controlled QQ addition incorrect at width 2+
- BUG-QFT-DIV: QFT division/modulo pervasively broken
- BUG-WIDTH-ADD: Mixed-width QFT addition off-by-one
- 32-bit multiplication segfault (buffer overflow)

**Technical Debt Incurred:**
- BK CLA subtraction falls back to RCA (carry-copy ancilla not uncomputed in reverse)
- Kogge-Stone CLA stubs return NULL (BK only implementation)
- Controlled multiplication uses scope-depth workaround (BUG-COND-MUL-01)
