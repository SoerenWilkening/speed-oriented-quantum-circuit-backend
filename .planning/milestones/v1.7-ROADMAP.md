# Milestone v1.7: Bug Fixes & Array Optimization

**Status:** ✅ SHIPPED 2026-02-02
**Phases:** 37, 40 (38-39, 41 deferred to future milestone)
**Total Plans:** 2

## Overview

v1.7 addresses one deferred bug from v1.5/v1.6 verification (division overflow) and optimizes array element-wise operations with classical values to use direct CQ_* calls instead of allocating temporary qint objects. Two bug fixes (modular reduction corruption, controlled multiplication corruption) and the final verification phase were deferred due to scaling issues and dependencies.

## Phases

### Phase 37: Division Overflow Fix

**Goal**: Division produces correct results for all operand combinations including when divisor >= 2^(w-1)
**Depends on**: Nothing (first phase in v1.7)
**Plans**: 1 plan

Plans:
- [x] 37-01-PLAN.md — Fix division loop bounds and remove xfail markers

**Details:**
- Added `max_bit_pos = self.bits - divisor.bit_length()` calculation before each classical-divisor loop
- Changed loop range from `range(self.bits - 1, -1, -1)` to `range(max_bit_pos, -1, -1)`
- Removed 13 overflow xfail markers (BUG-DIV-01 resolved)
- Documented 9 MSB comparison leak cases as separate BUG-DIV-02
- Switched test simulator from statevector to matrix_product_state for large circuits

### Phase 40: Array Classical Optimization

**Goal**: Array element-wise operations with classical values use direct CQ_* calls instead of creating temporary qint objects
**Depends on**: Nothing (independent optimization)
**Plans**: 1 plan

Plans:
- [x] 40-01-PLAN.md — Remove temporary qint allocation for classical operands in array in-place operations

**Details:**
- Removed scalar int qint wrapping in `_inplace_binary_op` (1 temp qint per operation)
- Removed list-of-int qint wrapping in `_inplace_binary_op` (1 temp qint per element per operation)
- All 6 qint in-place operators handle int natively via CQ_* C backend calls
- Out-of-place `_elementwise_binary_op` already passed int directly (no changes needed)

### Deferred Phases

The following phases were planned for v1.7 but deferred:

- **Phase 38: Modular Reduction Fix** — Beauregard-style subtract-and-check-sign attempted but hit scaling issues (duplicate qubit arguments for N>=7, 524GB+ memory for N=13). Needs fundamentally different circuit structure.
- **Phase 39: Controlled Multiplication Fix** — Not started; depends on investigation.
- **Phase 41: Verification & Regression** — Depends on phases 38-39 completion.

## Milestone Summary

**Key Decisions:**
- max_bit_pos = self.bits - divisor.bit_length() for safe division loop bounds
- Keep MSB comparison leak cases as BUG-DIV-02 (separate root cause from overflow)
- Switch to matrix_product_state simulator for large circuits
- Remove all temporary qint wrapping; qint operators handle int natively via CQ_*

**Issues Resolved:**
- BUG-DIV-01: Division overflow for divisor >= 2^(w-1) — fixed with safe loop bounds
- Unnecessary temporary qint allocation in array element-wise operations — eliminated

**Issues Deferred:**
- BUG-DIV-02: MSB comparison leak in division (9 cases per test file)
- BUG-MOD-REDUCE: _reduce_mod result corruption (scaling issues with Beauregard approach)
- BUG-COND-MUL-01: Controlled multiplication corruption (not started)

**Technical Debt Incurred:**
- Modular reduction needs fundamentally different circuit structure for larger moduli
- Controlled multiplication investigation still needed
