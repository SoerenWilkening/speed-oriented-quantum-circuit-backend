# Requirements Archive: v2.3 Hardcoding Right-Sizing

**Archived:** 2026-02-08
**Core Value:** Write quantum algorithms in natural programming style that compiles to efficient, memory-optimized quantum circuits.

## v2.3 Requirements — Final Status

### Benchmarking

- [x] **BENCH-01**: Benchmark measures package import time — 192ms median across 20 subprocess iterations
- [x] **BENCH-02**: Benchmark measures first-call generation cost for all 9 operation types at widths 1-16 — 144 data points collected
- [x] **BENCH-03**: Benchmark measures per-call dispatch overhead — hardcoded 2-6x faster than dynamic cache hit
- [x] **BENCH-04**: Benchmark produces comparison report with import time amortization analysis — break-even at 550 first calls

### Addition Right-Sizing

- [x] **ADD-01**: Data-driven decision: KEEP all addition widths 1-16 hardcoded (2-6x dispatch speedup justifies 192ms import)
- [x] **ADD-02**: Shared QFT/IQFT sub-sequences factored out — 32.9% source line reduction (79,867 to 53,598)
- [x] **ADD-03**: N/A — all widths kept hardcoded, removal path not needed
- [x] **ADD-04**: No performance regression verified — 60-94% first-call improvement, 11.1% binary size reduction, zero new test failures

### Other Operations Evaluation

- [x] **EVAL-01**: Multiplication — "investigate" (48x addition cost, CQ_mul not cached, 1.9-28.7ms per call)
- [x] **EVAL-02**: Bitwise — "skip" (max 288us, trivial generation, XOR only 13us)
- [x] **EVAL-03**: Division — "skip" (Python-level loop cost, not C sequence generation)
