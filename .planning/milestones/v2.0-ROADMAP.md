# Milestone v2.0: Function Compilation

**Status:** SHIPPED 2026-02-04
**Phases:** 48-51
**Total Plans:** 8

## Overview

`@ql.compile` decorator that captures gate sequences on first call, optimizes them, and replays with qubit remapping on subsequent calls — enabling compiled quantum functions that work inside controlled contexts and support inverse generation, nesting, and debug introspection.

## Phases

### Phase 48: Core Capture-Replay

**Goal**: Users can decorate a quantum function with `@ql.compile` and call it multiple times with different quantum arguments, getting correct results from cached gate replay with qubit remapping
**Depends on**: Nothing (first phase of v2.0)
**Plans**: 2 plans

Plans:

- [x] 48-01: Cython infrastructure (extract_gate_range, inject_remapped_gates, helpers)
- [x] 48-02: Python compile decorator, __init__.py wiring, and test suite

**Details:**
Requirements: CAP-01, CAP-02, CAP-03, CAP-04, CAP-05, CAP-06, INF-01, INF-02
Success Criteria:
  1. A function decorated with `@ql.compile` produces the same circuit output as the undecorated version on first call
  2. Calling the compiled function a second time with different qint arguments replays gates onto the new qubits
  3. The returned qint/qbool from a compiled function is usable in subsequent quantum operations
  4. Calling with different classical parameter values or different qint widths triggers re-capture
  5. Creating a new circuit via `ql.circuit()` invalidates the compilation cache

### Phase 49: Optimization & Uncomputation

**Goal**: Captured gate sequences are optimized before caching, and compiled function outputs integrate correctly with the automatic uncomputation system
**Depends on**: Phase 48
**Plans**: 2 plans

Plans:

- [x] 49-01: Gate list optimizer, optimize parameter, and stats exposure
- [x] 49-02: Uncomputation integration and comprehensive tests

**Details:**
Requirements: OPT-01, OPT-02
Success Criteria:
  1. A compiled function produces fewer gates than the undecorated version
  2. Replayed gates come from the optimized sequence
  3. A qint returned from a compiled function is correctly uncomputed when it goes out of scope inside a `with` block

### Phase 50: Controlled Context

**Goal**: Compiled functions work correctly inside `with` blocks, producing controlled gate variants
**Depends on**: Phase 49
**Plans**: 2 plans

Plans:

- [x] 50-01: Core controlled context implementation (derivation, cache key, replay remapping)
- [x] 50-02: Controlled context tests (basic, cache separation, nested with, custom key)

**Details:**
Requirements: CTL-01, CTL-02, CTL-03
Success Criteria:
  1. Calling a compiled function inside a `with qbool:` block produces controlled gates
  2. The same compiled function called outside and inside a `with` block uses separate cache entries
  3. Nested `with` blocks around compiled function calls produce the correct multi-controlled gates

### Phase 51: Differentiators & Polish

**Goal**: Compiled functions support inverse generation, debug introspection, nesting, and the feature has comprehensive test coverage
**Depends on**: Phase 50
**Plans**: 2 plans

Plans:

- [x] 51-01: Inverse generation, debug mode, and nesting depth limit (compile.py)
- [x] 51-02: Comprehensive test suite for inverse, debug, nesting, and composition

**Details:**
Requirements: INV-01, INV-02, DBG-01, DBG-02, NST-01, NST-02, INF-03
Success Criteria:
  1. Calling `.inverse()` on a compiled function produces the adjoint of the compiled sequence
  2. Debug mode prints original operation count alongside optimized gate count and reports cache hits/misses
  3. A compiled function calling another compiled function produces correct results
  4. Comprehensive test suite covers all compilation scenarios

## Milestone Summary

**Key Decisions:**

- Capture-replay approach over abstract tracing (JAX-style) — compatible with existing Cython layer, no operator method rewrites needed
- Python-level gate list optimizer instead of C-level circuit_optimize() — functionally equivalent, simpler to implement
- Controlled variant derivation from uncontrolled (not post-hoc control addition) — correct quantum semantics
- Eager caching of both controlled and uncontrolled variants — avoids redundant re-capture
- First call inside `with` block runs uncontrolled body — accepted trade-off, both variants eagerly cached

**Issues Resolved:**

- Qubit overflow in inverse function deallocation (f4786c3) — prevented negative offset in allocator_dealloc()
- Global state snapshot/restore not needed — first call is real execution, no tracing snapshot required
- Uncomputation integration worked out-of-the-box — existing _build_return_qint already correct

**Issues Deferred:**

- True nested `with qbool1: with qbool2:` raises NotImplementedError — pre-existing limitation (controlled quantum-quantum AND)
- Parametric compilation (compile once for all classical values) — deferred to v2.1+
- Resource estimation for compiled functions — deferred to v2.1+

**Technical Debt Incurred:**

- OPT-01 uses Python-level optimizer instead of C-level circuit_optimize() — could be optimized to C for performance
- 4 pre-existing uncomputation test failures (2 comparison ancilla, 2 OOM) — unrelated to v2.0
