# Milestone v1.0: Initial Release

**Status:** SHIPPED 2026-01-27
**Phases:** 1-10
**Total Plans:** 41

## Overview

This milestone transformed Quantum Assembly from a mid-restructuring prototype into a production-ready quantum programming framework. The journey followed a bottom-up approach: first establishing a solid C foundation by eliminating architectural anti-patterns (global state, memory bugs), then building variable-width integer support and complete arithmetic operations, and finally polishing the Python API with comprehensive documentation.

## Milestone Summary

**Decimal Phases:**
None in v1.0 (all phases were planned milestone work)

**Key Decisions:**
- Bottom-up restructuring (C first): Foundation must be solid before adding features
- Open source release target: Requires clean code, docs, tests
- Keep circuit compilation model: Direct execution is future; current approach works
- Use Ruff instead of Black + isort + Flake8: Single tool, 10-100x faster
- types.h as foundation module: Single source of truth for shared types
- Right-aligned q_address array layout: indices [64-width] through [63] for variable-width
- MAXLAYERINSEQUENCE for QQ_mul layer allocation: Original formula underestimated
- NumPy-style docstrings for Python API: Scientific Python conventions
- Single long page README format: GitHub-rendered documentation

**Issues Resolved:**
- Global state eliminated from C backend
- Memory bugs fixed (sizeof, NULL checks, initialization)
- Variable-width arithmetic working (1-64 bits)
- QQ_mul segfault fixed via MAXLAYERINSEQUENCE allocation
- qint_mod type checking fixed with isinstance()

**Issues Deferred:**
- Virtual environment symlinks to macOS (needs venv setup for contributors)
- 65+ Ruff violations in legacy code (tracked, not blocking)

**Technical Debt Incurred:**
- QQ_mul uses MAXLAYERINSEQUENCE (10000) instead of precise formula
- apply_merge() placeholder for future phase rotation merging
- QQ_equal/QQ_less_than C functions exist but unused (Python uses subtraction-based)
- qint_mod * qint_mod raises NotImplementedError (by design)
