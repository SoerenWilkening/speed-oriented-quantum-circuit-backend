**File structure:**
1. Module docstring: "Bitwise operation verification: AND, OR, XOR, NOT through full pipeline."
2. Imports: warnings, pytest, verify_helpers (format_failure_message, generate_exhaustive_pairs, generate_sampled_pairs), quantum_language as ql
3. `warnings.filterwarnings("ignore", message="Value .* exceeds")` at module level

4. Oracle dicts:
```python
# Python native bitwise oracles (binary ops)
BINARY_OPS = {
    "and": lambda a, b, w: a & b,
    "or":  lambda a, b, w: a | b,
    "xor": lambda a, b, w: a ^ b,
}

# Quantum-quantum operator callables
QL_OPS_QQ = {
    "and": lambda qa, qb: qa & qb,
    "or":  lambda qa, qb: qa | qb,
    "xor": lambda qa, qb: qa ^ qb,
}

# Classical-quantum operator callables
QL_OPS_CQ = {
    "and": lambda qa, b: qa & b,
    "or":  lambda qa, b: qa | b,
    "xor": lambda qa, b: qa ^ b,
}
```

5. Data generation functions:
   - `_exhaustive_binary_cases()`: widths 1-4, all pairs from `generate_exhaustive_pairs(width)`, crossed with all 3 binary op names. Returns list of `(width, a, b, op_name)`.
   - `_sampled_binary_cases()`: widths 5-6, boundary pairs `[(0,0), (0,max), (max,0), (max,max)]` merged with `generate_sampled_pairs(width, sample_size=10)`, crossed with all 3 op names. Returns list of `(width, a, b, op_name)`.
   - `_exhaustive_not_cases()`: widths 1-4, all single values 0 to 2^width-1. Returns list of `(width, a)`.
   - `_sampled_not_cases()`: widths 5-6, boundary values `[0, 1, max-1, max]` plus `generate_sampled_pairs(width, sample_size=10)` (extract unique a values from pairs). Returns list of `(width, a)`.

6. Module-level constants: `EXHAUSTIVE_BINARY`, `SAMPLED_BINARY`, `EXHAUSTIVE_NOT`, `SAMPLED_NOT`

**Test functions (6 parametrized functions):**

`test_qq_bitwise_exhaustive(verify_circuit, width, a, b, op_name)`:
- Parametrized over `EXHAUSTIVE_BINARY` with `ids=lambda args: f"w{args[0]}_{args[3]}_{args[1]}_{args[2]}"`
- Compute `expected = BINARY_OPS[op_name](a, b, width)`
- Inside circuit_builder (bind all closure vars with defaults):
  ```python
  def circuit_builder(a=a, b=b, w=width, op=QL_OPS_QQ[op_name], exp=expected):
      qa = ql.qint(a, width=w)
      qb = ql.qint(b, width=w)
      _result = op(qa, qb)
      return exp
  ```
- Call `verify_circuit(circuit_builder, width=width)` -- bitwise ops return same width as operands
- Assert with `format_failure_message(f"qq_{op_name}", [a, b], width, expected, actual)`

`test_cq_bitwise_exhaustive(verify_circuit, width, a, b, op_name)`:
- Same pattern but CQ: `_result = op(qa, b)` where b is plain int
- Use `QL_OPS_CQ` dict

`test_not_exhaustive(verify_circuit, width, a)`:
- Parametrized over `EXHAUSTIVE_NOT`
- `expected = ((1 << width) - 1) ^ a` (bitwise complement within width)
- Inside circuit_builder:
  ```python
  def circuit_builder(a=a, w=width, exp=expected):
      qa = ql.qint(a, width=w)
      _result = ~qa  # in-place NOT
      return exp
  ```
- **IMPORTANT**: NOT is in-place. The `~qa` flips bits of `qa` and returns `qa` (same object). The result sits on the same qubits as the input. Try `verify_circuit(circuit_builder, width=width)` first. If that fails because the standard extraction reads wrong bits, try `verify_circuit(circuit_builder, width=width, in_place=True)` instead. Test with a simple known case first (NOT of 0 at width 2 should be 3).
- If neither works, use a calibration approach: run a known case, inspect the full bitstring, determine the correct extraction slice, and implement a helper. Document the approach in a comment.

`test_qq_bitwise_sampled(verify_circuit, width, a, b, op_name)`:
- Parametrized over `SAMPLED_BINARY`
- Same QQ pattern as exhaustive

`test_cq_bitwise_sampled(verify_circuit, width, a, b, op_name)`:
- Parametrized over `SAMPLED_BINARY`
- Same CQ pattern as exhaustive

`test_not_sampled(verify_circuit, width, a)`:
- Parametrized over `SAMPLED_NOT`
- Same NOT pattern as exhaustive

**Failure handling:**
- All bitwise tests must pass strictly. No xfail unless a genuine C backend bug is found.
- If a bug is found, document it as BUG-BIT-XX with a descriptive comment and mark affected tests as `xfail(strict=True, reason="BUG-BIT-XX: description")`.
- If a bug is trivial (off-by-one, wrong mask), fix it inline and document the fix.

**Anti-patterns to avoid:**
- Do NOT use the same qint for both operands (e.g., `qa & qa` may trigger optimization)
- Always bind closure variables with default arguments in circuit_builder
- Always assign result to `_result` to keep qint alive
- For NOT, understand it is in-place (same register) -- result width equals input width
