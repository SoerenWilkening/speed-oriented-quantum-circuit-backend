Add two module-level helper functions (NOT methods on qarray, since the tree logic is reusable):

1. `_reduce_tree(elements, op)` - Pairwise tree reduction:
   - Takes a list of elements and a binary operator function (e.g., lambda a, b: a & b)
   - Iteratively reduces pairs: level 0 has all elements, level 1 has results of pairs, etc.
   - At each level, pair elements [0,1], [2,3], [4,5]... and apply op
   - If odd number of elements at any level, carry the last element forward unpaired
   - Continue until one element remains
   - Returns the final single element
   - Each op call uses existing qint operators which allocate fresh result qubits automatically

2. `_reduce_linear(elements, op)` - Linear chain reduction:
   - Takes a list of elements and a binary operator function
   - Sequentially: result = elements[0], then result = op(result, elements[i]) for i=1..n-1
   - Returns the final accumulated result

Both functions assume elements list has >= 2 items (caller handles edge cases).

Do NOT define these as cdef functions - use regular `def` since they take Python callable `op` parameter.

  <action>
Add three methods to the qarray cdef class:

1. `def all(self)`:
   - If empty (len == 0): raise ValueError("cannot reduce empty array")
   - If single element: return self._elements[0] directly
   - Get flat elements list: `elems = list(self._elements)` (copy to avoid mutation)
   - Check qubit_saving mode: `if _get_qubit_saving_mode(): return _reduce_linear(elems, lambda a, b: a & b)`
   - Otherwise: `return _reduce_tree(elems, lambda a, b: a & b)`
   - For qint arrays: this is bitwise AND across all elements, result is qint
   - For qbool arrays: this is logical AND, result is qbool
   - Docstring: "Reduce array with AND. Returns single element (bitwise AND for qint, logical AND for qbool)."

2. `def any(self)`:
   - Same structure as all() but uses `lambda a, b: a | b` (OR operator)
   - Docstring: "Reduce array with OR. Returns single element (bitwise OR for qint, logical OR for qbool)."

3. `def parity(self)`:
   - Same structure as all() but uses `lambda a, b: a ^ b` (XOR operator)
   - Docstring: "Reduce array with XOR. Returns single element (bitwise XOR for qint, logical XOR for qbool)."

All three methods:
- Work on flattened elements (multi-dimensional arrays automatically flatten via self._elements)
- Return the same type as input elements (qint->qint, qbool->qbool) because existing operators preserve types
- Do NOT modify the original array (operators return fresh qubits)
  </action>
