  <name>Task 1: Add sum() method to qarray</name>
  <files>src/quantum_language/qarray.pyx</files>
  <action>
Add `sum` method to qarray cdef class:

```
def sum(self, *, width=None):
```

Implementation:
1. Edge cases:
   - Empty array (len == 0): raise ValueError("cannot reduce empty array")
   - Single element: return self._elements[0] directly (no width override needed for single)

2. Width determination for result:
   - If `width` parameter provided: use it
   - If self._dtype is qbool: compute popcount width = ceil(log2(len(self) + 1)) using `import math; math.ceil(math.log2(len(self) + 1))`, minimum 1
   - Otherwise (qint): use self._width (match input element width, no auto-expansion)

3. For qbool arrays, convert elements to qint before summing:
   - Need to handle that qbool + qbool may not give correct width
   - Strategy: For qbool sum, create a custom op that handles width:
     - `lambda a, b: a + b` works because qbool inherits from qint, and + returns qint with appropriate width
   - Actually, since qbool is a 1-bit qint, `a + b` where both are qbool returns a qint (the __add__ on qint handles this)

4. Apply reduction:
   - `elems = list(self._elements)`
   - `op = lambda a, b: a + b`
   - If `_get_qubit_saving_mode()`: `result = _reduce_linear(elems, op)`
   - Else: `result = _reduce_tree(elems, op)`
   - Return result

5. Docstring: "Sum all elements. Returns qint. For qbool arrays, returns popcount. Use width= to override result width."

Note: The width parameter controls the DESIRED result width. Since qint.__add__ determines result width from operand widths, the tree reduction will use operand widths. If the user wants a wider result, they may need explicit width — but for this phase, just document that result width matches input element width unless width= is specified. If width= is provided and differs, the result may need wrapping in a new qint with the specified width — but this is a stretch goal. For now, document the width= parameter but the primary behavior is: result width = max of operand widths (determined by qint.__add__).

Actually, per CONTEXT.md: "Default result width matches input element width" — so just use the operator directly. The width= parameter is documented as optional override for future use. For now, the simple implementation is: reduce with +, return result. Document width= but note it controls desired precision.

  <name>Task 2: Add module-level functions and Python sum() support</name>
  <files>src/quantum_language/__init__.py</files>
  <action>
Add four module-level reduction functions to __init__.py, AFTER the existing `array()` function:

```python
def all(arr):
    """AND-reduce a quantum array.

    Parameters
    ----------
    arr : qarray
        Array to reduce.

    Returns
    -------
    qint or qbool
        AND reduction of all elements.
    """
    return arr.all()

def any(arr):
    """OR-reduce a quantum array.

    Parameters
    ----------
    arr : qarray
        Array to reduce.

    Returns
    -------
    qint or qbool
        OR reduction of all elements.
    """
    return arr.any()

def parity(arr):
    """XOR-reduce a quantum array.

    Parameters
    ----------
    arr : qarray
        Array to reduce.

    Returns
    -------
    qint or qbool
        XOR reduction of all elements.
    """
    return arr.parity()
```

NOTE: Do NOT add a module-level `sum()` function because it would shadow Python's built-in `sum()`. Python's built-in `sum(arr)` will work via iteration + __radd__ on qint. Users can also call `arr.sum()` directly.

Add "all", "any", "parity" to the __all__ list in the "Utilities" section.

IMPORTANT: These function names shadow Python builtins (all, any). This is intentional for the ql namespace — users call `ql.all(arr)`, not bare `all(arr)`. The shadows only apply within __init__.py itself, which is acceptable.
