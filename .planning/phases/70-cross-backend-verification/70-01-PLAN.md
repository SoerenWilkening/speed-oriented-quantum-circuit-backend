**Helper functions (module-level):**

1. `_get_num_qubits(qasm_str)` -- Extract qubit count from OpenQASM `qubit[N]` declaration. Same pattern as test_toffoli_multiplication.py.

2. `_simulate_and_extract(qasm_str, num_qubits, result_start, result_width, use_mps=False)` -- Simulate QASM and extract integer from result register.
   - If `use_mps=True`: transpile with `basis_gates=["cx", "u1", "u2", "u3", "x", "h", "ccx", "id"]`, optimization_level=0, use `AerSimulator(method="matrix_product_state")`
   - If `use_mps=False`: use `AerSimulator(method="statevector")`
   - Run with shots=1, extract bitstring, compute result from `result_start` and `result_width` positions using standard Qiskit bitstring indexing (position i = qubit N-1-i).

3. `_run_backend(backend, build_fn, width, use_mps=False)` -- Run a circuit with a specific backend and return the integer result.
   - Call `gc.collect()` then `ql.circuit()` to reset state
   - Set `ql.option("fault_tolerant", backend == "toffoli")`
   - Call `build_fn()` which returns `(result_qint, expected_value, keepalive_list)`
   - Get `result_start = result_qint.allocated_start`
   - Export QASM via `ql.to_openqasm()`
   - Set `keepalive = None` to release references
   - Call `_simulate_and_extract` with appropriate params
   - Return the actual integer result

4. `_compare_backends(build_fn, width, use_mps_toffoli=False)` -- Run both backends and return (toffoli_result, qft_result).
   - Call `_run_backend("toffoli", build_fn, width, use_mps=use_mps_toffoli)`
   - Call `_run_backend("qft", build_fn, width, use_mps=False)` (QFT never needs MPS)
   - Return the tuple

**Testing pattern for each test method:**
```python
def test_qq_add(self, width):
    if width <= 4:
        pairs = generate_exhaustive_pairs(width)
    else:
        pairs = generate_sampled_pairs(width, sample_size=20)

    failures = []
    for a, b in pairs:
        def build_fn(a=a, b=b, w=width):
            qa = ql.qint(a, width=w)
            qb = ql.qint(b, width=w)
            qc = qa + qb
            return qc, (a + b) % (1 << w), [qa, qb, qc]

        toffoli_result, qft_result = _compare_backends(build_fn, width)
        if toffoli_result != qft_result:
            failures.append(
                f"w={width} a={a} b={b}: toffoli={toffoli_result}, qft={qft_result}"
            )

    assert not failures, f"{len(failures)} cross-backend mismatches:\n" + "\n".join(failures[:20])
```

**Important details:**
- Default parameter binding in closures (`a=a, b=b, w=width`) to avoid late-binding issues
- For controlled variants (cQQ, cCQ), ONLY test ctrl=1 cross-backend equivalence exhaustively. Test ctrl=0 for a small subset (width<=2 only) to verify the no-op behavior matches, but do not exhaustively test ctrl=0 at all widths (it would double the already large test count for minimal benefit)
- For controlled additions at width 5+, use sampled pairs with smaller sample_size=10 to keep MPS simulation time reasonable
- Mark controlled addition tests at width>=5 with `@pytest.mark.slow` using the marker registered in pytest.ini
