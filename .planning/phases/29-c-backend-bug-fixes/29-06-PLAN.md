**Fix QQ_add bit-ordering (lines 183-190).**

The current code has `num_t control = bits + bit;` where `bit` iterates from bits-1 (MSB) to 0 (LSB). Analysis of 2-bit QQ_add shows the control bits are SWAPPED -- MSB control applies phases that LSB control should apply.

**2-bit trace proving the bug:**
Current code produces:
- qubit 3 (b1, MSB) controls a1 with PI and a0 with PI/2
- qubit 2 (b0, LSB) controls a0 with PI

Correct QFT addition requires:
- qubit 2 (b0, LSB) controls a0 with PI and a1 with PI/2
- qubit 3 (b1, MSB) controls a1 with PI

**Apply fix (line 187):** Change `num_t control = bits + bit;` to `num_t control = 2 * bits - 1 - bit;`

**Also check cQQ_add** (lines 318-428) for same pattern. If `bits + bit` appears as control qubit, apply same reversal: `2 * bits - 1 - bit`.
