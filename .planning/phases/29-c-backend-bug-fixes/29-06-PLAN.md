**Part B: Fix QQ_add bit-ordering**

QQ_add (lines 132-206) generates controlled-phase gates for adding one quantum register to another. The current code (lines 183-190):

```c
for (int bit = (int)bits - 1; bit >= 0; --bit) {
    for (int i = 0; i < bits - rounds; ++i) {
        num_t layer = 2 * bits + i + 2 * rounds - 1;
        num_t target = bits - i - 1 - rounds;
        num_t control = bits + bit;
        double value = 2 * M_PI / (pow(2, i + 1));
        gate_t *g = &add->seq[layer][add->gates_per_layer[layer]++];
        cp(g, target, control, value);
    }
    rounds++;
}
```

This implements QFT addition between two quantum registers. The standard QFT addition formula for `|a> + |b>`:
1. Apply QFT to `|a>` (target register, qubits 0 to bits-1)
2. For each control qubit `b_k` (from second register), apply controlled phase rotations to target qubits
3. Apply inverse QFT to `|a>`

The controlled-phase gate between control qubit `b_k` and target qubit `a_j` should have phase `2*PI / 2^(j-k+1)` when j >= k (higher target qubits get smaller phases from lower control bits).

**Analyze the current formula:**
- `bit` iterates from bits-1 down to 0. Variable `bit` represents which bit of the second operand (control).
- `control = bits + bit` maps to the actual qubit index in the second register.
- The outer loop variable `bit` goes MSB-first (bits-1, bits-2, ..., 0).
- `rounds` counts how many outer iterations have completed.
- `target = bits - i - 1 - rounds`
- `value = 2 * M_PI / pow(2, i + 1)`

**The key question:** Does this formula correctly implement the QFT addition? Compare with a known-correct reference:

For QFT addition of register B (control) into register A (target, in Fourier basis):
- For each target qubit `a_j` (j=0 is LSB), apply controlled rotation from `b_k`:
  - Phase = `2*PI * b_k / 2^(j-k+1)` for j >= k

In the current code, when `bit = bits-1` (MSB of B, first iteration, rounds=0):
- i=0: target = bits-1, value = PI (= 2*PI/2^1)
- i=1: target = bits-2, value = PI/2 (= 2*PI/2^2)
- ...
- i=bits-1: target = 0, value = 2*PI/2^bits

When `bit = bits-2` (second iteration, rounds=1):
- i=0: target = bits-2, value = PI
- i=1: target = bits-3, value = PI/2
- ...

This pattern is: MSB of B controls phases on all target qubits (largest phase on MSB target), then next-MSB of B controls phases on all but MSB target, etc.

**Compare with the reference pattern for QFT addition:**
The standard implementation has control `b_k` (k-th bit of B) applying phase `2*PI/2^(j-k+1)` to target `a_j` for j >= k.

Let me re-derive. In QQ_add, the second register qubits are at indices `bits` to `2*bits-1`. The qubit at index `bits+bit` represents bit `bit` of the second operand. If bit=0 is LSB of second operand and bit=bits-1 is MSB:

With `bit` iterating from bits-1 (MSB) to 0 (LSB):
- bit=bits-1 (MSB, weight 2^(bits-1)): controls all target qubits
  - target bits-1 (MSB): phase = PI = 2*PI/2^1
  - target bits-2: phase = PI/2 = 2*PI/2^2
  - target 0 (LSB): phase = 2*PI/2^bits

  For standard formula with k=bits-1:
  - target j=bits-1: phase = 2*PI/2^(bits-1-(bits-1)+1) = 2*PI/2^1 = PI  ✓
  - target j=bits-2: phase = 2*PI/2^(bits-2-(bits-1)+1) = 2*PI/2^0 = 2*PI  ✗!

Wait -- in the standard formula, `b_k` only applies to `a_j` where j >= k. For k=bits-1 (MSB), only j=bits-1 should receive a phase. But the code applies phases to ALL target qubits from the MSB control.

**This suggests the QQ_add implementation uses a REVERSED bit ordering** where `bit` variable does NOT directly correspond to standard k. The `bit` variable in the outer loop represents the control bit index in the register layout, but the qubit at `bits + bit` might represent a different logical bit position.

**Investigation approach:**
1. Write a standalone analysis: For 2-bit QQ_add, enumerate all gate parameters (target, control, phase) from the current code
2. Compare with the known-correct 2-bit QFT addition circuit
3. Identify the mapping discrepancy
4. Apply the minimal fix

**For 2-bit QQ_add (bits=2):**
QFT on qubits 0,1. Then:

Round 0 (bit=1, rounds=0):
- i=0: layer=3, target=1, control=3, value=PI
- i=1: layer=4, target=0, control=3, value=PI/2
rounds=1

Round 1 (bit=0, rounds=1):
- i=0: layer=5, target=0, control=2, value=PI
rounds=2

**Correct 2-bit QFT addition (|a0,a1> + |b0,b1>, 0=LSB):**
After QFT on a:
- b0 (LSB, qubit bits+0=2) controls:
  - a0 (LSB): phase 2*PI/2^1 = PI
  - a1 (MSB): phase 2*PI/2^2 = PI/2
- b1 (MSB, qubit bits+1=3) controls:
  - a1 (MSB): phase 2*PI/2^1 = PI

Current code produces:
- qubit 3 (b1, MSB) controls a1 with PI and a0 with PI/2
- qubit 2 (b0, LSB) controls a0 with PI

Correct should be:
- qubit 2 (b0, LSB) controls a0 with PI and a1 with PI/2
- qubit 3 (b1, MSB) controls a1 with PI

**The current code has the control bits SWAPPED!** MSB control (b1) is applying the phases that LSB control (b0) should apply, and vice versa.

**Fix:** Reverse the control qubit mapping. Change `num_t control = bits + bit;` to `num_t control = bits + (bits - 1 - bit);` OR equivalently `num_t control = 2*bits - 1 - bit;`. This maps:
- First iteration (bit=bits-1): control = bits + 0 = bits (LSB of second register)
- Last iteration (bit=0): control = bits + bits-1 = 2*bits-1 (MSB of second register)

This swaps which control qubit drives which set of phase rotations, matching the standard formula.
