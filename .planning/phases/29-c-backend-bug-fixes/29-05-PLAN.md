**Root cause hypothesis:**
The `CQ_mul` function (line 72-150) uses `two_complement(value, bits)` to get the binary representation of the classical multiplicand. Like BUG-04 in addition, the `two_complement()` function returns `bin[]` in MSB-first order (`bin[0]` = MSB), but `CQ_mul` likely treats it as LSB-first.

**Examine CQ_mul line 132-133:**
```c
for (int bit_int2 = 0; bit_int2 < bits; ++bit_int2) {
    value += bin[bit_int2] * 2 * M_PI / (pow(2, i + 1)) * pow(2, bits - bit_int2 - 1);
}
```

Here `bin[bit_int2]` is accessed with `bit_int2` starting from 0. With MSB-first ordering:
- `bin[0]` = MSB = the highest bit
- `bin[bits-1]` = LSB = the lowest bit

And it's multiplied by `pow(2, bits - bit_int2 - 1)`:
- When `bit_int2=0`: `pow(2, bits-1)` -- highest power for MSB. This is CORRECT for MSB-first.
- When `bit_int2=bits-1`: `pow(2, 0)` -- lowest power for LSB. This is CORRECT for MSB-first.

So actually, the multiplication formula in CQ_mul may already correctly handle the MSB-first ordering from `two_complement()`. The bug may be elsewhere.

**Debug strategy:**
1. Add printf debugging to CQ_mul to see:
   - What `bin[]` values are returned by `two_complement()`
   - What rotation values are computed
   - What `layer` values are used
   - Whether `used_layer` is set correctly

2. Also check cCQ_mul (line 244-351) which is the conditional variant -- it has a similar structure with `values[]` precomputed.

3. Compare CQ_mul structure with the working CQ_add after BUG-04 fix:
   - Both use QFT -> phase rotations -> inverse QFT
   - Addition works (after 29-03 fix), multiplication returns 0
   - The key difference is how rotations are computed

4. Check if the issue is that `used_layer` is set incorrectly, causing the QFT_inverse to be placed at the wrong position, or that layers overlap/conflict.

**Potential issues to investigate:**
a) Layer counting: `layer = 2 * bits + 2 * rounds - 1` (line 126) -- does this correctly advance past the QFT layers?
b) The rotation formula: `bin[bit_int2] * 2 * M_PI / pow(2, i+1) * pow(2, bits - bit_int2 - 1)` -- is this the correct phase for QFT-based multiplication?
c) Layer overlap: After the loop, `mul->used_layer = layer` (line 145). Does this account for all layers used?
