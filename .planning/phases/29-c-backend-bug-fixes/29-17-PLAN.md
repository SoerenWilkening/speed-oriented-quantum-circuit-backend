CQ_mul (classical * quantum multiplication) works correctly. QQ_mul (quantum * quantum) does not. Both implement Draper QFT-based multiplication but QQ_mul uses a CCP decomposition since both operands are quantum.

**Step 1: Understand what CQ_mul does correctly.**

CQ_mul (lines 75-156) implements:
1. QFT on result register
2. For each control bit (from MSB to LSB): Apply controlled phase rotations to result register, where the phase encodes the classical value's contribution at that bit position
3. Inverse QFT

The key formula in CQ_mul (line 142):
```c
value += bin[bits - 1 - bit_int2] * 2 * M_PI / (pow(2, i + 1)) * pow(2, bit_int2);
```

This accumulates phase contributions from each bit of the classical operand.

**Step 2: Compare with QQ_mul structure.**

QQ_mul (lines 157-250) does:
1. QFT on result register (block before line 203)
2. Block 1 (lines 208-214): CCP decomposition — first blocks of controlled phase rotations using CP_sequence
3. Block 2 (lines 218-235): Intermediate step with CX-all_rot-CX pattern and culmination CP gates

The issue is likely in:
- **CP_sequence target formula** (line 24): `num_t target = bits - i - 1 - rounds;` — This formula may be incorrect. In QQ_add (which works), the target follows a different pattern.
- **Block 2 culmination CP targets** (line 232): `cp(g, bits - i - 1, 3 * bits - bit_int2 - 1, value);` — The target `bits - i - 1` may need reversal.
- **all_rot function** (line 64): Uses `CP_sequence` internally, which inherits the target formula.

**Step 3: Perform paper analysis.**

For 2-bit multiplication (a * b where a=1, b=1, bits=2):
- QFT on result register (qubits 0,1)
- Block 1: CP_sequence with rounds=0,1 and controls at qubits 3,2 (2*bits-1-bit for bit=1,0)
  - CP_sequence target = bits - i - 1 - rounds = 2-i-1-rounds
  - Round 0 (bit=1, control=3): targets 1,0 (i=0: 2-0-1-0=1; i=1: 2-1-1-0=0)
  - Round 1 (bit=0, control=2): target 0 (i=0: 2-0-1-1=0, only 1 iteration since bits-rounds=1)

Compare with QQ_add (which works — verified in plan 29-09):
Read the QQ_add function in IntegerAddition.c to compare the target qubit mapping pattern. The QQ_add fix used the QFT convention where target j gets phase 2*pi*k/2^(j+1).
