**MATHEMATICAL DERIVATION (read this first, do NOT skip):**

The Draper QFT addition algorithm works as follows:
1. Apply QFT to target register (qubits 0..n-1)
2. For each control qubit representing bit position `k` (k=0 is LSB), apply controlled phase rotations to target qubits
3. Apply inverse QFT

After QFT, qubit `j` of the target register encodes exp(2*PI*i * a / 2^(j+1)) where a is the integer value (with qubit 0 as the first qubit processed by QFT).

To add the value of the control register, for each control bit `k`, we need to add phase `2*PI / 2^(j - k + 1)` to target qubit `j`, for all `j >= k`.

Looking at the QFT function in gate.c: it processes qubit 0 first (H on qubit 0), then applies CPs from higher-indexed qubits. The QFT function uses `cp(target=j, control=j+i+1, PI/2^(i+1))`. This means after QFT, qubit `j` encodes the frequency component where smaller j = higher frequency (qubit 0 gets the PI component from each bit).

**Current QQ_add code structure (lines 187-199):**
```
outer loop: bit = bits-1 downto 0, rounds = 0,1,...
inner loop: i = 0 to bits-rounds-1
  target = bits - i - 1 - rounds    <-- THIS IS WRONG
  control = bits + (bits - 1 - bit)  <-- This is correct (from plan 29-06)
  phase = 2*PI / 2^(i+1)
```

The outer loop with `bit` going from `bits-1` to `0` and control = `2*bits-1-bit` means:
- rounds=0: control=bits (LSB of control register, bit position k=0)
- rounds=1: control=bits+1 (bit position k=1)
- ...
- rounds=bits-1: control=2*bits-1 (MSB, bit position k=bits-1)

So `rounds` corresponds to the bit position `k` of the control register.

For control bit position k=rounds, we need target qubits j >= k with phase 2*PI/2^(j-k+1).
The inner loop variable `i` goes from 0 to bits-rounds-1, and the phase is 2*PI/2^(i+1).

For the phase to match: 2*PI/2^(i+1) = 2*PI/2^(j-k+1), so j = k + i = rounds + i.

**THE FIX: Change the target from `bits - i - 1 - rounds` to `rounds + i`.**

Current (WRONG):
```c
num_t target = bits - i - 1 - rounds;
```

Fixed (CORRECT):
```c
num_t target = rounds + i;
```

This is a single-line change on line 190 of IntegerAddition.c.

**Why the old target was wrong:** `bits - i - 1 - rounds` maps i=0 to the highest target qubit (bits-1-rounds), decreasing as i increases. But the Draper algorithm needs i=0 to map to the lowest valid target (rounds), increasing as i increases. The old formula was a mirror image.

**Why previous attempts failed to catch this:** Plans 29-03 through 29-08 focused on control qubit reversal (which was also needed and correctly applied in 29-06). The target qubit issue was noted in verification reports as "target qubit mapping or phase formula may be wrong" but was never actually changed.
