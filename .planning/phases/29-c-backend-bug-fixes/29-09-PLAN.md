**DERIVATION FROM ACTUAL gate.c CODE (read this first, do NOT skip):**

**Step 1: What does gate.c QFT actually do?**

The QFT function in `c_backend/src/gate.c` (lines 304-369) has this core loop structure:

```c
for (int j = 0; j < num_qubits; ++j) {
    h(&qft->seq[...], offset + j);                          // H on qubit j
    for (int i = 0; i < num_qubits - 1 - j; ++i) {
        cp(&qft->seq[...], offset + j,                      // target = j
           offset + j + i + 1, M_PI / pow(2, i + 1));       // control = j+i+1, phase = PI/2^(i+1)
    }
}
```

Concrete example for num_qubits=4:
- j=0: H(q0), CP(target=0, ctrl=1, PI/2), CP(target=0, ctrl=2, PI/4), CP(target=0, ctrl=3, PI/8)
- j=1: H(q1), CP(target=1, ctrl=2, PI/2), CP(target=1, ctrl=3, PI/4)
- j=2: H(q2), CP(target=2, ctrl=3, PI/2)
- j=3: H(q3)

**Key observation:** Qubit 0 is processed FIRST and receives phase contributions from ALL other qubits. This means qubit 0 encodes the highest-frequency Fourier component. After QFT, qubit j encodes the Fourier component for 2^(j+1) — i.e., qubit j carries the frequency that completes a full cycle every 2^(j+1) basis states.

**Step 2: What does the Draper adder require?**

To add integer b to the QFT-encoded register, the Draper adder applies phase 2*PI*b/2^(j+1) to each target qubit j. For a quantum control register, each control bit at position k (0=LSB) contributes weight 2^k, so the phase contribution of control bit k to target qubit j is:

  phase(j, k) = 2*PI * 2^k / 2^(j+1)

This is only non-trivial when j >= k (otherwise the phase is a multiple of 2*PI and has no effect).

**Step 3: Map the QQ_add loop variables to j and k**

Current QQ_add code structure (lines 187-199):
```
outer loop: bit = bits-1 downto 0, rounds = 0,1,...
inner loop: i = 0 to bits-rounds-1
  target = bits - i - 1 - rounds    <-- THIS IS WRONG
  control = bits + (bits - 1 - bit)  <-- This is correct (from plan 29-06)
  phase = 2*PI / 2^(i+1)
```

The outer loop maps `rounds` to control bit position k:
- rounds=0: control=bits (qubit index for LSB of control register, bit position k=0)
- rounds=1: control=bits+1 (bit position k=1)
- rounds=bits-1: control=2*bits-1 (MSB, bit position k=bits-1)

So `rounds = k` (the bit position of the control qubit).

**Step 4: Derive target independently from phase**

From the gate.c QFT code, target qubit j in the QFT domain means qubit index j (since QFT processes qubit 0 first). The Draper adder needs to apply gates to target qubits j = k, k+1, ..., n-1 for each control bit k.

With k = rounds and the inner loop i = 0, 1, ..., bits-rounds-1:
- We need target qubit j to range from k to n-1
- Therefore: **j = rounds + i** (when i=0, j=rounds=k; when i=bits-rounds-1, j=bits-1=n-1)

**Step 5: Derive phase independently from target**

The required phase for control bit k on target qubit j is:
  phase(j, k) = 2*PI * 2^k / 2^(j+1)

Substituting j = rounds + i and k = rounds:
  phase = 2*PI * 2^rounds / 2^(rounds + i + 1) = 2*PI / 2^(i+1)

This matches the existing code formula `2 * M_PI / pow(2, i + 1)`. The phase depends ONLY on the inner loop variable i because the offset between target qubit position (j) and control bit position (k) is exactly i, and the ratio 2^k / 2^(j+1) = 1/2^(j-k+1) = 1/2^(i+1).

**Both derivations (target and phase) are grounded in gate.c's QFT convention where qubit 0 = first processed = highest frequency, and they agree: target = rounds + i, phase = 2*PI/2^(i+1).**

**THE FIX: Change the target from `bits - i - 1 - rounds` to `rounds + i`.**

Current (WRONG):
```c
num_t target = bits - i - 1 - rounds;
```

Fixed (CORRECT):
```c
num_t target = rounds + i;
```

This is a single-line change on line 190 of IntegerAddition.c.

**Why the old target was wrong:** `bits - i - 1 - rounds` maps i=0 to the highest target qubit (bits-1-rounds), decreasing as i increases. But the Draper algorithm needs i=0 to map to the lowest valid target (rounds), increasing as i increases. The old formula was a mirror image.
