The root cause of BUG-04 is a bit-ordering mismatch between `two_complement()` and the rotation accumulation in CQ_add/cCQ_add.

**Root cause analysis:**
`two_complement(value, bits)` in `Integer.c` returns bin[] with MSB-first ordering:
- `bin[0]` = MSB (highest bit)
- `bin[n-1]` = LSB (lowest bit)

But the rotation accumulation loop in CQ_add (lines 49-56) treats `bin[bit_idx]` with bit_idx=0 as LSB (per the comments "bit_idx=0 is LSB"). This is WRONG -- the comments are incorrect, or the loop needs to reverse the indexing.

**The standard QFT addition formula:**
For adding classical value `c` to quantum register `|a>`:
1. Apply QFT to `|a>`
2. For each qubit `k` (0=LSB to bits-1=MSB), apply phase rotation `2*PI * c_k / 2^(k-j+1)` where c_k is the k-th bit of c (LSB=0)
3. Apply inverse QFT

The correct rotation for qubit index `qubit` (0=qubit0...bits-1=qubitN) from bit `bit_idx` of the classical value is:
`phase = c_bit * 2*PI / 2^(qubit - bit_idx + 1)`
where `c_bit` is the actual binary bit at position `bit_idx` counting from LSB.

Since `two_complement()` returns MSB-first, to get the LSB-first bit at index `bit_idx`, we need `bin[bits - 1 - bit_idx]`.

**Fix CQ_add rotation accumulation (around lines 49-56):**
Change:
```c
for (int bit_idx = 0; bit_idx < bits; ++bit_idx) {
    for (int qubit = bit_idx; qubit < bits; ++qubit) {
        rotations[qubit] += bin[bit_idx] * 2 * M_PI / pow(2, qubit - bit_idx + 1);
    }
```
To:
```c
for (int bit_idx = 0; bit_idx < bits; ++bit_idx) {
    for (int qubit = bit_idx; qubit < bits; ++qubit) {
        rotations[qubit] += bin[bits - 1 - bit_idx] * 2 * M_PI / pow(2, qubit - bit_idx + 1);
    }
```
