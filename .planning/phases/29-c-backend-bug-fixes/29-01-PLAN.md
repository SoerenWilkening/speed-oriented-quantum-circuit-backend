**Diagnose BUG-01:** Read qint.pyx __sub__ method (around line 852-854). The bug is that out-of-place subtraction creates a new qint with `self.value` (the initialization value), not the quantum state. For `qint(3) - qint(7)`:
- __sub__ creates `a = qint(value=self.value, width=result_width)` -- this copies 3 as the initial value
- Then `a -= other` subtracts 7 from the quantum state of `a`
- The problem is that `a` is initialized with `self.value` (which is the classical init value), but the circuit should perform `self - other` on the quantum state

**Debug approach:**
1. Read __sub__ carefully and trace through the logic
2. Compare with __add__ to understand the intended pattern for out-of-place operations
3. Also check __isub__ (in-place subtraction) to understand the inversion path
4. The fix should ensure the out-of-place subtraction correctly produces `self - other` in quantum state

**Apply minimal fix** to __sub__ to produce correct quantum subtraction result. The fix must handle unsigned wrapping (e.g., 3-7 on 4 bits = 12 = 16-4).

**Diagnose BUG-02:** Read qint.pyx __le__ method (around lines 1947-1959). The bug is in how `is_negative` and `is_zero` are combined to determine `<=`. For `5 <= 5`:
- `self -= other` gives 0
- `is_negative` checks MSB (should be 0 for zero result)
- `is_zero` checks equality to 0 (should be 1)
- `result = is_negative OR is_zero` should give 1 (true)
- But returns 0 -- so the bug is in the zero check, the OR logic, or the qbool initialization

**Debug approach:**
1. Read __le__ carefully, tracing through each step
2. Read __eq__ (equality check used for is_zero) to verify it works correctly
3. Read qbool class to understand initialization and XOR/OR behavior
4. Check if the MSB extraction bit index is correct: `self[64 - self.bits]`
5. Compare with __lt__ (less-than) to see if it has similar issues
6. Also check __ge__ and __gt__ which may share similar logic patterns

**Apply minimal fix** to __le__ (and potentially related comparison operators if same root cause). The fix must correctly combine the negative and zero checks.
