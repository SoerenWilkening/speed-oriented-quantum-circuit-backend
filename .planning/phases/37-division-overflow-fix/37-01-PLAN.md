  <files>src/quantum_language/qint_division.pxi</files>
  <action>
In `qint_division.pxi`, fix the three classical-divisor division loops (`__floordiv__`, `__mod__`, `__divmod__`) by calculating the maximum safe bit position before the loop and using it as the loop upper bound.

The fix for each of the three loops (lines 65, 176, 271):

1. Before the loop, calculate: `max_bit_pos = self.bits - divisor.bit_length()`
2. Change the loop from `for bit_pos in range(self.bits - 1, -1, -1)` to `for bit_pos in range(max_bit_pos, -1, -1)`

The formula ensures `divisor << bit_pos < 2^width` for all iterations. When `max_bit_pos < 0` (divisor uses all width bits), `range(negative, -1, -1)` produces an empty range — quotient stays 0, which is correct.

Do NOT clamp `max_bit_pos` to 0 — let it be negative so the range is naturally empty.
Do NOT add any runtime guards or conditional checks inside the loop.
Do NOT modify the quantum-divisor code paths (`_floordiv_quantum`, `_mod_quantum`, `_divmod_quantum`).
Do NOT change any other behavior (division-by-zero, negative divisor handling, etc.).

The three changes are identical in structure — just add the `max_bit_pos` calculation before each loop and change the loop range.
