**Test classes to create:**

**1. `TestToffoliClassicalDivision`:**
- `_run_toffoli_div(width, a_val, divisor)` helper: Creates circuit with explicit `ql.option("fault_tolerant", True)`, runs `q = a // divisor`, exports QASM, simulates with MPS, extracts quotient from bitstring[n-2*width:n-width].
- `test_div_exhaustive` -- parametrized over widths 2-3, all (a, divisor) pairs where divisor in [1, 2^width). Use `_mark_msb_leak_cases` for xfails.

- Width 1 is skipped because division by 1 always returns a, and width 1 has only divisor=1.

**2. `TestToffoliClassicalModulo`:**
- `_run_toffoli_mod(width, a_val, divisor)` helper: Similar but `r = a % divisor`. Remainder register position: needs investigation. In qint_division.pxi, `__mod__` returns the remainder register. The allocation order is: input(w), quotient(w), remainder(w), plus ancillae. So remainder starts at qubit 2*width. Extract from `bitstring[n-3*width:n-2*width]`.
- NOTE: Actually check what `__mod__` returns. `__mod__` calls `__divmod__` which returns (quotient, remainder). `__mod__` returns the remainder. The Python variable points to the remainder register's qubits. Need to determine the physical qubit indices of the remainder register.
- For safety, use a simpler approach: build the circuit, check all qubits, identify which register holds the correct answer. Alternatively, replicate the _simulate_and_extract pattern from test_toffoli_multiplication.py to read specific qubit ranges.
- Test at widths 2-3 exhaustive.

**3. `TestToffoliQuantumDivision`:**
- Tests `a // b` where both a and b are quantum registers.
- `_run_toffoli_qdiv(width, a_val, b_val)` helper: Creates `qa = ql.qint(a, width=w)`, `qb = ql.qint(b, width=w)`, `q = qa // qb`, simulates with MPS.
- Exhaustive at widths 2-3 (skip divisor=0 cases).

**4. `TestToffoliQuantumModulo`:**
