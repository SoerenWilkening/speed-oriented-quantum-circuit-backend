  <name>Task 1: Create compile.py with CompiledFunc, CompiledBlock, and virtual qubit mapping</name>
  <files>src/quantum_language/compile.py</files>
  <action>
  Create `src/quantum_language/compile.py` (~250 lines) implementing:

  1. **`compile` function** (the decorator): Support both `@ql.compile` and `@ql.compile()` and `@ql.compile(max_cache=N, key=..., verify=...)` forms. Use the pattern from research Pattern 6.

  2. **`CompiledFunc` class**: Wraps the decorated function. On `__call__`:
     - Classify args into quantum (qint/qbool instances) and classical (everything else)
     - Build cache key: `(tuple(classical_args), tuple(widths))` unless custom `key` function provided
     - On cache miss: call `_capture()` -- record start layer, collect parameter qubit indices, execute function normally, record end layer, call `extract_gate_range()`, build virtual mapping, cache result
     - On cache hit: call `_replay()` -- build virtual-to-real mapping from caller's qints, allocate fresh ancillas for internal qubits, set layer_floor, call `inject_remapped_gates()`, restore layer_floor, construct return qint
     - Expose `clear_cache()` method
     - Use `functools.update_wrapper` to preserve function metadata

  3. **`CompiledBlock` class**: Stores captured gate sequence with:
     - `gates`: list of gate dicts with virtual qubit indices
     - `total_virtual_qubits`: total count of virtual qubits
     - `param_qubit_ranges`: list of (start_virtual_idx, width) per quantum arg
     - `internal_qubit_count`: number of ancilla/temporary qubits
     - `return_qubit_range`: (start_virtual_idx, width) or None
     - `return_is_param_index`: int index if return value is one of the input params, None otherwise

  4. **`_build_virtual_mapping(gates, param_qubit_indices)`**: Map real qubit indices to virtual namespace. Parameter qubits mapped first (in argument order), then ancilla/temporary qubits. Return (virtual_gates, real_to_virtual, total_virtual_count). Follow research Pattern 5.

  5. **`_build_return_qint(block, virtual_to_real, start_layer, end_layer)`**: Construct a usable qint from replay-mapped qubits. Use `qint(create_new=False, bit_list=..., width=...)` pattern from qbool.copy(). Set ownership metadata: `allocated_start`, `allocated_qubits=True`, `_start_layer`, `_end_layer`, `operation_type='COMPILED'`. Follow research return value construction pattern.

  6. **Cache invalidation**: On module load, register `_clear_all_caches()` via `_register_cache_clear_hook()` from `_core`. Maintain a global list of weak references to `CompiledFunc` instances. `_clear_all_caches()` iterates and calls `clear_cache()` on each live instance.

  7. **Error handling**: Wrap function execution in try/except. On exception during capture, do NOT cache the partial result -- let the exception propagate. On next call, capture retries from scratch. Raise `TypeError` with clear message if argument type mismatch detected.

  Handle kwargs: quantum args in kwargs are also classified and their widths included in the cache key. Use `inspect.signature` to bind args for correct ordering.

  For in-place modification (return value IS one of the input params): detect this by checking if the returned qint's qubit indices overlap with any parameter's qubit indices. Set `return_is_param_index` accordingly. On replay, return the caller's qint directly instead of constructing a new one.
  </action>
