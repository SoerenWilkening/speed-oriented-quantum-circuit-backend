<objective>
Refactor legacy semi-classical logic functions to take width/bits as explicit parameter instead of reading QPU_state->Q0->MSB.

Purpose: Functions like q_and_seq, cq_and_seq, cqq_and_seq, etc. currently read `QPU_state->Q0->MSB + 1` to determine the bit width. By adding an explicit parameter, we eliminate the dependency on global state. Note: The modern width-parameterized functions (Q_and, CQ_and, Q_or, CQ_or, Q_xor, Q_not, etc.) already take explicit parameters and don't need changes.

Output: Parameterized legacy functions that work identically but without global state dependency.
</objective>
