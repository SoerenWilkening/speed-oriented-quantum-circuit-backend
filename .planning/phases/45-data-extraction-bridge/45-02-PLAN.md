  **1. test_draw_data_empty_circuit(clean_circuit):**
  - Call `clean_circuit.draw_data()` with no gates added
  - Assert returns dict with `num_layers`, `num_qubits`, `gates`, `qubit_map`
  - Assert `gates` is empty list
  - Assert no crash or segfault

  **2. test_draw_data_basic_structure(clean_circuit):**
  - Create `a = ql.qint(5, width=4)` (initializes 4-qubit register to value 5)
  - Call `clean_circuit.draw_data()`
  - Assert `num_layers > 0`
  - Assert `num_qubits > 0`
  - Assert `len(gates) > 0`
  - Assert each gate has all required keys: layer, target, type, angle, controls
  - Assert each gate's `target < num_qubits`
  - Assert each gate's `controls` is a list with all values `< num_qubits`
  - Assert `layer` values are `< num_layers`

  **3. test_draw_data_gate_types(clean_circuit):**
  - Build a circuit that exercises multiple gate types. Use `ql.qint()` for X gates, and check that gate type integers are non-negative.
  - Define `GATE_NAMES = {0: 'X', 1: 'Y', 2: 'Z', 3: 'R', 4: 'H', 5: 'Rx', 6: 'Ry', 7: 'Rz', 8: 'P', 9: 'M'}` for reference.
  - Assert all gate types are valid keys in GATE_NAMES (0-9 range).

  **4. test_draw_data_qubit_compaction(clean_circuit):**
  - Create a circuit where qubit indices are sparse. For example, create two qint variables of width 4 -- the second one will likely start at a higher qubit index. The compaction should produce dense indices 0..N-1.
  - Assert `num_qubits` equals the number of actually used qubits (not the max sparse index + 1).
  - Assert all gate targets are in range `[0, num_qubits)`.
  - Assert all control qubits are in range `[0, num_qubits)`.

  **5. test_draw_data_qubit_map(clean_circuit):**
  - Create a circuit and call `draw_data()`
  - Assert `len(qubit_map) == num_qubits`
  - Assert `qubit_map` values are sorted ascending (dense rows preserve qubit order)
  - Assert all values are non-negative integers

  **6. test_draw_data_controlled_gates(clean_circuit):**
  - Build a circuit with controlled operations (e.g., addition which produces CNOT/Toffoli gates internally).
  - Call `draw_data()` and find gates with non-empty `controls` lists.
  - Assert at least one gate has 1+ controls (proves control extraction works).
  - Assert each control qubit index is `< num_qubits`.

  **7. test_draw_data_angles(clean_circuit):**
  - Build a circuit with rotation gates (QFT-based operations like addition produce P/R gates with angles).
  - Call `draw_data()` and find gates where `type` corresponds to angle-bearing gates (P=8, R=3, Rx=5, Ry=6, Rz=7).
  - If any angle-bearing gates exist, assert their angle is a float (not NaN, not infinity).

  **8. test_draw_data_scale_200_qubits(clean_circuit):**
  - Create multiple qint variables to use 200+ qubits total. For example, create 50 qint variables of width 4 each (200 qubits).
  - Call `draw_data()` -- must not crash, segfault, or raise exception.
  - Assert `num_qubits >= 200` (or close, depending on compaction).
  - Assert `num_layers > 0`.
  - Assert result is a valid dict with expected structure.
