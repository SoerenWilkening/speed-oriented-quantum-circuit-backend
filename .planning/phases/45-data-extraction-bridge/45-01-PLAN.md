  Implement `circuit_to_draw_data()` following the pattern from the research document (Example 1). Key requirements:

  1. **Qubit remap table**: Iterate `q = 0` to `q <= circ->used_qubits` (inclusive -- `used_qubits` is max INDEX not count). Check `circ->used_occupation_indices_per_qubit[q] != 0` to detect used qubits. Build `remap[sparse] = dense` and a reverse `qubit_map[dense] = sparse`.

  2. **Empty circuit guard**: If `circ == NULL` return NULL. If `circ->used == 0`, return a valid `draw_data_t` with zero counts, NULL arrays, and the qubit_map still populated.

  3. **Two-pass approach**: First pass counts total controls across all gates. Second pass fills parallel arrays. This avoids realloc.

  4. **Control qubit extraction**: Use the same `_get_control_qubit()` helper that already exists in the file. For each control, apply remap: `data->ctrl_qubits[ctrl_idx++] = remap[_get_control_qubit(g, c)]`.

  5. **Gate type**: Cast `g->Gate` (Standardgate_t enum) to `unsigned int`.

  6. **Gate count via iteration**: Count gates by iterating layers (sum of `used_gates_per_layer[layer]` for `layer = 0..used_layer-1`), NOT `circ->used` which may include freed gates. Actually, use `circ->used` as the research suggests -- but verify by checking the existing `circuit_to_qasm_string()` pattern.

  Implement `free_draw_data()` as shown in the research (Example 2), adding `free(data->qubit_map)`.

  - MAXCONTROLS boundary: gates with `NumControls > MAXCONTROLS` use `large_control` pointer

  ```python
  def draw_data(self):
      """Extract circuit gate data as Python dict for rendering.

      Returns dict with keys:
        - num_layers (int): Number of circuit layers
        - num_qubits (int): Number of active qubits (dense, after compaction)
        - gates (list[dict]): Per-gate data with keys: layer, target, type, angle, controls
        - qubit_map (list[int]): Maps dense row index -> original sparse qubit index
      Qubit indices in gates are compacted (sparse -> dense sequential).
      """
      cdef draw_data_t *data = circuit_to_draw_data(<circuit_t*>_circuit)
      if data == NULL:
          raise RuntimeError("Failed to extract draw data")
      try:
          gates = []
          for i in range(data.num_gates):
              controls = []
              for j in range(data.gate_num_ctrl[i]):
                  controls.append(int(data.ctrl_qubits[data.ctrl_offsets[i] + j]))
              gates.append({
                  'layer': int(data.gate_layer[i]),
                  'target': int(data.gate_target[i]),
                  'type': int(data.gate_type[i]),
                  'angle': float(data.gate_angle[i]),
                  'controls': controls,
              })
          qmap = []
          for i in range(data.num_qubits):
              qmap.append(int(data.qubit_map[i]))
          return {
              'num_layers': int(data.num_layers),
              'num_qubits': int(data.num_qubits),
              'gates': gates,
              'qubit_map': qmap,
          }
      finally:
          free_draw_data(data)
  ```

  **Critical:** The `try/finally` block ensures `free_draw_data` is called even if an exception occurs during Python dict construction. This prevents memory leaks.

  **Note:** The `_circuit` global variable is already used by other methods in this class (see `gate_counts` property). Use the same cast pattern: `<circuit_t*>_circuit` or `<circuit_s*>_circuit` -- match whatever the existing code uses for `circuit_to_qasm_string` or `circuit_gate_counts` calls.
