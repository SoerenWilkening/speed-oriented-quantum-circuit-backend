**Generate hardcoded sequences by running the inline CQ/cCQ generators for value=1 at each width and transcribing the gate patterns into static C files.** The approach: value=1 in binary is bit[0]=1, bit[1..N-1]=0. Apply the simplification rules from Plan 01's inline generators to derive the exact gate sequence.

For value=1 (increment), the CQ CDKM pattern is:
- First MAJ (i=0): bit[0]=1, carry=|0> -> X(temp[0]), X(self[0]), X(carry), CCX(temp[0], carry, self[0])
- Remaining MAJs (i=1..N-1): bit[i]=0 -> skip 2 CX, emit CCX(temp[i], temp[i-1], self[i])
- UMA chain: standard (3 gates per position)
- Cleanup: X(temp[0]) only (bit[0] is the only set bit)

For value=1, the cCQ CDKM pattern is:
- First cMAJ (i=0): bit[0]=1 -> CX(temp[0], ext_ctrl) init, then standard cMAJ(carry, self[0], temp[0], ext_ctrl)
- Remaining cMAJs (i=1..N-1): bit[i]=0 -> skip 2 CCX, emit MCX(temp[i], [temp[i-1], self[i], ext_ctrl])
- cUMA chain: standard (3 gates per position)
- Cleanup: CX(temp[0], ext_ctrl) only

**Per-width files (toffoli_cq_inc_seq_N.c, N=1..8):**

Follow the exact same file structure as toffoli_add_seq_2.c:
- `#include "toffoli_sequences.h"` at top
- `#ifdef TOFFOLI_SEQ_WIDTH_N` guard (reuse same guards as QQ/cQQ)
- CQ increment: static const gate arrays for each layer, static const sequence_t
- cCQ increment: dynamic init function with caching (same pattern as cQQ for MCX large_control allocation)
- Dispatch helpers: `get_hardcoded_toffoli_CQ_inc_N(int bits)` and `get_hardcoded_toffoli_cCQ_inc_N(int bits)`

CQ qubit layout: [0..N-1]=temp, [N..2N-1]=self, [2N]=carry (same as toffoli_CQ_add)
cCQ qubit layout: [0..N-1]=temp, [N..2N-1]=self, [2N]=carry, [2N+1]=ext_ctrl

Width 1 special case: CQ is just X(0) (flip the single qubit). cCQ is CX(0, 1) (controlled flip). No carry qubit needed.

**Update toffoli_add_seq_dispatch.c:**

Add forward declarations and two new dispatch functions:
- `get_hardcoded_toffoli_CQ_inc(int bits)` — switch on bits 1-8, calls per-width CQ helpers
- `get_hardcoded_toffoli_cCQ_inc(int bits)` — switch on bits 1-8, calls per-width cCQ helpers

Use same `#ifdef TOFFOLI_SEQ_WIDTH_N` guards for conditional compilation.

**Update toffoli_sequences.h:**

Add two new public API declarations:


**Update ToffoliAddition.c:**

In `toffoli_CQ_add()`, after bounds check and bin conversion, add an early-return path:
```c
if (value == 1 && bits <= TOFFOLI_HARDCODED_MAX_WIDTH) {
    const sequence_t *hardcoded = get_hardcoded_toffoli_CQ_inc(bits);
    if (hardcoded != NULL) {
        free(bin);
        // Return a copy since CQ sequences must be caller-owned (freed after use)
        // Copy the static sequence into a fresh allocation
        return copy_hardcoded_sequence(hardcoded);
    }
}
```

Add a static `copy_hardcoded_sequence(const sequence_t *src)` helper that deep-copies a const sequence_t into a fresh malloc'd sequence_t (because CQ callers free the sequence). This must copy large_control arrays for MCX gates.

Same pattern for `toffoli_cCQ_add()` with `get_hardcoded_toffoli_cCQ_inc()`.

NOTE: For value=-1 (decrement), the inline generator handles it via two's complement (value=-1 in N bits = all 1s). No special hardcoding for negative values — the existing inline generator handles subtraction via gate reversal at the dispatch level.
