**1. Add `bk_cla_ancilla_count(int bits)` function (non-static, declared in header):**

Computes the exact total ancilla count for BK CLA:
- n-1 generate ancilla g[0..n-2]
- tree_size propagate-product ancilla (BK tree internal nodes)
- n-1 carry-copy ancilla c[0..n-2]
- Total: 2*(n-1) + tree_size

To compute tree_size, iterate through the BK tree levels:
- **Up-sweep:** At level k (k=0,1,...), stride = 2^(k+1). Merge positions are stride-1, 2*stride-1, ... that are < n-1. Each merge uses 1 tree ancilla for propagate product.
- **Down-sweep:** For each odd multiple of 2^k that was NOT already computed in the up-sweep, 1 tree ancilla for propagate product.

For reference values: bits=2 -> tree_size=0, total=2; bits=3 -> tree_size=0, total=4; bits=4 -> tree_size=1, total=7; bits=8 -> tree_size=4, total=18.

Add declaration `int bk_cla_ancilla_count(int bits);` to `toffoli_arithmetic_ops.h`.

**2. Add BK tree merge-list helper:**

Add a static helper that generates the ordered list of (merge_pos, partner_pos, level, is_downsweep) tuples for a given n. Store in a stack-allocated array (max ~128 entries, safe for bits <= 64). This helper is used by the full algorithm in Task 2.

```c
typedef struct {
    int pos;        // position being updated (target of group generate/propagate)
    int partner;    // position being merged from (source)
    int level;      // tree level (0-based)
    int is_down;    // 0 = up-sweep, 1 = down-sweep
} bk_merge_t;

static int bk_compute_merges(int n_carries, bk_merge_t *merges, int max_merges);
```

This returns the count of merges written. The up-sweep merges come first, then down-sweep merges.

**Part A: Full BK algorithm in toffoli_QQ_add_bk() for bits >= 3**

Fill in the bits >= 3 path (currently returns NULL). Use the bk_compute_merges() helper to get the ordered merge list. The algorithm has 6 phases:

**Qubit layout for toffoli_QQ_add_bk(n):**
```
[0..n-1]              = register a (source, PRESERVED)
[n..2n-1]             = register b (target, gets a+b)
[2n..3n-2]            = generate ancilla g[0..n-2]
[3n-1..3n-2+tree_sz]  = tree propagate-product ancilla (BK internal)
[3n-1+tree_sz..end]   = carry-copy ancilla c[0..n-2]
```

Use an int array `prop_src[MAX_BITS]` to track which qubit holds the current propagate source for each carry position. Initially prop_src[i] = b[i] (= n+i). After tree merges at higher levels, prop_src[pos] points to the tree ancilla holding the group propagate.

**Phase A: Initialize generate and propagate**
For i = 0 to n-2: CCX(target=g[i], ctrl1=a[i], ctrl2=b[i])
For i = 0 to n-1: CX(target=b[i], control=a[i])

**Phase B: BK prefix tree (up-sweep then down-sweep)**
Use the merge list from bk_compute_merges(). For each merge (pos, partner, level, is_down):
  Allocate next tree ancilla (tree_anc_idx, starting from qubit 3n-1).
  1. CCX(target=g[pos], ctrl1=prop_src[pos], ctrl2=g[partner])
  2. CCX(target=tree_anc[tree_anc_idx], ctrl1=prop_src[pos], ctrl2=prop_src[partner])
  Update prop_src[pos] = tree_anc[tree_anc_idx].
  Increment tree_anc_idx.

**Phase C: Copy carries**
For i = 0 to n-2: CX(target=c[i], control=g[i])

**Phase D: Reverse BK prefix tree**
Replay every gate from Phase B in REVERSE order. Since CCX is self-inverse, replaying backwards uncomputes. Iterate the merge list in reverse, emitting the same 2 CCX gates per merge in reverse within-merge order (tree ancilla CCX first, then generate CCX).

After Phase D: g[i] restored to single-bit generates, tree ancilla all |0>.

**Phase E: Uncompute propagates and generates**
For i = n-1 down to 0: CX(target=b[i], control=a[i])
For i = n-2 down to 0: CCX(target=g[i], ctrl1=a[i], ctrl2=b[i])

After Phase E: g[] all |0>, b[] restored to original, a[] unchanged.

**Phase F: Sum extraction**
For i = 0 to n-1: CX(target=b[i], control=a[i])
For i = 1 to n-1: CX(target=b[i], control=c[i-1])

Result: b[] = a+b, a[] unchanged, g[] = |0>, tree ancilla = |0>, c[] = carry values (dirty).

**Gate counting:** Before allocating, compute the exact layer count:
- Phase A: (n-1) CCX + n CX = 2n-1 layers
- Phase B: 2 gates per merge * num_merges = 2*num_merges layers
- Phase C: (n-1) CX layers
- Phase D: same as Phase B = 2*num_merges layers
- Phase E: n CX + (n-1) CCX = 2n-1 layers
- Phase F: n CX + (n-1) CX = 2n-1 layers
- Total: 3*(2n-1) + 4*num_merges + (n-1) = 7n - 4 + 4*num_merges

Use alloc_sequence() with this exact count. Cache in precompiled_toffoli_QQ_add_bk[bits].
