**3. Implement `toffoli_QQ_add_bk()` in `ToffoliAddition.c`:**

Add the Brent-Kung CLA QQ adder. This is the most complex part. Follow the algorithm from RESEARCH.md Pattern 1.

**Qubit layout (MUST match hot_path dispatch in Task 2):**
```
[0..bits-1]       = register a (source, preserved)
[bits..2*bits-1]  = register b (target, gets a+b)
[2*bits..4*bits-3] = 2*(bits-1) ancilla qubits:
    [2*bits..3*bits-2]   = generate ancilla g[0..bits-2] (bits-1 qubits)
    [3*bits-1..4*bits-3] = propagate ancilla (bits-1 qubits for tree intermediates)
Total: 4*bits - 2 qubits
```

IMPORTANT: For bits < 2, return NULL (CLA makes no sense for 1-bit; dispatch will use RCA). For bits == 2, the prefix tree is trivial (1 level).

**Algorithm phases:**

Phase A: Compute initial generate and propagate signals
- For i = 0 to bits-1: `ccx(g[i], a[i], b[i])` -- g[i] = a[i] AND b[i]
- For i = 0 to bits-1: `cx(b[i], a[i])` -- p[i] = a[i] XOR b[i], stored in-place on b[i]
- All g[i] can be on same layer; all p[i] can be on same layer.

Phase B: Brent-Kung up-sweep (compute carries at power-of-2 positions)
- For level k = 0, 1, ..., ceil(log2(bits))-1:
  - stride = 1 << (k+1)
  - For positions i where (i+1) % stride == 0 AND i < bits-1:
    - j = i - (1 << k)  (the lower merge partner)
    - Merge (g[i], p) with (g[j], p):
      - `ccx(g[i], p_qubit_for_i, g[j])` -- g[i] ^= p[i] * g[j]
      - For propagate: need ancilla for p[i]*p[j]. Use p_ancilla array.

CRITICAL: The Brent-Kung prefix tree for n bits computes carries c[0] through c[n-2]. Carry c[i] = g[i] after the prefix computation. The carry for position i tells whether there's a carry INTO position i+1.

Specifically for the merge operation (G, P) = (G_high, P_high) o (G_low, P_low):
- G_new = G_high XOR (P_high AND G_low)
- P_new = P_high AND P_low

Since XOR with AND is not directly a single gate, the standard quantum approach is:
- `ccx(target=g_high, ctrl1=p_high, ctrl2=g_low)` -- g_high ^= p_high * g_low

For the propagate update, we need p_new = p_high AND p_low. Since AND is destructive in quantum computing, we need an ancilla. However, for carry computation only the G values matter for the final carry. The P values are only needed as intermediates during the tree.

A simpler approach used in practice: only track G (generate/carry) through the tree, using P (propagate, stored in-place on b) as controls. The key insight is that after Phase A, b[i] holds p[i] = a[i] XOR b[i]. During the prefix tree, we only update g[i] values:

For each merge at level k: `ccx(g[i], b[i], g[j])` -- this computes g[i] ^= p[i] * g[j]

But this only handles the generate part. For multi-level trees, we need the compound propagate. Let me be precise:

For Brent-Kung with just generate tracking (sufficient for carry computation):
- Up-sweep: at each level, merge adjacent pairs by: `ccx(g[high], b[high], g[low])`
  This computes: g[high] ^= p[high] * g[low], which gives the carry.
  But p[high] after this is the ORIGINAL p[high], not the compound propagate.

For a 2-level tree (n=4), this means:
- Level 1: merge positions (0,1) and (2,3)
  - g[1] ^= p[1] * g[0] -- now g[1] = carry into bit 2
  - g[3] ^= p[3] * g[2] -- now g[3] = carry into bit 4 (partial)
- Level 2: merge positions (1,3)
  - g[3] ^= p[3] * g[1] -- WRONG: p[3] is still the original p[3], not compound p[3]*p[2]

This means we DO need to track compound propagates for multi-level trees. Use ancilla for propagate intermediates.

**Recommended approach (from Draper et al. 2004):**

Use 2*(bits-1) ancilla total: (bits-1) for generate values and (bits-1) for propagate intermediates.

Actually, re-reading the research more carefully: the standard quantum CLA uses the following simplified approach for Brent-Kung:

1. Compute g[i] and p[i] for all bit positions
2. Up-sweep: At each tree level, for the positions that merge, compute:
   - g_merged = g_high XOR (p_high AND g_low)  via Toffoli on g_high ancilla
   - p_merged = p_high AND p_low  (need ancilla for this since AND is destructive)
3. Down-sweep: Fill in missing carries
4. Sum computation and uncomputation

Given the complexity, implement this carefully with explicit ancilla tracking. Use (bits-1) ancilla for g and allocate additional ancilla for propagate products as needed.

ALTERNATIVE SIMPLER APPROACH: Since the generates g[i] already use ancilla, and propagates p[i] are stored in b[i], we can compute the prefix tree differently. For each merge, we need both G and P updates. Store P intermediates in additional ancilla.

For n=4 (as a concrete example):
- g ancilla: g[0], g[1], g[2] at indices [2n, 2n+1, 2n+2] = [8, 9, 10]
- p ancilla for tree: p_tree[0], p_tree[1], p_tree[2] at indices [2n+n-1, ...] = [11, 12, 13]

Phase A: Init
- g[i] = a[i] AND b[i] for i=0,1,2,3 (but we only need g[0..n-2] = g[0..2] for carries into positions 1,2,3)
- Wait -- we need g[i] for i=0..n-1 to compute carries for positions 1..n. But carry into position n is the overflow carry, which for modular addition we don't need. So we need carries c[0]..c[n-2], which come from g[0]..g[n-2].
- p[i] = a[i] XOR b[i] stored in b[i] for i=0..n-1

Actually let me step back. The Brent-Kung CLA needs to compute carry[i] for i=0..n-2 (carry into bit positions 1..n-1). Then sum[i] = p[i] XOR carry[i-1] for i=1..n-1, and sum[0] = p[0] (no carry in).

The carry for position i is: c[i] = g[i] OR (p[i] AND c[i-1]), where g[i]=a[i]*b[i], p[i]=a[i]^b[i].

The prefix tree computes this recurrence in O(log n) depth. For Brent-Kung, the up-sweep computes carries at positions 2^k - 1, and the down-sweep fills in the rest.

For implementation simplicity and correctness, implement the BK tree with ONLY generate tracking (no explicit propagate ancilla), using the standard approach from Draper:

**Use n-1 ancilla for generate values g[0]..g[n-2]. The propagate is stored in b[i].**

For the up-sweep, the merge operation "G[i:j] = G[i:m] + P[i:m]*G[m:j]" can be implemented as:
- `Toffoli(target=g[i], ctrl1=b[i], ctrl2=g[j])`

BUT this only works for single-level merges where p[i] is the original propagate. For multi-level merges, you need the COMPOUND propagate P[i:m] which is the AND of individual propagates.

Given this complexity, use the well-known Draper approach: compute compound propagates using extra ancilla, with careful uncomputation. This needs roughly 2n-2 total ancilla (n-1 for generate, plus auxiliaries for compound propagates that get reused across levels).

IMPLEMENTATION DECISION: Given the algorithmic complexity and the risk of off-by-one errors in the prefix tree, implement the BK QQ adder for the specific case structure. Build the gate list programmatically with explicit position tracking. For the initial implementation:

1. Use n-1 ancilla for g[0]..g[n-2]
2. For compound propagates in multi-level merges, use a "compute, use, uncompute" pattern where you temporarily compute p_compound in an ancilla, use it for the g merge, then uncompute it.
3. Total ancilla = 2*(n-1): n-1 for g, n-1 for temporary p products.

Allocate the sequence with generous layer count (estimate 20*bits layers for safety; set used_layer at the end).

"""Phase 71: CLA Adder - Smoke Tests.

Tests Brent-Kung CLA QQ addition at widths >= CLA_THRESHOLD (4).
Uses Qiskit AerSimulator for exhaustive statevector verification.
"""

import gc
import warnings
import pytest
from qiskit_aer import AerSimulator
import qiskit.qasm3
from verify_helpers import format_failure_message
import quantum_language as ql

warnings.filterwarnings("ignore", message="Value .* exceeds")


def _enable_toffoli_cla():
    """Enable Toffoli mode with CLA enabled (default)."""
    ql.option("fault_tolerant", True)
    ql.option("cla", True)  # Ensure CLA is not overridden


def _simulate_and_extract(qasm_str, num_qubits, result_start, result_width):
    """Same helper as test_toffoli_addition.py."""
    circuit = qiskit.qasm3.loads(qasm_str)
    if not circuit.cregs:
        circuit.measure_all()
    simulator = AerSimulator(method="statevector")
    job = simulator.run(circuit, shots=1)
    result = job.result()
    counts = result.get_counts()
    bitstring = list(counts.keys())[0]
    msb_pos = num_qubits - result_start - result_width
    lsb_pos = num_qubits - 1 - result_start
    result_bits = bitstring[msb_pos : lsb_pos + 1]
    return int(result_bits, 2)


class TestBKQQAddition:
    """Test Brent-Kung QQ addition at CLA-eligible widths."""

    @pytest.mark.parametrize("width", [4, 5, 6])
    def test_bk_qq_add_exhaustive(self, width):
        """Exhaustive test: a + b mod 2^width for all input pairs."""
        _enable_toffoli_cla()
        modulus = 1 << width
        failures = []

        for a_val in range(modulus):
            for b_val in range(modulus):
                expected = (a_val + b_val) % modulus
                ql.init(2 * width)
                qa = ql.qint(width, a_val)
                qb = ql.qint(width, b_val)
                qa += qb
                qasm_str = ql.qasm()
                num_qubits = ql._get_num_qubits()
                # Result in qa (self), starts at allocated_start of qa
                result_start = 0  # qa is first allocated
                actual = _simulate_and_extract(qasm_str, num_qubits, result_start, width)
                if actual != expected:
                    failures.append(f"  {a_val} + {b_val}: expected {expected}, got {actual}")
                gc.collect()
