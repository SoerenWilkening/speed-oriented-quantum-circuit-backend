**1. Implement `toffoli_QQ_add_ks()` in ToffoliAddition.c:**

Kogge-Stone prefix tree for QQ addition. This is the depth-optimized variant.

**Qubit layout:**
```
[0..bits-1]       = register a (source, preserved)
[bits..2*bits-1]  = register b (target, gets a+b)
[2*bits..]        = ancilla qubits for KS tree
Total: 2*bits + ks_ancilla_count(bits)
```

Kogge-Stone tree structure:
- At each level k (0 to ceil(log2(n))-1), EVERY position i >= 2^k merges with position i - 2^k
- This produces n*ceil(log2(n)) merge operations total
- Each merge may need ancilla for propagate products

The KS tree is simpler to implement than BK because it has a regular pattern: at level k, every position i where i >= 2^k does a merge. No up-sweep/down-sweep asymmetry.

Key differences from BK:
- More ancilla: approximately n*ceil(log2(n)) for intermediate generate/propagate values
- Fewer depth levels: ceil(log2(n)) vs 2*ceil(log2(n))-2
- More regular structure (easier to implement correctly)

Cache the KS QQ sequences (precompiled_toffoli_QQ_add_ks[65]).

Compute the exact ancilla count for a given n before allocation. Define a helper:
```c

    /* Count ancilla needed for Kogge-Stone prefix tree */
    /* n-1 for initial generate values + additional for tree intermediates */
    /* ... compute based on tree structure ... */
}
```

**2. Implement `toffoli_CQ_add_bk()` and `toffoli_CQ_add_ks()` in ToffoliAddition.c:**

Use the temp-register approach (same pattern as existing `toffoli_CQ_add`):
1. X-init temp register with classical value bits
2. Run the QQ CLA adder (BK or KS) on temp + self registers
3. X-cleanup temp register (QQ CLA preserves the a-register)

**CQ BK qubit layout:**
```
[0..bits-1]       = temp register (initialized to classical value, cleaned to |0>)
[bits..2*bits-1]  = self register (target, gets self + value)
[2*bits..4*bits-3] = CLA ancilla (2*(bits-1) for BK)
Total: 4*bits - 2 qubits
```

**CQ KS qubit layout:**
```
[0..bits-1]       = temp register
[bits..2*bits-1]  = self register (target)
[2*bits..]        = CLA ancilla (ks_ancilla_count for KS)
Total: 2*bits + ks_ancilla_count(bits)
```

These are NOT cached (value-dependent, like existing CQ). Caller must free via toffoli_sequence_free().

The implementation for CQ CLA follows the EXACT same pattern as `toffoli_CQ_add()`:
- Count x_count (number of 1-bits in value)
- Allocate sequence with 2*x_count + CLA_layers layers
- Phase 1: X gates on temp register
- Phase 2: Copy the gate generation logic from `toffoli_QQ_add_bk()` / `toffoli_QQ_add_ks()` but inline it (or call a shared internal helper that takes the sequence and layer pointer)

HOWEVER: User locked decision says "each variant is inline/self-contained (no shared generate/propagate abstraction)." So each CQ variant should contain its own complete CLA logic, not call the QQ function. BUT we can have an internal static helper for the core BK/KS tree logic that is NOT a generate/propagate abstraction but rather a "emit BK tree gates into sequence" helper. This is a code organization choice, not a shared abstraction in the API sense.

Pragmatic approach: Create internal static helpers `_emit_bk_tree(seq, layer, bits, a_start, b_start, anc_start)` and `_emit_ks_tree(seq, layer, bits, a_start, b_start, anc_start)` that emit the CLA tree gates. These are used by both QQ and CQ variants. This is NOT a "shared generate/propagate abstraction" -- it's just DRY code within the same file. The "inline/self-contained" constraint means BK and KS should not share a common tree abstraction with each other, but QQ and CQ of the SAME variant can share an internal helper.

**3. Add CQ CLA dispatch in `hot_path_add_cq()`:**

In the uncontrolled Toffoli CQ path (the `} else {` block after `if (controlled)` inside `ARITH_TOFFOLI`), add CLA dispatch BEFORE the existing RCA code:

For the uncontrolled CQ path, the CLA dispatch needs to:
1. Check `cla_override == 0 && self_bits >= CLA_THRESHOLD`
2. Calculate total ancilla needed: self_bits (temp) + cla_ancilla_count
3. Allocate the combined ancilla block
4. Build the qubit array with temp in [0..bits-1], self in [bits..2*bits-1], CLA ancilla in [2*bits..]
5. Call `toffoli_CQ_add_bk()` or `toffoli_CQ_add_ks()` based on qubit_saving
6. Run, free sequence and ancilla

Layout for CQ CLA:
```
tqa[0..bits-1]       = temp ancilla (from allocated block)
tqa[bits..2*bits-1]  = self_qubits (target)
tqa[2*bits..]        = CLA ancilla (from allocated block, after temp)
```

Total allocation: self_bits + cla_ancilla_count (temp + CLA ancilla in one block)

Silent fallback to existing RCA CQ code if allocation fails.

        """KS QQ addition: all input pairs."""
        ql.option("fault_tolerant", True)

        ql.option("cla", True)
        modulus = 1 << width
        failures = []
        for a_val in range(modulus):
            for b_val in range(modulus):
                expected = (a_val + b_val) % modulus
                ql.init(2 * width)
