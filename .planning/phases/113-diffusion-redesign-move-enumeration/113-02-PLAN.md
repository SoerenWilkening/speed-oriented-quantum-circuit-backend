Purpose: The current itertools.combinations enumeration is intractable for d_max=24 (KNK endgame). The counting circuit sums validity bits into a count register and dispatches rotations via comparison, achieving linear gate complexity.

**Step 2: Implement `_counting_diffusion`** as a new method on QWalkTree in walk.py:

```python
def _counting_diffusion(self, depth):
    """Counting-based variable diffusion -- O(d_max) gate complexity.

    Replaces itertools.combinations enumeration with:
    1. Sum validity bits into count register via qarray.sum()
    2. For each d_val in 1..d_max: compare count==d_val, then controlled rotations
    3. S_0 reflection (unchanged)
    4. Forward rotations (mirror of step 2)
    5. Uncompute count register
    """
```

Implementation details:
- Allocate validity qbools and evaluate children (same as old code, reuse `_evaluate_children`)
- Create validity qarray: `ql.qarray(validity, dtype=ql.qbool)` -- wrap list of qbool objects
- Compute count: `count = validity_arr.sum(width=count_width)` where `count_width = ceil(log2(d_max + 1))`
- If qarray.sum() has issues with qbool elements at width, try promoting to 1-bit qints first (Open Question 1 from research)
- For dispatch loop (d_val = 1 to d_max):
  - `cond = (count == d_val)` -- returns qbool
  - Get cond's physical qubit: `cond_qubit = int(cond.qubits[63])`
  - Build ctrl_qubits = `[h_qubit_idx, cond_qubit]` (avoid nested `with qbool:` per Pitfall 2)
  - Use `_emit_multi_controlled_ry` for parent-child Ry split with these controls
  - For cascade: each child i's cascade rotation additionally controlled on `validity[i]` physical qubit. Use `_emit_multi_controlled_ry` with ctrl_qubits = `[h_qubit_idx, cond_qubit, validity_qubit_i]` (3 controls -- `_emit_multi_controlled_ry` handles arbitrary depth via recursive peeling)
- S_0 reflection: identical to old code (`with h_control: diffusion(h_child, branch_reg)`)
- Forward U: mirror of U_dagger with positive angles
- Uncompute count: use `@ql.compile(inverse=True)` wrapper around the sum computation, call `.inverse()` after rotations
- Uncompute validity: reuse `_uncompute_children` (same as old code)

**Step 3: Wire QWalkTree._variable_diffusion to call _counting_diffusion:**
- Rename old method to `_variable_diffusion_legacy` (keep for equivalence testing)
- Make `_variable_diffusion` call `_counting_diffusion`

**CRITICAL:** Use `_emit_multi_controlled_ry` with explicit control qubit lists instead of nested `with qbool:` contexts to avoid the known nesting limitation. The h[depth] control, count==d_val condition, and validity[i] per-child control are all passed as ctrl_qubits.
