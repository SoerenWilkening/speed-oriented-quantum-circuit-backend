  <action>
Add Doxygen-style comments to arithmetic_ops.h and comparison_ops.h.

For arithmetic_ops.h, document:
- @file comment explaining purpose (width-parameterized arithmetic operations)
- QQ_add, cQQ_add, CQ_add, cCQ_add: Addition operations
- QQ_mul, cQQ_mul, CQ_mul, cCQ_mul: Multiplication operations
- Parameter documentation: bits (width), value (for classical ops)
- Return value: sequence_t* (caller owns)
- Note about OWNERSHIP

Example:
```c
/**
 * @file arithmetic_ops.h
 * @brief Width-parameterized quantum arithmetic operations.
 *
 * Provides addition and multiplication for quantum integers.
 * All operations are width-parameterized (bits parameter).
 *
 * OWNERSHIP: Returned sequence_t* is owned by caller.
 * Functions cache results in precompiled_*_width[] arrays.
 */

/**
 * @brief Quantum-quantum addition for variable-width integers.
 *
 * Generates addition circuit: A += B where A, B are quantum integers.
 *
 * @param bits Width of operands (1-64)
 * @return sequence_t* Addition instruction sequence (caller owns)
 *
 * @note Result cached in precompiled_QQ_add_width[bits]
 */
sequence_t *QQ_add(int bits);
```

For comparison_ops.h, document:
- QQ_equal: Quantum equality check
- QQ_less_than: Quantum less-than comparison
- CQ_equal: Classical-quantum equality
