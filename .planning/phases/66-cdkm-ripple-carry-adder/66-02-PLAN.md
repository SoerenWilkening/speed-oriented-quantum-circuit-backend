**Test Class 1: TestToffoliQQAddition** (Success Criterion 1)
- Parametrize over widths 1, 2, 3, 4.
- For each width, test ALL input pairs: `a` in `range(2**width)`, `b` in `range(2**width)`.
- Each test:
  1. `ql.circuit()`, `ql.option('fault_tolerant', True)`
  2. Create `qa = ql.qint(a, width=width)`, `qb = ql.qint(b, width=width)`
  3. `qc = qa + qb`
  4. Expected: `(a + b) % (2**width)`
  5. Use `verify_circuit` fixture to simulate and check.
- Also test that the circuit contains ONLY CCX/CX/X gates (no CP/H/P gates). Use `ql.gate_counts()` or `ql.stats()` to check. If `gate_counts_t` is accessible via Python: verify `p_gates == 0`, `h_gates == 0`. If not accessible directly, verify via OpenQASM export (no `cp`, `h`, or `p` gates in the QASM string).

**Test Class 2: TestToffoliCQAddition** (Success Criterion 2)
- Parametrize over widths 1, 2, 3, 4.
- For each width, test ALL input pairs: `a` in `range(2**width)`, `b` (classical) in `range(2**width)`.
- Each test:
  1. `ql.circuit()`, `ql.option('fault_tolerant', True)`
  2. Create `qa = ql.qint(a, width=width)`
  3. `qa += b` (in-place classical addition)
  4. Expected: `(a + b) % (2**width)`
  5. Simulate and verify.

**Test Class 3: TestToffoliSubtraction** (Success Criterion 3)
- Parametrize over widths 1, 2, 3, 4.
- **QQ subtraction:** For all pairs a, b: `qa = ql.qint(a, width=width)`, `qb = ql.qint(b, width=width)`, `qc = qa - qb`. Expected: `(a - b) % (2**width)`.
- **CQ subtraction:** For all pairs a, b: `qa = ql.qint(a, width=width)`, `qa -= b`. Expected: `(a - b) % (2**width)`.

**Test Class 4: TestToffoliMixedWidth** (Success Criterion 4)
- Test width combinations: (2,3), (3,4), (4,6).
- For each combination (w_a, w_b), test representative input pairs:
  - (0, 0), (1, 1), (max_a, 1), (1, max_b), (max_a, max_b)
  - Where max_a = 2**w_a - 1, max_b = 2**w_b - 1.
- Each test: `qa = ql.qint(a, width=w_a)`, `qb = ql.qint(b, width=w_b)`, `qc = qa + qb`. Result width = max(w_a, w_b). Expected: `(a + b) % (2**result_width)`.
- Also test mixed-width subtraction for the same combinations.

**Test Class 5: TestToffoliAncillaLifecycle** (Success Criterion 5)
- Test that ancilla is properly managed:
  1. Perform two sequential additions: `a + b`, then `c + d`. If ancilla leaked from the first, the second would get a dirty qubit and produce wrong results. Verify both results are correct.
  2. Test with the allocator stats (if accessible via Python). After addition, check that `ancilla_allocations` increased but `current_in_use` did not (ancilla was freed).

**Test Class 6: TestToffoliGateTypeVerification** (Success Criterion 1 & 2 -- gate purity)
- For widths 2, 3, 4: perform Toffoli QQ addition, export to OpenQASM, parse the QASM string, verify it contains ONLY `cx`, `ccx`, and `x` gate instructions (no `h`, `p`, `cp`, `rz`, etc.).
- Use string matching on the QASM output: check that every gate line matches one of the allowed Toffoli gates.

**Test Class 7: TestToffoliQFTFallback**
- Verify that controlled operations still work when fault_tolerant is True (they fall back to QFT).
- Test: `ql.option('fault_tolerant', True)`, create controlled addition scenario (via `with` block or controlled context), verify it produces correct results using QFT-style gates.
