<objective>
Replace the single-qubit freed stack in the qubit allocator with a block-based free-list that supports contiguous multi-qubit allocation and reuse with adjacent-block coalescing. Add comprehensive C-level unit tests.

Purpose: Toffoli arithmetic (CDKM adder) requires multi-qubit ancilla blocks. The current allocator only reuses single freed qubits, wasting memory when blocks of 2+ qubits are freed and later needed. This fix enables efficient ancilla block lifecycle for all subsequent Toffoli phases.
Output: Updated qubit_allocator.h/.c with block free-list, new test_allocator_block.c.
</objective>

  <action>
**In `qubit_allocator.h`:**

Add a new struct before `qubit_allocator_t`:
```c
typedef struct {
    qubit_t start;
    num_t count;
} qubit_block_t;
```

Replace the freed-stack fields in `qubit_allocator_t`:
- Remove: `num_t freed_count;`, `qubit_t *freed_stack;`, `num_t freed_capacity;`
- Add: `qubit_block_t *freed_blocks;` (sorted by start index), `num_t freed_block_count;`, `num_t freed_block_capacity;`

Keep `allocator_stats_t stats;` field in the same position. Keep `#ifdef DEBUG_OWNERSHIP` fields at the end of the struct (important: Cython compatibility -- Cython accesses stats through accessor function, never dereferences struct directly, so field reordering before stats is safe as long as Cython only uses `allocator_get_stats()`).

**In `qubit_allocator.c`:**

1. **Update `FREED_STACK_INITIAL_SIZE` to `FREED_BLOCKS_INITIAL_SIZE`** (or keep the value 32).

2. **`allocator_create()`:** Replace `freed_stack` allocation with `freed_blocks`:
   ```c
   alloc->freed_blocks = malloc(FREED_BLOCKS_INITIAL_SIZE * sizeof(qubit_block_t));
   alloc->freed_block_count = 0;
   alloc->freed_block_capacity = FREED_BLOCKS_INITIAL_SIZE;
   ```

3. **`allocator_destroy()`:** Replace `free(alloc->freed_stack)` with `free(alloc->freed_blocks)`.

4. **`allocator_alloc()`:** Replace the single-qubit reuse logic (lines 93-98) with block-based search:
   ```
   For any count (1 or more):
   a) Search freed_blocks for first block where block.count >= count (first-fit)
   b) If found:
      - start_qubit = block.start
      - If block.count == count: remove block from array (memmove remaining down)
      - If block.count > count: shrink block (block.start += count, block.count -= count)
      - Return start_qubit
   c) If not found: fall through to fresh allocation from next_qubit (existing code)
   ```
   The "no defragmentation" rule means we never try to combine non-adjacent freed blocks to satisfy a request. We simply allocate fresh.

5. **`allocator_free()`:** Replace the single-qubit push logic (lines 171-187) with block insertion + coalescing:
   ```
   For any count (1 or more):
   a) Find insertion point in freed_blocks to maintain sort by start index
      (linear scan is fine -- freed_block_count is small, allocator is not hot path)
   b) Insert new block (start, count) at that position (memmove to make room)
   c) Coalesce with next block: if new_block.start + new_block.count == next_block.start,
      merge (new_block.count += next_block.count, remove next_block)
   d) Coalesce with previous block: if prev_block.start + prev_block.count == new_block.start,
      merge (prev_block.count += new_block.count, remove new_block)
   e) Expand freed_blocks array if freed_block_count reaches freed_block_capacity (realloc to 2x)
   ```

6. **Keep `allocator_get_stats()`, `allocator_reset_stats()`, `circuit_get_allocator()` unchanged.**

7. **Keep `#ifdef DEBUG_OWNERSHIP` code unchanged** -- `allocator_set_owner` and `allocator_get_owner` work per-qubit and are independent of the block free-list.

Important implementation notes:
- Single-qubit operations (count=1) are just blocks of size 1. No special case needed -- the same code path handles both.
- The `allocator_free()` signature does NOT change: `int allocator_free(qubit_allocator_t *alloc, qubit_t start, num_t count)`. Research confirms all three callers already pass (start, count).
- Use `memmove()` (not `memcpy()`) for overlapping array shifts.
