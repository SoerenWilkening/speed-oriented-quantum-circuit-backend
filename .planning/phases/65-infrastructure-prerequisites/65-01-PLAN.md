<objective>
Fix the GateValue negation bug in both `reverse_circuit_range()` and `run_instruction()` so self-inverse gates (X, Y, Z, H, M) retain their GateValue when circuits are reversed or instructions are inverted. Add C-level unit tests verifying correctness for all gate categories.

Purpose: Toffoli circuits use X/CX/CCX gates extensively. Without this fix, reversing a Toffoli circuit (for uncomputation) would corrupt GateValue from 1 to -1, causing silent computation errors. The `run_instruction()` fix prevents the same latent bug from manifesting when Toffoli sequences are inverted in Phase 66+.
Output: Patched execution.c, new test_reverse_circuit.c, updated Makefile.

  <name>Task 1: Fix GateValue negation for self-inverse gates in execution.c</name>
  <files>c_backend/src/execution.c</files>
  <action>
Two functions in execution.c have the same bug: they negate GateValue unconditionally, which corrupts self-inverse gates. Apply the same fix to both.

**In `reverse_circuit_range()` (line 84):**
Replace:
```c
g.GateValue = -g.GateValue; // Micro-opt: pow(-1,1) is always -1 (Phase 61)
```
With an inline switch/case per user decision (no helper function):
```c
// Invert gate value only for non-self-inverse gates (Phase 65)
switch (g.Gate) {
    case X:
    case Y:
    case Z:
    case H:
    case M:
        // Self-inverse gates: GateValue unchanged
        break;
    default:
        // Phase/rotation gates (P, Rx, Ry, Rz, R): negate for inversion
        g.GateValue = -g.GateValue;
        break;
}
```

**In `run_instruction()` (line 48):**
Replace:
```c
g.GateValue *= (invert ? -1.0 : 1.0); // Micro-opt: avoid pow() (Phase 61)
```
With the same switch/case pattern, but only when `invert` is true:
```c
// Invert gate value only for non-self-inverse gates (Phase 65)
if (invert) {
    switch (g.Gate) {
        case X:
        case Y:
        case Z:
        case H:
        case M:
            // Self-inverse gates: GateValue unchanged
            break;
        default:
            // Phase/rotation gates: negate for inversion
            g.GateValue = -g.GateValue;
            break;
    }
}
```

Update the comment on line 83 of reverse_circuit_range to read:
```c
// Invert the gate value for rotation gates; self-inverse gates keep original value
```

Note: The run_instruction() fix goes beyond the strict CONTEXT.md scope (which only mentions reverse_circuit_range), but the research identifies this as the identical bug that will become critical when Toffoli sequences are inverted in Phase 66. Fixing now per Claude's discretion for "additional safety checks deemed necessary." For QFT, this is a no-op: P gates still get negated (correct), H gates have GateValue=0 so -0.0==0.0 (no behavior change).
