<objective>
Add NULL checks after all malloc/calloc operations in C backend

Purpose: The codebase has ~50+ allocation calls with no error handling. If memory allocation fails, the code immediately dereferences NULL, causing segmentation faults. This makes the library unreliable for production use.

Output: Every allocation is followed by a NULL check with appropriate error handling (return NULL for functions that return pointers, or early return for void functions).
</objective>
