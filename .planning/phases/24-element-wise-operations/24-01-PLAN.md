**1. Helper methods:**

`_validate_shape(self, other)` — Validates `self._shape == other._shape`, raises `ValueError` with message: `f"Shape mismatch for element-wise operation: cannot operate on arrays with shapes {self._shape} and {other._shape}"`.

`_elementwise_binary_op(self, other, op_func, result_dtype=None)` — Generic element-wise operation:
- If `other` is `int` or `qint`: scalar broadcast — convert int to `qint(other, width=self._width)`, apply `op_func(elem, scalar)` for each element
- If `other` is `qarray`: validate shapes with `_validate_shape`, apply `op_func(self._elements[i], other._elements[i])` for each i
- Else: return `NotImplemented`
- Create result via `self._create_view(result_elements, self._shape)`
- If `result_dtype is not None`: set `result._dtype = result_dtype` and `result._width = 1` (for qbool comparisons)
- Return result

`_inplace_binary_op(self, other, iop_name)` — Generic in-place operation:
- `iop_name` is a string like `"__iadd__"` — used to call the in-place method on each element via `getattr(elem, iop_name)(operand)`
- If `other` is `int` or `qint`: scalar broadcast — convert int to `qint(other, width=self._width)`, call in-place on each element
- If `other` is `qarray`: validate shapes, call in-place on corresponding elements
- Else: return `NotImplemented`
- Return `self`

**Important Cython note:** For isinstance checks, use `type(other) == int or isinstance(other, qint)` pattern consistent with existing qint.pyx. For qint construction from int, use `qint(other, width=self._width)`.

**2. Arithmetic operators (section comment: # ============ Arithmetic Operators ============):**

```python
def __add__(self, other):
    return self._elementwise_binary_op(other, lambda a, b: a + b)

def __radd__(self, other):
    return self._elementwise_binary_op(other, lambda a, b: a + b)

def __iadd__(self, other):
    return self._inplace_binary_op(other, "__iadd__")

def __sub__(self, other):
    return self._elementwise_binary_op(other, lambda a, b: a - b)

def __rsub__(self, other):
    # NOT commutative: other - self
    if isinstance(other, (int, qint)):
        if isinstance(other, int):
            other = qint(other, width=self._width)
        result_elements = [other - elem for elem in self._elements]
        return self._create_view(result_elements, self._shape)
    return NotImplemented

def __isub__(self, other):
    return self._inplace_binary_op(other, "__isub__")

def __mul__(self, other):
    return self._elementwise_binary_op(other, lambda a, b: a * b)

def __rmul__(self, other):
    return self._elementwise_binary_op(other, lambda a, b: a * b)

def __imul__(self, other):
    return self._inplace_binary_op(other, "__imul__")
```

**3. Bitwise operators (section comment: # ============ Bitwise Operators ============):**

Same pattern for `__and__`/`__rand__`/`__iand__`, `__or__`/`__ror__`/`__ior__`, `__xor__`/`__rxor__`/`__ixor__`.
- `__and__` uses `lambda a, b: a & b`
- `__or__` uses `lambda a, b: a | b`
- `__xor__` uses `lambda a, b: a ^ b`
- Reverse operators delegate to forward (all commutative)
- In-place use `"__iand__"`, `"__ior__"`, `"__ixor__"`
