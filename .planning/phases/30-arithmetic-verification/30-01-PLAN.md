Create tests/test_add.py with the following structure:

1. Add `warnings.filterwarnings("ignore", message="Value .* exceeds")` at module level to suppress cosmetic UserWarnings for values >= 2^(width-1).

2. Generate test case data at MODULE LEVEL (not inside test functions):
   - `EXHAUSTIVE_QQ_ADD`: List of (width, a, b) for widths 1-4, using `generate_exhaustive_pairs(width)` from verify_helpers
   - `SAMPLED_QQ_ADD`: List of (width, a, b) for widths 5-8, using `generate_sampled_pairs(width, sample_size=10)` plus boundary pairs (0,0), (0,max), (max,0), (max,max)

3. Create 4 parametrized test functions:
   - `test_qq_add_exhaustive(verify_circuit, width, a, b)`: QQ addition at 1-4 bits
     - Build circuit: `qa = ql.qint(a, width=w); qb = ql.qint(b, width=w); _r = qa + qb`
     - Expected: `(a + b) % (1 << w)`
     - Use default argument binding: `def circuit_builder(a=a, b=b, w=width):`
   - `test_qq_add_sampled(verify_circuit, width, a, b)`: QQ addition at 5-8 bits (same circuit pattern)
   - `test_cq_add_exhaustive(verify_circuit, width, a, b)`: CQ addition at 1-4 bits
     - Build circuit: `qa = ql.qint(a, width=w); qa += b`
     - Expected: `(a + b) % (1 << w)`
   - `test_cq_add_sampled(verify_circuit, width, a, b)`: CQ addition at 5-8 bits

4. Use `format_failure_message()` from verify_helpers in all assertions.

Important patterns:
- Default argument binding in closures: `def circuit_builder(a=a, b=b, w=width):`
- verify_circuit returns (actual, expected) tuple
- All arithmetic uses unsigned wrap: `(a + b) % (1 << width)`
