  <name>Task 1: Build custom division/modulo test helper and create division tests</name>
  <files>tests/test_div.py</files>
  <action>
Create tests/test_div.py with CUSTOM result extraction (do NOT use verify_circuit fixture).

**Critical background:** The `verify_circuit` fixture extracts result from `bitstring[:width]`, which is the LAST allocated register (highest qubit indices). For division, the allocation order is: input(w qubits) -> quotient(w qubits) -> remainder(w qubits) -> ancillae. The quotient is NOT the last allocated register. Research empirically found quotient at bitstring[5:8] for 3-bit (11-qubit circuit).

**Approach -- Calibration-based extraction:**

1. Build a helper function `_run_and_extract(circuit_builder_fn, total_qubits, result_slice)`:
   - Calls `ql.circuit()`, runs circuit_builder_fn, calls `ql.to_openqasm()`
   - Loads QASM into Qiskit, adds measurements, simulates with AerSimulator(method='statevector'), shots=1
   - Returns `int(bitstring[result_slice], 2)`

2. Build a calibration function `_calibrate_div_position(width)`:
   - Run a KNOWN division case: e.g., `ql.qint(width * known_value, width=width) // known_divisor` where expected quotient is known
   - Scan the bitstring to find where the expected quotient bits appear
   - Return the slice that correctly extracts the quotient
   - Cache the result per width
   - Use known case: for width w, `(2**(w-1)) // 1` should give `2**(w-1)` -- a non-trivial value that's easy to locate

3. Generate test data at MODULE LEVEL:
   - `EXHAUSTIVE_DIV`: List of (width, a, divisor) for widths 1-3, where divisor in range(1, 2^width) (skip 0)
   - `SAMPLED_DIV`: List of (width, a, divisor) for width 4, sampled ~30 pairs with divisor in range(1, 2^width)

4. Create parametrized test functions:
   - `test_div_exhaustive(width, a, divisor)`: Floor division at 1-3 bits
     - Expected: `a // divisor`
     - Use calibrated extraction to get actual quotient
   - `test_div_sampled(width, a, divisor)`: Floor division at 4 bits

5. Add `warnings.filterwarnings("ignore", message="Value .* exceeds")` at module level.

**Important edge cases:**
- Only test classical divisors (int, not qint) per CONTEXT.md
- Skip divisor=0 (ZeroDivisionError)
- All quotients are unsigned: `a // divisor` (Python floor division is correct for non-negative integers)
  </action>
  <verify>Run `python -m pytest tests/test_div.py -v --tb=short 2>&1 | tail -20` and confirm all tests pass.</verify>
  <done>Floor division tests pass for exhaustive widths 1-3 and sampled width 4, with correct quotient extraction from non-standard bitstring positions.</done>
</task>
