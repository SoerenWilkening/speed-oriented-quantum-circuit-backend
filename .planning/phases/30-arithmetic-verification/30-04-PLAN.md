  <action>
Create tests/test_modular.py with custom result extraction for qint_mod operations.

**Critical background:** `qint_mod` operations allocate intermediate qubits for mod reduction (_reduce_mod uses comparison + conditional subtraction). The result register is NOT at `bitstring[:width]`. Custom extraction is needed, similar to division/modulo tests.

**Structure:**

1. Build a helper `_run_modular_test(op_name, a_val, b_val, modulus_n)`:
   - Calls `ql.circuit()`
   - Creates `a = ql.qint_mod(a_val, N=modulus_n)`
   - Performs operation based on op_name:
     - "add": `result = a + b_val`
     - "sub": `result = a - b_val`
     - "mul": `result = a * b_val`
   - Exports to QASM, simulates, extracts result with calibrated position
   - Returns actual integer value

2. Build calibration for qint_mod result position:
   - For each modulus N, determine the bit width: `width = (N-1).bit_length()` or similar
   - Calibrate with a known case per operation type and modulus
   - Cache calibrated positions

3. Test data -- representative inputs (NOT exhaustive, per CONTEXT.md):
   - Test with several moduli: N=3, N=5, N=7, N=8 (power of 2 edge case), N=13
   - For each modulus, test ~10-15 representative (a, b) pairs covering:
     - No reduction needed: a + b < N
     - Reduction needed: a + b >= N
     - Subtraction underflow: a - b < 0 (should wrap mod N)
     - Multiplication overflow: a * b >= N
     - Edge values: a=0, b=0, a=N-1, b=N-1

4. Create parametrized test functions:
   - `test_modular_add(a, b, n)`: `(a + b) % n` for representative inputs
   - `test_modular_sub(a, b, n)`: `(a - b) % n` for representative inputs
   - `test_modular_mul(a, b, n)`: `(a * b) % n` for representative inputs (b is int, not qint_mod)
   - `test_modular_result_reduced(a, b, n)`: Verify result < N for all operations

5. Add a separate non-parametrized test:
   - `test_qint_mod_mul_qint_mod_raises()`: Verify `qint_mod(5, N=7) * qint_mod(3, N=7)` raises `NotImplementedError`
   - Use `clean_circuit` fixture for this test (no simulation needed)

6. Add `warnings.filterwarnings("ignore", message="Value .* exceeds")`.

**Important:**
- qint_mod * qint_mod raises NotImplementedError -- this is by design, test should assert the error
- All results must be < N (properly reduced)
- Only test qint_mod * int (classical multiplier), NOT qint_mod * qint_mod
