  <name>Task 1: Replace classical init with XOR-copy in __add__, __radd__, __sub__</name>
  <files>src/quantum_language/qint.pyx</files>
  <action>
In `__add__` (line ~795), `__radd__` (line ~840), and `__sub__` (line ~910), replace:

```python
a = qint(value = self.value, width = result_width)
```

with the XOR-into-zero quantum copy pattern (same pattern already proven correct in `__floordiv__` and `__mod__`):

```python
a = qint(width=result_width)
a ^= self
```

This allocates a zero-initialized result with the correct width, then XOR-copies self's quantum state into it via CNOT gates. The upper bits (if result_width > self.bits) stay |0> as zero-extension, which is correct for unsigned integers.

**Important details:**
- The `^= self` call applies CNOT from each of self's qubits to the corresponding qubit in `a`. Since `a` starts at |0>, this copies self's state (including superposition) into `a`.
- The layer tracking code AFTER the copy+operation (`a._start_layer = start_layer`, `a._end_layer = ...`, `a.operation_type`, `a.add_dependency`) must remain unchanged. The start_layer captured before the XOR-copy ensures the full operation (copy + arithmetic) is tracked as one unit for uncomputation.
- Do NOT change `__iadd__` or `__isub__` -- those are in-place and already correct.
- Do NOT change `__mul__`, `__rmul__`, bitwise ops, `__floordiv__`, `__mod__` -- those are already correct per the research audit.

After edits, rebuild Cython: `python3 setup.py build_ext --inplace`
