  **Use 2-bit QInts (width=2) to stay safely under 17-qubit simulation limit.**
  Qubit budget: outer(2) + outer_cond(1) + inner(2) + inner_cond(1) + result(3) + ancilla(~4) = ~13 qubits.

  **Test class: TestNestedWithBlocks**

  Uses the `verify_circuit` fixture from conftest.py (in parent tests/ directory).
  Add `sys.path` adjustment if needed to import from tests/conftest.py, OR add a conftest.py
  in tests/python/ that imports the fixture, OR use direct simulation pattern from test_conditionals.py.

  **Since tests/python/ has its own conftest.py, use the direct simulation pattern from test_bug07_cond_mul.py instead of verify_circuit.**

  **Test cases (CQ comparisons as conditions):**

  1. `test_nested_both_true`: outer=True, inner=True
     - outer: a(3) > 1 -> True; inner: b(3) > 1 -> True
     - Inside outer: result += 1; Inside inner: result += 2
     - Expected result: 3 (1 + 2)

  2. `test_nested_outer_true_inner_false`: outer=True, inner=False
     - outer: a(3) > 1 -> True; inner: b(0) > 1 -> False
     - Inside outer: result += 1; Inside inner: result += 2
     - Expected result: 1 (only outer op)

  3. `test_nested_outer_false_inner_true`: outer=False, inner=True
     - outer: a(0) > 1 -> False; inner: b(3) > 1 -> True
     - Inside outer: result += 1; Inside inner: result += 2
     - Expected result: 0 (outer blocks everything including inner)

  4. `test_nested_both_false`: outer=False, inner=False
     - outer: a(0) > 1 -> False; inner: b(0) > 1 -> False
     - Inside outer: result += 1; Inside inner: result += 2
     - Expected result: 0

  5. `test_nested_subtraction`: nested conditional subtraction
     - outer=True, inner=True
     - Inside outer: result -= 1; Inside inner: result -= 1
     - result starts at 3, expected: 1 (3 - 1 - 1)

  6. `test_nested_assignment_in_inner`: arithmetic in inner block only
     - outer=True, inner=True
     - Only inside inner: result += 3
     - Expected result: 3

  **Implementation pattern (direct simulation):**
  ```python

      gc.collect()
      ql.circuit()

      b = ql.qint(3, width=2)
      inner_cond = b > 1
      result = ql.qint(0, width=3)
      with outer_cond:
          result += 1
          with inner_cond:
              result += 2
      # Extract result via allocated_start and simulate
  ```

  Add `warnings.filterwarnings("ignore", message="Value .* exceeds")`.

  <name>Task 2: Create circuit reset behavior tests</name>
  <files>tests/python/test_circuit_reset.py</files>
  <action>
  Create `tests/python/test_circuit_reset.py` testing that `ql.circuit()` properly resets all circuit state.

  **Test class: TestCircuitReset**

  Test cases:

  1. `test_reset_no_gate_leakage`:
     - Build circuit 1: ql.circuit(); a = ql.qint(3, width=3); b = a + 1
     - Save QASM from circuit 1 for reference
     - Build circuit 2: gc.collect(); ql.circuit(); c = ql.qint(5, width=3)
     - Get QASM from circuit 2
     - Assert circuit 2 QASM does NOT contain addition gates from circuit 1
     - Assert circuit 2 QASM contains only initialization of value 5

  2. `test_reset_qubit_allocation_fresh`:
     - ql.circuit(); a = ql.qint(0, width=8) -- allocates qubits 0-7
     - gc.collect(); ql.circuit() -- reset
     - b = ql.qint(0, width=3)
     - Assert b.allocated_start == 0 (not 8)

  3. `test_reset_simulation_independence`:
     - Build complex circuit: ql.circuit(); x = ql.qint(7, width=3); y = x + 1
     - Reset and build simple one: gc.collect(); ql.circuit()
     - a = ql.qint(3, width=3); b = a + 1
     - Simulate and verify result is 4 (not contaminated by prior circuit)
     - Use direct simulation pattern with _simulate_and_extract

  4. `test_multiple_resets`:
     - Do 3 cycles of: gc.collect(); ql.circuit(); ql.qint(N, width=3)
     - After each reset, verify allocated_start == 0
     - After each reset, verify QASM only contains current circuit

  5. `test_reset_after_conditional`:
     - Build circuit with conditional: ql.circuit(); a = ql.qint(3, width=2); cond = a > 1; with cond: ...
     - gc.collect(); ql.circuit() -- reset
     - Build simple circuit and verify no conditional gates leak through

  **Important:** Always call gc.collect() before ql.circuit() to prevent GC-induced gate injection
  from previous test's qint destructors.
  </action>
