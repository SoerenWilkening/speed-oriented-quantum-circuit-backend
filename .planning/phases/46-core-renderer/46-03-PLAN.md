**Scale tests (using synthetic draw_data -- no Cython build needed):**

1. `test_scale_200_qubits_10k_gates` -- Create synthetic draw_data with `num_layers=10000`, `num_qubits=200`. Generate 10,000 gate dicts with random layers (0-9999), random targets (0-199), random types (0-9), empty controls. Call `render(data)`. Assert:

   - Image size is `(10000 * CELL_W, 200 * CELL_H)` = `(30000, 600)`
   - Image mode is 'RGB'
   Use `import random; random.seed(42)` for reproducibility.

2. `test_scale_200_qubits_with_controls` -- Same as above but include ~1000 CNOT gates (controls=[random control qubit]) among the 10,000 gates. Verify rendering completes and image dimensions are correct. This tests control line rendering at scale.

3. `test_scale_memory_reasonable` -- Create draw_data for 200 qubits x 10,000 layers. After rendering, check `np.array(img).nbytes` is approximately `30000 * 600 * 3 = 54,000,000` bytes (~51.5 MB). Allow 10% tolerance. This confirms no memory bloat from temporary allocations.

4. `test_scale_rendering_time` -- Time the rendering of 200 qubits x 10,000 gates. Assert it completes in under 30 seconds. Use `import time; start = time.time()`. This is a soft performance gate -- if NumPy bulk ops are used correctly, it should take well under 5 seconds.

5. `test_scale_png_save` -- Render 200 qubits x 10,000 gates, save to a temporary file (`/tmp/test_scale_circuit.png`), verify the file exists and has size > 0. Use `os.path.getsize()`. Then clean up the file.

**Integration test (requires Cython build -- mark with pytest.mark):**

6. `test_integration_real_circuit` -- Mark with `@pytest.mark.integration` (so it can be skipped if Cython extensions aren't built). Build a small real circuit:

   ```python
   img = render(data)
   ```
   Assert image is a valid PIL Image with non-zero dimensions. Save to `/tmp/test_integration_circuit.png` for visual inspection. Clean up after.

**Note:** For the integration test, use `pytest.importorskip("quantum_language._core")` or a try/except to gracefully skip if the Cython extension isn't available. The scale tests use synthetic data and should always run.
