<objective>
Implement the CCX -> Clifford+T decomposition that activates when `ql.option('toffoli_decompose', True)` is set. Every CCX gate emitted by sequence generators and inline paths is replaced with the standard 15-gate Clifford+T sequence (2H + 4T + 3Tdg + 6CX).

Purpose: Enable users to get actual Clifford+T basis gate circuits for fault-tolerant compilation, with exact T-count reporting instead of estimates.
Output: CCX decomposition helper, integration into all CCX-emitting codepaths, Clifford+T test suite.

NOTE: This plan depends on Plan 74-03 because it modifies the same files (ToffoliAdditionCDKM.c, ToffoliAdditionCLA.c, ToffoliMultiplication.c). Plan 74-03 rewrites the MCX emission points to use AND-ancilla CCX. This plan then wraps those same CCX calls with Clifford+T decomposition when `toffoli_decompose=1`. The AND-ancilla-rewritten versions from 74-03 are the starting point for this plan's modifications.

**Create the decomposition helper in gate.c:**

Add a function `emit_ccx_clifford_t(circuit_t *circ, qubit_t target, qubit_t ctrl1, qubit_t ctrl2)` that emits the standard exact CCX decomposition as 15 individual gates via `add_gate()`. The decomposition sequence (from RESEARCH.md):

```
H(target)
CX(target, ctrl2)
Tdg(target)
CX(target, ctrl1)
T(target)
CX(target, ctrl2)
Tdg(target)
CX(target, ctrl1)
T(target)
T(ctrl2)
H(target)
CX(ctrl2, ctrl1)
T(ctrl1)
Tdg(ctrl2)
CX(ctrl2, ctrl1)
```

Each gate uses `memset(&g, 0, sizeof(gate_t))` then the appropriate gate function (h, cx, t_gate, tdg_gate) then `add_gate(circ, &g)`. This is the circuit-based helper for inline emission paths.

**Also create a sequence-based variant** `emit_ccx_clifford_t_seq(sequence_t *seq, int *layer, qubit_t target, qubit_t ctrl1, qubit_t ctrl2)` for use in sequence-building functions. This emits the same 15 gates into sequential layers of the sequence. Each gate increments `*layer`. Verify the sequence has enough allocated layers.

**Declare both in gate.h:**
```c
void emit_ccx_clifford_t(circuit_t *circ, qubit_t target, qubit_t ctrl1, qubit_t ctrl2);
void emit_ccx_clifford_t_seq(sequence_t *seq, int *layer, qubit_t target, qubit_t ctrl1, qubit_t ctrl2);
```

Note: gate.h needs a forward declaration of circuit_t. Check if it already includes circuit types. If not, add `struct circuit_s; typedef struct circuit_s circuit_t;` forward declaration, or include the appropriate header. Be careful not to create circular includes.

**Integrate into the AND-ancilla-rewritten inline emitters from Plan 74-03:**

IMPORTANT: Plan 74-03 has already rewritten the MCX emission points to use AND-ancilla CCX. The files now contain CCX calls where MCX(3) used to be. This plan adds a `decompose` parameter to those same emitters so that when `toffoli_decompose=1`, each CCX call is replaced with `emit_ccx_clifford_t`.

**Mode A -- Sequence generators (CDKM cached sequences):**
In ToffoliAdditionCDKM.c, the sequence builder functions (toffoli_QQ_add, toffoli_CQ_add, etc.) build sequence_t objects with CCX gates. When `toffoli_decompose` is active, these should call `emit_ccx_clifford_t_seq` instead of `ccx()`.

HOWEVER: Cached sequences (toffoli_QQ_add returns cached) are built once and reused. The toffoli_decompose flag is a runtime option that can change between calls. This means cached sequences must be built for the current mode, OR separate caches exist for decomposed vs non-decomposed.

**Decision (Claude's discretion):** The simplest approach is to NOT decompose inside cached sequence builders. Instead, decompose at the point of consumption -- when a sequence is replayed into a circuit via `run_instruction`, check the circuit's toffoli_decompose flag and decompose each CCX as it's replayed.

Actually, the cleaner approach matching the user's decision ("inside sequence generators, check toffoli_decompose flag") is: make the sequence generators check the flag and build separate cached versions. Use separate cache variables: `static sequence_t *cached_QQ_add[MAX_BITS]` (normal) and `static sequence_t *cached_QQ_add_decomposed[MAX_BITS]` (Clifford+T). The generator checks the circuit's toffoli_decompose flag (needs circuit access or a global flag).

**Simplest correct approach:** Since sequence generators don't have circuit access (they return standalone sequences), and the user wants separate hardcoded decomposed files (Plan 05), do this:
1. For the INLINE paths (toffoli_mul_qq, emit_cMAJ_decomposed, etc. that use add_gate(circ, &g)), check `circ->toffoli_decompose` and call `emit_ccx_clifford_t` instead of adding a CCX gate directly.
2. For CACHED sequences, defer to Plan 05 (hardcoded decomposed sequences provide separate fast paths).
3. For the DYNAMIC generation path (width > 8, no hardcoded), add a toffoli_decompose parameter to the sequence builder functions, or check a global.

For this plan: Focus on the inline circuit-based paths first. Modify the inline emission helpers in ToffoliAdditionCDKM.c (emit_MAJ, emit_UMA, emit_cMAJ, emit_cUMA -- now AND-ancilla-rewritten by Plan 74-03), ToffoliAdditionCLA.c (inline BK phases), and ToffoliMultiplication.c (emit_cMAJ_decomposed, emit_cUMA_decomposed, toffoli_mul_qq CCX emissions) to check a `toffoli_decompose` parameter and call the Clifford+T helper instead of CCX when active.

For inline helpers: Add an `int decompose` parameter (1=Clifford+T, 0=CCX) to each inline helper. The caller passes `circ->toffoli_decompose`. When decompose=1, instead of `ccx(&g, target, c1, c2); add_gate(circ, &g);`, call `emit_ccx_clifford_t(circ, target, c1, c2)`.

**Sequence-level CCX decomposition** for the layer count: When building sequences with decompose=1, each CCX becomes 15 layers instead of 1. Update `compute_CQ_layer_count` and similar functions to account for this 15x expansion of CCX layers.

**Practical approach for this plan:** Since this touches many functions, focus on:
1. The `emit_ccx_clifford_t` circuit helper (always correct, no caching issues)
2. Make all inline emitters in ToffoliMultiplication.c use it when `circ->toffoli_decompose == 1`
3. For CDKM cached sequences that DON'T have circuit access: skip decomposition (CCX stays). The decomposed hardcoded sequences (Plan 05) will handle widths 1-8, and width 9+ dynamic uses inline generation that has circuit access.
4. For CLA inline paths that have circuit access: use emit_ccx_clifford_t.

This ensures all inline paths decompose CCX when the option is on. Sequence-cached paths produce CCX (as before) and will be superseded by decomposed hardcoded sequences in Plan 05.

5. **Independence test:**
   - toffoli_decompose works with fault_tolerant=False (decomposes CCX from QFT path if any)

Example test:
```python
def test_t_count_exact_with_decompose():
    """T-count is exact (sum of T+Tdg) when decompose is on."""
    c = ql.circuit()
    ql.option('fault_tolerant', True)
    ql.option('toffoli_decompose', True)
    a = ql.qint(2, 3)
    b = ql.qint(3, 3)
    a += b
    gc = c.gate_counts
    t_actual = gc.get('T_gates', 0) + gc.get('Tdg_gates', 0)
```
