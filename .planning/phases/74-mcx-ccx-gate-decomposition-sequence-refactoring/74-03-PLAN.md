**CDKM MCX points (from RESEARCH.md audit):**

1. **emit_cMAJ() -- MCX(3) at step 3**: Replace `mcx(g, c, [a, b, ext_ctrl], 3)` with:
   - CCX(and_anc, a, b) -- compute AND
   - CCX(c, and_anc, ext_ctrl) -- apply via ancilla
   - CCX(and_anc, a, b) -- uncompute AND
   The AND-ancilla qubit must be allocated. For cQQ_add, the layout is [0..bits-1]=a, [bits..2*bits-1]=b, [2*bits]=carry, [2*bits+1]=ext_ctrl. Place and_anc at [2*bits+2].

2. **emit_cUMA() -- MCX(3) at step 1**: Same pattern as cMAJ. Replace `mcx(g, c, [a, b, ext_ctrl], 3)` with the 3-CCX AND-ancilla decomposition using the same and_anc qubit.

3. **emit_cCQ_MAJ() -- MCX(3) at bit=0 non-first**: Same pattern. For cCQ_add, layout is [0..bits-1]=temp, [bits..2*bits-1]=self, [2*bits]=carry, [2*bits+1]=ext_ctrl. Place and_anc at [2*bits+2].

**CLA MCX points:**

4. **toffoli_cQQ_add_bk() -- CCX->MCX(3)**: When injecting ext_ctrl into a CCX gate, this creates MCX(3). Decompose using AND-ancilla. The and_anc index depends on the CLA qubit layout -- ext_ctrl is at 2*bits + bk_cla_ancilla_count(bits), so and_anc goes at ext_ctrl + 1.

5. **toffoli_cQQ_add_bk() -- MCX(n)->MCX(n+1)**: When injecting ext_ctrl into an existing MCX(n), this creates MCX(n+1). For these, use recursive AND-ancilla decomposition. Each level reduces control count by 1, using 1 additional ancilla. For MCX(4), need 2 ancilla; for MCX(k), need k-2 ancilla. Allocate these from the next available qubit indices after ext_ctrl.

**Implementation approach for all 5 points:**
- Create a static helper function `emit_mcx3_decomposed(seq, layer, target, ctrl1, ctrl2, ext_ctrl, and_anc)` for the sequence-based emission pattern (since CDKM adders build sequences, not circuits)
- For circuit-based emission (CLA inline paths), create `emit_mcx3_decomposed_circ(circ, target, ctrl1, ctrl2, ext_ctrl, and_anc)`
- For recursive MCX(k>3) in CLA, create `emit_mcx_recursive_decomposed(circ, target, controls, num_controls, anc_start)` that recursively reduces to CCX

**CRITICAL qubit layout updates:**
- `toffoli_cQQ_add()`: Total qubits changes from 2*bits+2 to 2*bits+3 (add 1 AND-ancilla). Update the function docs and any callers in hot_path_add_toffoli.c that pass qubit counts.

- Update `toffoli_arithmetic_ops.h` docstrings to reflect new qubit layouts.

**WARNING:** The qubit_array mapping in hot_path_add.c / hot_path_add_toffoli.c must also be updated to account for the extra AND-ancilla. Each controlled variant now needs 1 more ancilla qubit. Update `allocator_alloc()` calls in the dispatch functions.
