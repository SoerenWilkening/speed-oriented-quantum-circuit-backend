  <action>
Create a new generation script `scripts/generate_toffoli_decomp_seq.py` that produces hardcoded C files for MCX-decomposed controlled QQ addition sequences.

**Script approach:**
Follow the proven pattern from `scripts/generate_toffoli_seq.py` (856 lines). The new script generates cQQ sequences where MCX(3) gates are replaced with the 3-CCX AND-ancilla decomposition.

The script should:
1. Simulate the cQQ CDKM adder for each width (same algorithm as toffoli_cQQ_add)
2. When an MCX(3) would be emitted, instead emit 3 CCX gates (AND-ancilla decomposition)
3. Include the AND-ancilla qubit in the layout: for width N, qubits are [0..N-1]=a, [N..2N-1]=b, [2N]=carry, [2N+1]=ext_ctrl, [2N+2]=and_anc
4. Output static const arrays for the gate sequences (same format as existing toffoli_add_seq_N.c files)
5. Since the decomposed cQQ sequences contain only CCX (max 2 controls), they can use static const initialization (no dynamic large_control allocation needed -- this is a key advantage)

**Per-width files (`toffoli_decomp_seq_N.c`):**
Each file contains:
- `#include "sequences.h"` and `#include "toffoli_sequences.h"`
- `#ifdef SEQ_WIDTH_N` / `#endif` guards (matching existing pattern)
- Static const gate arrays for cQQ addition at that width
- Init function: `sequence_t *toffoli_decomp_cQQ_add_seq_N(void)` that returns a sequence built from the static data

For width 1: cQQ_add is a single CCX (no MCX, no decomposition needed). Still generate the file for consistency.
For widths 2-8: cMAJ and cUMA contain MCX(3) -> replaced with 3 CCX each. Layer count increases accordingly.

**Dispatch file (`toffoli_decomp_seq_dispatch.c`):**
```c
sequence_t *get_hardcoded_toffoli_decomp_cQQ_add(int bits) {
    switch (bits) {
        case 1: return toffoli_decomp_cQQ_add_seq_1();
        // ...
        case 8: return toffoli_decomp_cQQ_add_seq_8();
        default: return NULL; // Fall back to dynamic generation
    }
}
```

**Update `toffoli_sequences.h`:**
- Add function declarations for each per-width init function
- Add `get_hardcoded_toffoli_decomp_cQQ_add(int bits)` declaration
- Add SEQ_WIDTH_N defines if not already present

**Run the generation script** to produce all 8 C files plus the dispatch file.

NOTE: Keep the script separate from generate_toffoli_seq.py per user decision (separate scripts per backend). The script should be self-contained and runnable via `python3 scripts/generate_toffoli_decomp_seq.py`.
