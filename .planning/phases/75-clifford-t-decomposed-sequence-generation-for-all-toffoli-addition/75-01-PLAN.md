Extend the two existing CDKM generation scripts with a `--clifford-t` flag per user decision ("Extend existing generation scripts with Clifford+T mode"). This prevents algorithm divergence by reusing the existing CDKM gate generation logic and only adding a CCX->Clifford+T expansion step.

**Shared CliffordTGate infrastructure** (add to both scripts or to a shared module at Claude's discretion):

```python
@dataclass
class CliffordTGate:
    gate_type: str  # "H", "T", "Tdg", "X", "CX"
    target: int
    control: int = -1  # Only for CX
```

With `to_c_static()` method using these exact GateValue mappings:
- H: `.Gate = H, .GateValue = 0, .NumControls = 0`
- T: `.Gate = T_GATE, .GateValue = M_PI / 4.0, .NumControls = 0`
- Tdg: `.Gate = TDG_GATE, .GateValue = -M_PI / 4.0, .NumControls = 0`
- X: `.Gate = X, .GateValue = 1, .NumControls = 0`
- CX: `.Gate = X, .GateValue = 1, .NumControls = 1, .Control = {ctrl}`

**ccx_to_clifford_t(target, ctrl1, ctrl2)** function: Returns list of 15 CliffordTGate objects matching the exact decomposition from `gate.c:emit_ccx_clifford_t` (verified in 75-RESEARCH.md Pattern 1). The exact 15-gate sequence is:
1. H(target), 2. CX(target, ctrl2), 3. Tdg(target), 4. CX(target, ctrl1), 5. T(target), 6. CX(target, ctrl2), 7. Tdg(target), 8. CX(target, ctrl1), 9. T(target), 10. T(ctrl2), 11. H(target), 12. CX(ctrl2, ctrl1), 13. T(ctrl1), 14. Tdg(ctrl2), 15. CX(ctrl2, ctrl1).

**Extending `generate_toffoli_seq.py`** -- add `--clifford-t` flag:
- When `--clifford-t` is passed, the script reuses the existing `generate_qq_gates(bits)` function to get the list of ToffoliGate objects (CX/CCX), then adds a post-processing step that expands every CCX into 15 CliffordTGate objects via `ccx_to_clifford_t`. CX and X gates pass through as CliffordTGate.
- Add new C file generation functions: `generate_clifft_qq_static_c()` and `generate_clifft_cqq_static_c()` that emit CliffordTGate-based static const arrays (note: in Clifford+T mode, even cQQ becomes static const because MCX->CCX->Clifford+T produces max 1 control).
- Output files: `toffoli_clifft_qq_N.c` (8 files) and `toffoli_clifft_cqq_N.c` (8 files).
- Width 1 QQ: Single CX(target=0, control=1) -- no CCX, no expansion. 1 layer.
- Width N >= 2 QQ: Build CDKM adder via existing `generate_qq_gates(bits)`, expand CCX -> 15 CliffordTGate. Formula: 4N CX + 4N CCX in source = 4N + 60N = 64N layers for N >= 2.
- Qubit layout (QQ): [0..N-1] = a, [N..2N-1] = b, [2N] = carry ancilla. ALL sequences static const (max 1 control).

**Extending `generate_toffoli_decomp_seq.py`** -- add `--clifford-t` flag:
- When `--clifford-t` is passed, the script reuses the existing `generate_decomp_cqq_gates(bits)` function (which already decomposes MCX(3)->3 CCX via AND-ancilla) to get the list of ToffoliGate objects (all CX/CCX, no MCX), then adds a post-processing step expanding every CCX into 15 CliffordTGate objects.
- For cQQ: reuse the existing decomposed cQQ gate generation. The MCX->AND-ancilla decomposition already happened. Just expand CCX->Clifford+T on top.
- Output files: `toffoli_clifft_cqq_N.c` (8 files). NOTE: The cQQ Clifford+T files are generated by `generate_toffoli_decomp_seq.py` (not `generate_toffoli_seq.py`) because the decomp script already handles the MCX->AND-ancilla pattern that the cQQ Clifford+T needs.
- Width 1 cQQ: Single CCX(target=0, ctrl1=1, ctrl2=2) expands to 15 Clifford+T gates.
- Width N >= 2 cQQ: 10N CCX from decomposed CDKM -> 150N Clifford+T layers.
- Qubit layout (cQQ): [0..N-1] = a, [N..2N-1] = b, [2N] = carry, [2N+1] = ext_ctrl, [2N+2] = AND-ancilla. ALL sequences static const.

**Per-width C file format**: Follow existing patterns in `toffoli_add_seq_N.c` / `toffoli_decomp_seq_N.c`:
- `#include "toffoli_sequences.h"` then `#ifdef TOFFOLI_SEQ_WIDTH_N`
- `#include <math.h>` (needed for M_PI in T/Tdg GateValue)
- Static const gate arrays: `static const gate_t TOFFOLI_CLIFFT_QQ_N_LX[] = {...}`
- Static const layer pointers array + GPL array + sequence_t
- Per-width dispatch functions: `get_hardcoded_toffoli_clifft_QQ_add_N(bits)`, `get_hardcoded_toffoli_clifft_cQQ_add_N(bits)`

**CLI**: Both scripts gain `--clifford-t` flag. Default behavior (no flag) is unchanged. `--validate --clifford-t` checks Clifford+T layer count formulas.

IMPORTANT: The existing non-Clifford+T generation behavior of both scripts MUST remain unchanged. The `--clifford-t` flag is purely additive.

**CDKM CQ increment (value=1) Clifford+T generation** (follows `toffoli_cq_inc_seq_*.c` pattern):
- Width 1: Single X(target=0) -- no CCX, no expansion. 1 layer.
- Width N >= 2: Build CQ increment with classical-bit simplification for value=1 (only LSB=1), then expand CCX gates into Clifford+T.
- The CQ increment for value=1 eliminates gates at bit=0 (carry=|0>), folds X-init at bit=1. Use the same logic as Phase 73 CQ inline generator.
- Qubit layout: [0..N-1] = temp, [N..2N-1] = self, [2N] = carry.
- ALL sequences are static const.

**CDKM cCQ increment (value=1) Clifford+T generation**:
- Width 1: Single CX(target=0, control=1). 1 layer.
- Width N >= 2: Build cCQ increment with classical-bit simplification (CX-init, standard cMAJ at bit=1, skip at bit=0), then expand CCX gates into Clifford+T.
- Qubit layout: [0..N-1] = temp, [N..2N-1] = self, [2N] = carry, [2N+1] = ext_ctrl, [2N+2] = AND-ancilla.
- ALL sequences are static const.

Output files: `toffoli_clifft_cq_inc_N.c` (8 files) and `toffoli_clifft_ccq_inc_N.c` (8 files).
