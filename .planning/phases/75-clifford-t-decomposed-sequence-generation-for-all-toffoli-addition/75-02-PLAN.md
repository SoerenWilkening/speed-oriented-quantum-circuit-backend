<objective>
Create the BK CLA Clifford+T hardcoded sequence generation script and generate all 28 per-width C source files plus a unified dispatch file for BK CLA addition variants (QQ, cQQ, CQ-inc, cCQ-inc) at widths 2-8.

Purpose: Extend Clifford+T hardcoded coverage to the BK Carry Look-Ahead adder path. BK CLA has O(log n) depth, so pre-computed Clifford+T sequences eliminate both runtime CLA generation and CCX decomposition overhead. Width 1 is excluded because BK CLA returns NULL for bits < 2 (falls back to CDKM/RCA).

Output: 1 generation script + 28 per-width C files + 1 dispatch C file = 30 files total.
</objective>

  <files>scripts/generate_toffoli_clifft_cla.py</files>
  <action>
Create `scripts/generate_toffoli_clifft_cla.py` -- a new generation script producing Clifford+T hardcoded sequences for all 4 BK CLA addition variants (QQ, cQQ, CQ-inc, cCQ-inc) at widths 2-8.

**CliffordTGate dataclass**: Duplicate from Plan 01's script (or import from a shared module at Claude's discretion). Same `to_c_static()` method with exact GateValue mappings (H=0, T=M_PI/4.0, Tdg=-M_PI/4.0, X=1, CX=1). Same `ccx_to_clifford_t(target, ctrl1, ctrl2)` expansion function returning 15 gates.

**Python BK prefix tree implementation**: Port `bk_compute_merges()` from `ToffoliAdditionCLA.c` (lines 58-136) into Python. This is the most critical piece -- the merge list determines gate placement for Phases B and D. Must exactly match C implementation:
- Input: n_carries (= bits - 1)
- Output: list of (pos, partner) merge tuples
- Algorithm: up-sweep (doubling stride), down-sweep (filling gaps), tail merges for non-power-of-2
- Width 1 (n_carries=0): returns empty merge list (no CLA for 1-bit)

**BK CLA QQ Clifford+T generation** (follows `toffoli_QQ_add_bk()` structure from ToffoliAdditionCLA.c):
- Implement all 6 phases (A, B, C, D, E, F) in Python
- Phase A: (n-1) CCX + n CX
- Phase B: 2 gates per merge (CCX for generate update, CCX for propagate product) -- record for Phase D
- Phase C: (n-1) CX (carry copy)
- Phase D: Reverse of Phase B (replay gates backwards)
- Phase E: n CX + (n-1) CCX (uncompute)
- Phase F: n CX + (n-1) CX (sum extraction)
- Total source layers: 7*n - 4 + 4*num_merges
- Qubit layout: a[i]=i, b[i]=n+i, g[i]=2n+i, tree_anc starts at 3n-1, carry_copy starts at 3n-1+num_merges
- Track prop_src[i] (initially b[i]=n+i, updated during Phase B merges)
- After building all gates as ToffoliGate (CX/CCX), expand every CCX into 15 CliffordTGate via ccx_to_clifford_t. CX and X pass through.
- Count CCX and CX separately to compute exact Clifford+T layer count: num_CX * 1 + num_CCX * 15.
- ALL sequences are static const (max 1 control).

**BK CLA cQQ Clifford+T generation** (follows `toffoli_cQQ_add_bk()` pattern):
- Derive from QQ sequence by injecting ext_ctrl into every gate: X->CX, CX->CCX, CCX->MCX(3)->3 CCX (AND-ancilla decomposition).
- Then expand every resulting CCX into 15 Clifford+T gates.
- Qubit layout adds ext_ctrl and and_anc after CLA ancilla.
- ALL sequences are static const.

**BK CLA CQ increment (value=1) Clifford+T generation** (follows `toffoli_CQ_add_bk()` pattern):
- CQ uses temp-register approach: X-init temp to classical value, run QQ BK CLA gates (with classical-bit simplification for value=1), X-cleanup.
- For value=1 (only LSB=1): Phase A eliminates CCX at bit=0 (temp[0]=|0> means generate=0), Phase E eliminates CCX at bit=0 similarly, Phase F temp init/cleanup simplified.
- Build the gate list, then expand CCX -> Clifford+T.
- Qubit layout: temp[i]=i, self[i]=n+i, carry=2n, CLA ancilla starts at 2n+something.
- ALL sequences are static const.

**BK CLA cCQ increment (value=1) Clifford+T generation** (follows `toffoli_cCQ_add_bk()` pattern):
- Controlled CQ: CX-init (controlled by ext_ctrl) + copy cQQ gates + CX-cleanup.
- Build from cQQ variant, add controlled init/cleanup.
- Qubit layout adds ext_ctrl and and_anc.
- ALL sequences are static const.

**Per-width C file format**: Same as Plan 01 -- `#include "toffoli_sequences.h"`, `#ifdef TOFFOLI_SEQ_WIDTH_N`, `#include <math.h>`, static const arrays. Per-width dispatch functions named `get_hardcoded_toffoli_clifft_cla_QQ_add_N(bits)`, etc.

**Dispatch file** (`toffoli_clifft_cla_dispatch.c`): Same pattern as CDKM dispatch. Four dispatch functions:
- `get_hardcoded_toffoli_clifft_cla_QQ_add(int bits)` -- widths 2-8 (returns NULL for 1)
- `get_hardcoded_toffoli_clifft_cla_cQQ_add(int bits)` -- widths 2-8
- `get_hardcoded_toffoli_clifft_cla_CQ_inc(int bits)` -- widths 2-8
- `get_hardcoded_toffoli_clifft_cla_cCQ_inc(int bits)` -- widths 2-8

**CLI interface**: `--width N`, `--dry-run`, `--validate` flags.

**Validation mode**: For each width, verify that layer counts match the formula (count CX and CCX in source, compute CX*1 + CCX*15). Also verify that BK merge computation matches expected merge counts for each width (cross-check against known BK merge counts from Phase 71 implementation).
