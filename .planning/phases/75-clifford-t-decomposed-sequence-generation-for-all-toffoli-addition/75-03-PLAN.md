Then in each dispatch function, check the cache array first:
```c
// Example for QQ path:
if (circ->toffoli_decompose && result_bits <= 8) {
    const sequence_t *seq = precompiled_toffoli_clifft_QQ_add[result_bits];
    if (!seq) {
        seq = get_hardcoded_toffoli_clifft_QQ_add(result_bits);
        precompiled_toffoli_clifft_QQ_add[result_bits] = seq;
    }
    if (seq) {
        // use seq -- same tqa[] qubit layout as non-decomposed
    }
}
```

This matches the locked caching pattern: first call goes through the switch/case dispatch (analogous to "hardcoded -> cache"), subsequent calls hit the pointer array (O(1) lookup).

Modify the dispatch functions to check `circ->toffoli_decompose` and use Clifford+T sequences when the flag is set:

In `toffoli_qq_uncont()`:
- Before the current CLA dispatch block, add: if `circ->toffoli_decompose` is set:
  - For CLA path: lookup from `precompiled_toffoli_clifft_cla_QQ_add`, falling back to `get_hardcoded_toffoli_clifft_cla_QQ_add(result_bits)`
  - For RCA path: lookup from `precompiled_toffoli_clifft_QQ_add`, falling back to `get_hardcoded_toffoli_clifft_QQ_add(result_bits)`
  - If the Clifford+T dispatch returns NULL, fall through to the non-decomposed path.

In `toffoli_qq_cont()`:
- Same pattern: check `circ->toffoli_decompose`, use cached Clifford+T dispatch variants.
  - CLA: `precompiled_toffoli_clifft_cla_cQQ_add` cache
  - RCA: `precompiled_toffoli_clifft_cQQ_add` cache

In `toffoli_cq_uncont()`:
- For CLA path with value==1 and bits <= TOFFOLI_HARDCODED_MAX_WIDTH: lookup from CQ inc cache, use `copy_hardcoded_sequence()` (CQ sequences are caller-owned).
- For RCA path with value==1 and bits <= TOFFOLI_HARDCODED_MAX_WIDTH: same with RCA CQ inc cache.
- For general values (value != 1) or width > 8: fall through to dynamic path (which already handles Clifford+T via inline emission from Phase 74-04).

In `toffoli_cq_cont()`:
- Same pattern for controlled CQ with value==1: use cCQ inc caches.

IMPORTANT: The Clifford+T sequences have the SAME qubit layout as non-decomposed sequences (no extra qubits needed -- CCX expansion only adds gates, not qubits). So the existing qubit array setup code stays exactly the same.

Create `tests/python/test_clifford_t_hardcoded.py` with comprehensive tests for Clifford+T hardcoded sequences. Follow existing test patterns from `test_clifford_t_decomposition.py` and `test_hardcoded_sequences.py`.

**Test classes and methods:**

**class TestCliffordTGatePurity:**
Test that Clifford+T hardcoded sequences produce zero CCX/MCX gates.

- `test_cdkm_qq_purity_width_N` (parametrize N=1..4): With `toffoli_decompose=True`, create qint(a, N) + qint(b, N), check gate_counts has zero CCX and zero MCX. Verify T_gates > 0 and Tdg_gates > 0 (for N >= 2; N=1 has only CX).
- `test_cdkm_cqq_purity_width_N` (parametrize N=1..4): Controlled QQ addition inside `with` block. Verify zero CCX, zero MCX, T_gates > 0.
- `test_cdkm_cq_inc_purity_width_N` (parametrize N=1..4): CQ increment (a += 1) with toffoli_decompose=True. Verify zero CCX.
- `test_cdkm_ccq_inc_purity_width_N` (parametrize N=1..4): Controlled CQ increment inside `with` block. Verify zero CCX.
- `test_cla_qq_purity_width_N` (parametrize N=2..4): BK CLA QQ addition (cla_override=0, qubit_saving=True). Verify zero CCX.
- `test_cla_cqq_purity_width_N` (parametrize N=2..4): Controlled BK CLA QQ addition. Verify zero CCX.

**class TestCliffordTCorrectness:**
Test that Clifford+T sequences produce correct arithmetic results (equivalence with non-decomposed path).

- `test_cdkm_qq_equivalence_width_N` (parametrize N=1..4): For all input pairs (a, b) where a,b in 0..2^N-1, verify that addition result with toffoli_decompose=True matches result with toffoli_decompose=False.
- `test_cdkm_cq_inc_correctness_width_N` (parametrize N=1..4): For all inputs a in 0..2^N-1, verify (a+1) mod 2^N.
- `test_cla_qq_equivalence_width_N` (parametrize N=2..4): BK CLA QQ, compare decomposed vs non-decomposed.
- `test_subtraction_works_width_2` and `test_subtraction_works_width_3`: Verify subtraction (inverted sequences) works correctly with Clifford+T.

**class TestCliffordTTCount:**
Test T-count accuracy.

- `test_t_count_exact_cdkm_qq_width_N` (parametrize N=2..4): With toffoli_decompose=True, verify T-count = T_gates + Tdg_gates (exact, not estimated). Verify T/Tdg ratio is 4:3 per CCX (4T + 3Tdg per CCX in source).
- `test_t_count_exact_cla_qq_width_N` (parametrize N=2..4): Same for BK CLA path.
- `test_t_count_zero_for_width_1`: Width-1 QQ is single CX, no T gates.

**class TestCliffordTFallback:**
- `test_width_9_uses_dynamic_path`: Width 9 with toffoli_decompose=True should still work (dynamic path). Verify no crash and correct result for at least one input pair.
- `test_cla_width_1_uses_cdkm_clifft`: Width 1 with CLA + toffoli_decompose=True. BK CLA returns NULL for width 1, so should fall through to CDKM Clifford+T path.

Use these patterns for test setup:

```python
c = ql.circuit()
ql.option('fault_tolerant', True)
ql.option('toffoli_decompose', True)
# For CLA: ql.option('qubit_saving', True) to select BK variant
```

IMPORTANT: Keep qubit counts under 17 per test (project constraint). Width-4 QQ addition uses 2*4+1=9 qubits, well within limit.

Mark width 5+ tests as `@pytest.mark.slow` if they use more than a few input pairs. For widths 1-4, exhaustive testing is feasible.
