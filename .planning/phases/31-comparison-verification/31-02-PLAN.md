**Custom pipeline helper:**
```python
import warnings
import pytest
import qiskit.qasm3
from qiskit_aer import AerSimulator
import quantum_language as ql

warnings.filterwarnings("ignore", message="Value .* exceeds")

def _run_comparison_pipeline(width, a, b, op_name, variant='qq'):
    """Run comparison and return full bitstring for analysis.

    Returns (bool_result, bitstring, num_qubits).
    """
    ql.circuit()
    qa = ql.qint(a, width=width)

    QL_OPS_QQ = {
        'eq': lambda qa, qb: qa == qb,
        'ne': lambda qa, qb: qa != qb,
        'lt': lambda qa, qb: qa < qb,
        'gt': lambda qa, qb: qa > qb,
        'le': lambda qa, qb: qa <= qb,
        'ge': lambda qa, qb: qa >= qb,
    }
    QL_OPS_CQ = {
        'eq': lambda qa, b: qa == b,
        'ne': lambda qa, b: qa != b,
        'lt': lambda qa, b: qa < b,
        'gt': lambda qa, b: qa > b,
        'le': lambda qa, b: qa <= b,
        'ge': lambda qa, b: qa >= b,
    }

    if variant == 'qq':
        qb = ql.qint(b, width=width)
        _result = QL_OPS_QQ[op_name](qa, qb)
    else:
        _result = QL_OPS_CQ[op_name](qa, b)

    qasm_str = ql.to_openqasm()
    circuit = qiskit.qasm3.loads(qasm_str)
    if not circuit.cregs:
        circuit.measure_all()

    sim = AerSimulator(method="statevector")
    job = sim.run(circuit, shots=1)
    counts = job.result().get_counts()
    bitstring = list(counts.keys())[0]

    # Boolean result is at bitstring[:1] (last allocated qubit = highest index)
    bool_result = int(bitstring[0], 2)
    return bool_result, bitstring, len(bitstring)
```

**Step 1: Calibration tests per operator.**

For each operator and variant (QQ, CQ), run a known case (width=3, a=3, b=5) and print/assert the bitstring layout. The calibration test determines where operand `a` and operand `b` sit in the bitstring.

Expected layout for simple operators (eq, lt CQ):
- Total qubits = 2*width + ancillae + 1 (qbool)
- `a` at bitstring[n-width:n] (first allocated, lowest qubit indices, rightmost in bitstring)
- `b` at bitstring[n-2*width:n-width]

For gt (widened comparison):
- Additional 2*(width+1) temporary qubits shift positions

Create a calibration function:
```python
def _calibrate_extraction(width, a, b, op_name, variant='qq'):
    """Empirically determine operand extraction positions.

    Returns dict with keys: 'a_start', 'a_end', 'b_start', 'b_end'
    describing slice positions in the bitstring.
    Returns None if calibration fails (operand not found).
    """
    bool_result, bitstring, n = _run_comparison_pipeline(width, a, b, op_name, variant)

    # Try standard positions first
    a_val = int(bitstring[n-width:n], 2)
    b_val = int(bitstring[n-2*width:n-width], 2)

    if a_val == a and b_val == b:
        return {'a_start': n-width, 'a_end': n, 'b_start': n-2*width, 'b_end': n-width}

    # Search for operand values in bitstring windows
    for start in range(n - width + 1):
        end = start + width
        val = int(bitstring[start:end], 2)
        if val == a:
            # Found a, look for b in remaining positions
            for b_start in range(n - width + 1):
                if b_start == start:
                    continue
                b_end = b_start + width
                b_val = int(bitstring[b_start:b_end], 2)
                if b_val == b:
                    return {'a_start': start, 'a_end': end, 'b_start': b_start, 'b_end': b_end}

    return None  # Could not find operand positions
```

Use `@pytest.fixture(scope="module")` for calibration results so they only run once per operator.

**Representative test pairs per operator:** (width=3)
- (0, 0), (0, 7), (7, 0), (7, 7) -- boundaries

**xfail handling:** If operand positions are unstable or cannot be determined, mark as xfail with BUG-CMP-PRES-XX IDs. Use strict=True. This is expected for __gt__ and operators that delegate to __gt__ due to widened temporaries.

**Anti-patterns to avoid:**
- Do NOT assume identical qubit layout across operators -- __gt__ has extra temporaries
- Do NOT skip calibration and hardcode positions -- positions vary by operator
- Do NOT test self-comparison (qa op qa) -- triggers optimization, no circuit
