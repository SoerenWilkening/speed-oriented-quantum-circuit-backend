**Oracle definitions for tests (define at module level or as local functions):**
- Use `@ql.grover_oracle` and `@ql.compile` decorators
- Oracle bodies use comparison operators: `flag = (x == 5); with flag: pass`
- Use `ql.option("fault_tolerant", True)` inside oracle if needed (or ql.grover() sets it)
- For bit-check oracles (even numbers), use `flag = (x[0] == 0)` pattern -- BUT this may need `ql.qbool` cast. Alternatively: mark values < 4 in a 3-bit space, or use x == specific values. Actually, the simplest multi-solution oracle: mark x where x < 4 in a 3-bit space (4 solutions: 0,1,2,3).

**IMPORTANT test pattern:** Since ql.grover() creates its own internal circuit, oracle definitions using @ql.grover_oracle/@ql.compile should be defined fresh (or use factory functions) to avoid stale circuit references. Use factory functions inside tests:

```python
def test_grover_single_solution_3bit(self):
    @ql.grover_oracle
    @ql.compile
    def mark_five(x: ql.qint):
        flag = (x == 5)
        with flag:
            pass

    # Run multiple times due to probabilistic single-shot
    found_count = 0
    for _ in range(10):
        value, iters = ql.grover(mark_five, width=3)
        if value == 5:
            found_count += 1
    assert found_count >= 7, f"Expected >=7/10 hits, got {found_count}"
    assert iters == 1  # N=8, M=1 -> k=1
```
