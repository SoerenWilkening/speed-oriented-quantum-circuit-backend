From 93-RESEARCH.md (Two's complement CLA subtraction approach):
```
QQ subtraction a -= b in min_depth mode:
1. X-gate all bits of b (one's complement negation)
2. CLA QQ_add (a += ~b) -- forward CLA addition
3. CQ_add (a += 1) -- two's complement correction (+1)
4. X-gate all bits of b (restore b to original state)

CQ subtraction a -= classical_value in min_depth mode:
- Compute negated_value = (1 << width) - classical_value (classical two's complement)
- Call CLA CQ addition with negated_value (no X gates needed)
```

Implement CLA-based subtraction via two's complement in `min_depth` mode. There are 4 subtraction paths to handle: QQ uncontrolled, QQ controlled, CQ uncontrolled, CQ controlled.

**Key principle:** When `circ->tradeoff_min_depth == 1` and `invert == 1` (subtraction), convert the subtraction into a forward CLA addition using two's complement negation. This avoids the BK CLA limitation (carry-copy ancilla not properly uncomputed in reverse).

**1. QQ uncontrolled subtraction (`toffoli_qq_uncont` function):**

Before the existing CLA dispatch check (`if (!invert && circ->cla_override == 0 ...`), insert a new block:

```c
/* Two's complement CLA subtraction for min_depth mode:
 * a -= b  ===  a += (~b + 1)
 * Steps: X(b) -> CLA(a += b) -> CQ(a += 1) -> X(b)
 * This enables O(log n) depth subtraction using CLA instead of O(n) RCA. */
if (invert && circ->tradeoff_min_depth && circ->cla_override == 0
    && result_bits >= circ->tradeoff_auto_threshold) {

    int cla_ancilla_count = compute_cla_ancilla_count(circ, result_bits);
    qubit_t cla_ancilla = allocator_alloc(circ->allocator, cla_ancilla_count, true);
    if (cla_ancilla != (qubit_t)-1) {
        /* Step 1: X-gate each bit of other (one's complement negation) */
        for (i = 0; i < other_bits; i++) {
            gate_t xg;
            memset(&xg, 0, sizeof(gate_t));
            x(&xg, other_qubits[i]);
            add_gate(circ, &xg);
        }

        /* Step 2: CLA addition a += ~b (forward direction, invert=0) */
        for (i = 0; i < cla_ancilla_count; i++) {
            tqa[2 * result_bits + i] = cla_ancilla + i;
        }
        sequence_t *cla_seq;
        if (circ->qubit_saving) {
            cla_seq = toffoli_QQ_add_bk(result_bits);
        } else {
            cla_seq = toffoli_QQ_add_ks(result_bits);
        }
        if (cla_seq != NULL) {
            run_instruction(cla_seq, tqa, 0, circ);  /* invert=0: forward addition */
            allocator_free(circ->allocator, cla_ancilla, cla_ancilla_count);

            /* Step 3: CQ increment a += 1 (two's complement correction) */
            /* Use direct CDKM CQ add with value=1, allocating 1 ancilla */
            qubit_t inc_ancilla = allocator_alloc(circ->allocator, 1, true);
            if (inc_ancilla != (qubit_t)-1) {
                unsigned int inc_qa[128];
                for (i = 0; i < self_bits; i++) {
                    inc_qa[i] = self_qubits[i];
                }
                inc_qa[self_bits] = inc_ancilla;
                toffoli_CQ_add(inc_qa, self_bits, 1, 0, circ);  /* value=1, invert=0 */
                allocator_free(circ->allocator, inc_ancilla, 1);
            }

            /* Step 4: Restore other register via X gates */
            for (i = 0; i < other_bits; i++) {
                gate_t xg;
                memset(&xg, 0, sizeof(gate_t));
                x(&xg, other_qubits[i]);
                add_gate(circ, &xg);
            }
            return;
        }
        /* CLA failed -- fall through to RCA subtraction */
        allocator_free(circ->allocator, cla_ancilla, cla_ancilla_count);

        /* Undo X gates since we're falling through */
        for (i = 0; i < other_bits; i++) {
            gate_t xg;
            memset(&xg, 0, sizeof(gate_t));
            x(&xg, other_qubits[i]);
            add_gate(circ, &xg);
        }
    }
}
```

"""

Notes
-----
...

The 'tradeoff' option controls adder selection for Toffoli arithmetic:
- 'auto' (default): Uses CLA (carry look-ahead) for widths >= 4, CDKM
  (ripple-carry) for smaller widths. Balances depth and qubit count.
- 'min_depth': Always uses CLA for minimum circuit depth. Subtraction

of the tradeoff setting, as the Beauregard modular adder requires
subtraction comparisons that are not compatible with CLA.

The tradeoff option must be set before any arithmetic operations (+, -, *).

to prevent inconsistent circuits.
