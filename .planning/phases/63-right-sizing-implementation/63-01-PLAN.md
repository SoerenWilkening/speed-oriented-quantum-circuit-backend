  <action>
    Modify `scripts/generate_seq_all.py` to factor out shared QFT/IQFT sub-sequences. The key changes are in `generate_width_file()` and supporting functions.

    **CRITICAL DESIGN DECISION -- Boundary Merging:**
    The current `optimize_layers()` merges any consecutive layers with non-overlapping qubits. This causes the last QFT layer and first middle layer (and last middle layer and first IQFT layer) to potentially merge, making them unshareble. To enable clean sharing, apply optimization in THREE segments:
    1. Optimize QFT layers independently
    2. Optimize middle layers independently
    3. Optimize IQFT layers independently
    Then concatenate the three optimized segments. This prevents cross-boundary merging.

    **Part A: Static const sharing (QQ_add and cQQ_add)**

    1. Add a new function `generate_shared_qft_c(width)` that:
       - Calls `_generate_qft_layers(width)` to get raw QFT layers
       - Applies `optimize_layers()` to just the QFT layers
       - Uses `generate_c_sequence("SHARED_QFT", optimized_qft_layers, width)` to emit shared static const arrays
       - Returns the optimized QFT layers (for use in LAYERS array references)

    2. Add a new function `generate_shared_iqft_c(width)` that:
       - Calls `_generate_iqft_layers(width)` to get raw IQFT layers
       - Applies `optimize_layers()` to just the IQFT layers
       - Uses `generate_c_sequence("SHARED_IQFT", optimized_iqft_layers, width)` to emit shared static const arrays
       - Returns the optimized IQFT layers (for use in LAYERS array references)

    3. Add a new function `generate_qq_add_middle(width)` that:
       - Generates the QQ_add gate sequence WITHOUT QFT/IQFT (just the addition phase rotations)
       - This is the middle section of `generate_qq_add()`: the nested loop producing controlled rotations from the b-register
       - Applies `optimize_layers()` to just the middle layers
       - Uses `generate_c_sequence("QQ_ADD_MID", optimized_middle, width)` to emit only the unique middle layers
       - Returns the optimized middle layers

    4. Add a new function `generate_cqq_add_middle(width)` that:
       - Generates the cQQ_add gate sequence WITHOUT QFT/IQFT (Block 1 + Block 2 + Block 3)
       - This is the middle section of `generate_cqq_add()`: unconditional half-rotations, CNOT+neg rotations, controlled rotations
       - Applies `optimize_layers()` to just the middle layers
       - Uses `generate_c_sequence("cQQ_ADD_MID", optimized_middle, width)` to emit only the unique middle layers
       - Returns the optimized middle layers

    5. Add a new function `generate_composite_layers_array(name, shared_qft_layers, middle_layers, shared_iqft_layers, width)` that:
       - Emits the composite LAYERS array referencing shared QFT layer names, unique middle layer names, and shared IQFT layer names
       - Emits the composite GPL (gates_per_layer) array combining all three sections
       - Emits the composite sequence_t struct
       - Example output for QQ_ADD width 2:
         ```c
         static const gate_t *QQ_ADD_2_LAYERS[] = {
             SHARED_QFT_2_L0, SHARED_QFT_2_L1, SHARED_QFT_2_L2,  // shared QFT
             QQ_ADD_MID_2_L0, QQ_ADD_MID_2_L1,                     // unique middle
             SHARED_IQFT_2_L0, SHARED_IQFT_2_L1, SHARED_IQFT_2_L2  // shared IQFT
         };
         static const num_t QQ_ADD_2_GPL[] = {1, 1, 1, 1, 1, 2, 1, 1};
         static const sequence_t HARDCODED_QQ_ADD_2 = {...};
         ```

    6. Modify `generate_width_file(width)` to:
       - First emit shared QFT/IQFT static const arrays (called once per width)
       - Then emit QQ_ADD_MID unique layers + composite QQ_ADD LAYERS/GPL/struct
       - Then emit cQQ_ADD_MID unique layers + composite cQQ_ADD LAYERS/GPL/struct
       - Keep the CQ_add and cCQ_add template-init sections as-is for now (they are separate -- see Part B)
       - Keep the dispatch helpers unchanged

    **VALIDATION for Part A:** The total layer count for QQ_ADD and cQQ_ADD MUST remain the same as before (the composite array has exactly as many entries as the original). Run `python scripts/generate_seq_all.py --validate` to confirm layer counts match. NOTE: since boundary optimization is now prevented, the optimized layer counts MAY differ slightly from the previous expected values. Update the expected counts in `validate()` if needed, but the actual gate sequences must produce identical arithmetic results (validated by pytest).

    **Part B: Template-init sharing (CQ_add and cCQ_add)**

    7. Add a new function `generate_shared_qft_init_helper(width)` that emits a C helper function:
       ```c
       static void init_shared_qft_layers_N(sequence_t *seq, int start_layer) {
           // Initialize QFT layers starting at start_layer
           // Same logic as current CQ_add QFT initialization, but parameterized by start offset
       }
       ```
       This function sets `seq->gates_per_layer[start_layer + i]` and allocates `seq->seq[start_layer + i]` for each QFT layer.

    8. Add a new function `generate_shared_iqft_init_helper(width)` that emits a similar helper for IQFT layers.

    9. Modify `generate_cq_add_template_c(bits)` to:
       - Call `init_shared_qft_layers_N(seq, 0)` instead of emitting QFT initialization inline
       - Keep the rotation layer initialization (the unique middle part)
       - Call `init_shared_iqft_layers_N(seq, qft_layers + bits)` instead of emitting IQFT initialization inline

    10. Modify `generate_ccq_add_template_c(bits)` to:
        - Call `init_shared_qft_layers_N(seq, 0)` instead of emitting QFT initialization inline
        - Keep the rotation layer initialization (the unique middle part)

    - All 4 variants (QQ, cQQ, CQ, cCQ) must continue to produce IDENTICAL arithmetic results. The sharing is purely about code structure, not algorithm changes.
    - The `HARDCODED_QQ_ADD_N` and `HARDCODED_cQQ_ADD_N` struct names and dispatch helper function signatures MUST NOT change -- the dispatch file and sequences.h API depend on them.
