**Create `src/quantum_language/sim_backend.py`:**
Create the thin wrapper module as specified in 82-RESEARCH.md Pattern 1. Key design:
- Module-level `_QISKIT_AVAILABLE = None` and `_AER_AVAILABLE = None` sentinel caches
- `_check_qiskit()` and `_check_aer()` functions that lazily check importability (cached)
- `_require_backend()` raises ImportError if qiskit not available
- `_require_simulator()` raises ImportError if qiskit-aer not available (calls `_require_backend()` first)
- Error message: `"Simulation backend required. Install with: pip install quantum_language[verification]"` — NO mention of qiskit anywhere
- `load_qasm(qasm_str)` — calls `_require_backend()`, then `import qiskit.qasm3; return qiskit.qasm3.loads(qasm_str)`
- `simulate(circuit, *, shots=1, max_parallel_threads=4)` — calls `_require_simulator()`, then does the transpile+AerSimulator+run pipeline. If `not circuit.cregs`, call `circuit.measure_all()` first.
- Guards qiskit and qiskit-aer separately (some features may work with just qiskit)
- Add module-level docstring explaining this is a thin abstraction layer for future backend swaps

**Migrate `src/quantum_language/grover.py`:**
In the `_simulate_single_shot` function (around line 258-269):
1. Remove the 3 local import lines (`import qiskit.qasm3`, `from qiskit import transpile`, `from qiskit_aer import AerSimulator`)
2. Replace with `from .sim_backend import load_qasm, simulate` at the top of the function (or at module level within a lazy import pattern)
3. Replace the body: `circuit_qk = load_qasm(qasm_str)` and `counts = simulate(circuit_qk, shots=1)` — then parse bitstring as before
4. The `max_parallel_threads=4` is already the default in sim_backend.simulate, so no need to pass it

Since grover.py's `_simulate_single_shot` currently does the qiskit import inside the function body, the simplest migration is to add `from .sim_backend import load_qasm, simulate` at module level (top of file, in the imports section), then rewrite the function body to use `load_qasm` and `simulate`.

**Migrate `src/quantum_language/amplitude_estimation.py`:**
