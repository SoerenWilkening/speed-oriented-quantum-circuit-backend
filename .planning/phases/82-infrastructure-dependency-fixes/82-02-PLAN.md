  <action>
**Create `scripts/record_baseline.py`:**

The script parses the coverage JSON report (generated by `pytest --cov --cov-report=json`) and records a structured baseline. It should:

1. **Get git metadata:**
   - Current commit hash via `git rev-parse HEAD`
   - Current branch via `git rev-parse --abbrev-ref HEAD`
   - Timestamp at execution time (ISO 8601 format)

2. **Parse coverage JSON:**
   - Read `reports/coverage/coverage.json` (generated by prior `pytest --cov --cov-report=json:reports/coverage/coverage.json`)
   - Extract `totals.percent_covered` as overall percentage
   - Extract per-file data: `percent_covered`, `missing_lines` count, `covered_lines` count

3. **Per-module breakdown:**
   - Group files by top-level module directory (e.g., `quantum_language/grover.py`)
   - Record for each file: `percent_covered`, `missing_lines`, `covered_lines`, `total_lines`

4. **Automated gap identification:**
   - Identify files with coverage < 50% as "critical untested"
   - Identify files with coverage between 50-80% as "partially tested"
   - For files with coverage < 50%, list the count of missing lines
   - Sort critical files by missing line count (most missing first)

5. **Output structure (JSON):**
```json
{
  "timestamp": "2026-02-22T...",
  "commit": "abc123...",
  "branch": "develop",
  "overall_percent": 42.5,
  "total_lines": 5000,
  "covered_lines": 2125,
  "missing_lines": 2875,
  "modules": {
    "quantum_language/grover.py": {
      "percent": 65.0,
      "covered_lines": 130,
      "missing_lines": 70,
      "total_lines": 200
    }
  },
  "critical_untested": [
    {"file": "quantum_language/foo.py", "percent": 12.0, "missing_lines": 150}
  ],
  "partially_tested": [
    {"file": "quantum_language/bar.py", "percent": 60.0, "missing_lines": 40}
  ]
}
```

6. **Write baseline to `reports/coverage/baseline.json`**

7. **Print summary to stdout:**
   - Overall coverage percentage
   - Number of critical untested files
   - Top 5 files by missing lines
   - Message: "Baseline recorded at reports/coverage/baseline.json"

8. **CLI interface:** The script should be runnable as `python scripts/record_baseline.py` with no arguments. It should:
   - Check that `reports/coverage/coverage.json` exists (fail with clear message if not — user needs to run `make coverage` or `pytest --cov` first)
   - Create `reports/coverage/` directory if it doesn't exist
   - Overwrite previous baseline if it exists (with a warning message)

**Run the coverage pipeline and record baseline:**

After creating the script, execute the full pipeline:
1. Build with coverage: `. .venv/bin/activate && QUANTUM_COVERAGE=1 python3 setup.py build_ext --inplace --force`
   - NOTE: This may take a few minutes due to linetrace instrumentation rebuild
2. Run tests with coverage: `pytest tests/python -v --cov=quantum_language --cov-report=json:reports/coverage/coverage.json --cov-report=html:reports/coverage/html`
   - NOTE: Do NOT use `--cov-report=lcov` unless lcov is installed (it may error)
3. Run the baseline script: `python scripts/record_baseline.py`
4. Verify the baseline was recorded

**Important constraints:**
- Max 17 qubits for any simulation (project constraint — already enforced in tests)
- Max 4 threads for AerSimulator (project constraint — already enforced in code)
- The coverage build will be slower than normal due to linetrace; this is expected
- After recording baseline, rebuild WITHOUT coverage flags so normal builds aren't affected: `python3 setup.py build_ext --inplace --force`
