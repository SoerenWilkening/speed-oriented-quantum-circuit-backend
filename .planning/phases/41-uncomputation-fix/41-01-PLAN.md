<objective>
Integrate `.pxi` include files into `qint.pyx` and add missing layer tracking to arithmetic and division operations, fixing the uncomputation regression.

Purpose: The uncomputation regression is caused by two interacting issues: (1) comparison and arithmetic operations in `qint.pyx` lack `_start_layer`/`_end_layer` tracking, so `_do_uncompute()` never reverses their gates; (2) corrected `.pxi` files exist with proper layer tracking (for comparisons and bitwise) but are NOT included in the build. This plan integrates the `.pxi` files and adds layer tracking where missing.

Output: Modified `qint.pyx` with `include` directives replacing inline implementations, and updated `qint_arithmetic.pxi` / `qint_division.pxi` with proper layer tracking on all operations that create result qints.
</objective>

  </files>
  <action>
Add layer tracking to all operations in `qint_arithmetic.pxi` and `qint_division.pxi` that create new result qints. Follow the exact pattern from `__and__` in `qint_bitwise.pxi`:

**For `qint_arithmetic.pxi`**, add to `__add__`, `__radd__`, `__sub__`, `__mul__`, `__rmul__`:
1. Capture `start_layer = (<circuit_s*>_circuit).used_layer if _circuit_initialized else 0` BEFORE any gates are applied
2. After all gates, set `result._start_layer = start_layer`
3. Set `result._end_layer = (<circuit_s*>_circuit).used_layer if _circuit_initialized else 0`
4. Set `result.operation_type` to 'ADD', 'SUB', or 'MUL' as appropriate
5. Add `result.add_dependency(self)` and `result.add_dependency(other)` (if other is qint)

Do NOT modify in-place operations (`__iadd__`, `__isub__`, `__imul__`) -- they modify `self`, not a new result.

**For `qint_division.pxi`**, add to `__floordiv__`, `__mod__`, `__divmod__`, `__rfloordiv__`, `__rmod__`, `__rdivmod__`:
1. Same pattern: capture `start_layer` before, set `_start_layer`/`_end_layer` after
2. Set `operation_type` to 'DIV', 'MOD', or 'DIVMOD'
3. Add dependencies on self and divisor (if qint)

**Important:** The `_circuit` and `_circuit_initialized` variables are module-level globals in `qint.pyx` and are accessible from `.pxi` files since they're textually included. The `circuit_s` type is declared in `qint.pxd`. Check existing `.pxi` files (`qint_comparison.pxi`, `qint_bitwise.pxi`) for the exact syntax used.
