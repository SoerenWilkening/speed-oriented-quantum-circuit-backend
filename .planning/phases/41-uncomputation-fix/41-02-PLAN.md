  <name>Task 1: Add layer tracking to __lt__ and __gt__ results in qint.pyx</name>
  <files>src/quantum_language/qint.pyx</files>
  <action>
    **Background:** Decision D41-01-3 excluded layer tracking from __lt__ and __gt__ because "widened temps self-uncompute via GC; setting layer tracking would double-reverse gates." This reasoning was INCORRECT: the widened temps (temp_self, temp_other) do NOT have their own layer tracking, so they will NOT reverse any gates on GC. Only the result qbool needs layer tracking, and setting it on the result will correctly reverse all widened-temp gates (copy CNOTs, subtract, MSB extract) when the result is uncomputed. After reversal, the widened temps' qubits will be in all-zero state and GC can safely free them with no gates to reverse.

    **The qint_comparison.pxi file already has this fix** (lines 265-267 for __lt__, lines 376-378 for __gt__). The inline qint.pyx code needs to match.

    **Changes to make in `__lt__` (qint-vs-qint path, around line 1925):**

    Replace these lines:
    ```python
    result.operation_type = 'LT'
    # Note: NOT setting _start_layer/_end_layer for widened-temp comparisons.
    # The widened temps (temp_self, temp_other) self-uncompute via GC.
    # Setting layer tracking here would double-reverse those gates.
    return result
    ```

    With:
    ```python
    result.operation_type = 'LT'
    # Phase 41 gap closure: Add layer tracking so widened-temp gates are
    # reversed when result is uncomputed. The widened temps themselves have
    # no layer tracking, so there is no double-reversal risk.
    result._start_layer = start_layer
    result._end_layer = (<circuit_s*>_circuit).used_layer if _circuit_initialized else 0
    return result
    ```

    **Changes to make in `__gt__` (qint-vs-qint path, around line 2022):**

    Replace these lines:
    ```python
    result.operation_type = 'GT'
    # Note: NOT setting _start_layer/_end_layer for widened-temp comparisons.
    # The widened temps self-uncompute via GC.
    return result
    ```

    With:
    ```python
    result.operation_type = 'GT'
    # Phase 41 gap closure: Add layer tracking so widened-temp gates are
    # reversed when result is uncomputed. The widened temps themselves have
    # no layer tracking, so there is no double-reversal risk.
    result._start_layer = start_layer
    result._end_layer = (<circuit_s*>_circuit).used_layer if _circuit_initialized else 0
    return result
    ```

    **IMPORTANT:** Both methods already have `start_layer` captured at the top of the method and `_circuit`/`_circuit_initialized` declared as cdef variables. No new variable declarations are needed. The .pxi file confirms this pattern works.

    **Do NOT modify:** Any other comparison methods (__eq__, __ne__, __le__, __ge__) -- they already work correctly. __le__ and __ge__ delegate to __gt__ and __lt__ respectively, so they inherit the fix automatically.
