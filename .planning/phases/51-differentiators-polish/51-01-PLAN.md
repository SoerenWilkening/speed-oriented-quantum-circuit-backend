   - `__init__(self, original)`: Store `self._original = original`. Initialize `self._inv_cache = {}` (keyed same as original's cache, stores inverted CompiledBlocks). Use `functools.update_wrapper(self, original._func)`.
   - `__call__(self, *args, **kwargs)`: Classify args same way as CompiledFunc (call `self._original._classify_args(args, kwargs)`). Detect controlled context and build cache key same as CompiledFunc.__call__. Check `self._inv_cache` for an inverted block. If not found, check `self._original._cache` for the original block -- if not there either, call `self._original(*args, **kwargs)` to populate the cache, then look up the block. Invert the block's gates with `_inverse_gate_list(block.gates)`, create a new `CompiledBlock` copying all metadata from the original block (total_virtual_qubits, param_qubit_ranges, internal_qubit_count, return_qubit_range, return_is_param_index, original_gate_count, control_virtual_idx) but with the inverted gates. Cache the inverted block in `self._inv_cache`. Call `self._original._replay(inverted_block, quantum_args)`.
   - `.inverse()`: Return `self._original` (round-trip: inverse of inverse = original).
   - `clear_cache(self)`: Clear `self._inv_cache`.
   - `__repr__(self)`: Return `f"<InverseCompiledFunc {self._original._func.__name__}>"`.

8. Update `compile()` decorator function signature to accept `inverse=False` and pass through to `CompiledFunc`. When `inverse=True`, eagerly create the `_InverseCompiledFunc` in `__init__` (assign `self._inverse_func = _InverseCompiledFunc(self)`).

**Important implementation notes:**
- `_InverseCompiledFunc` does NOT inherit from `CompiledFunc`. It is a lightweight wrapper.
- `_InverseCompiledFunc` reuses the original's `_classify_args`, `_replay`, and cache, but does NOT modify the original's cache.
- When inverting a controlled block, preserve `control_virtual_idx` so replay maps the control qubit correctly.
- The inverted block's `return_qubit_range` and `return_is_param_index` are copied from the original (the return value is the same; inverse affects gate order, not return semantics).
