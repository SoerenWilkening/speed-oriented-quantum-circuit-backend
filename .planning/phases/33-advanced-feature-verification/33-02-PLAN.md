**Important constraints from RESEARCH.md:**

**Test structure — paired True/False tests:**

For each condition source, create TWO tests: one where the condition evaluates True (operation executes) and one where it evaluates False (operation is skipped). The result register starts at 0; if the gated `+= 1` executes, result is 1; if skipped, result is 0.

**CQ Greater-than conditional (gt, no known bugs for small values):**
- `test_cond_gt_true`: a=3 (width=3), condition `a > 1` is True. `with cond: result += 1`. Expected: 1.
  ```python
  def build():
      a = ql.qint(3, width=3)
      cond = (a > 1)
      result = ql.qint(0, width=3)
      with cond:
          result += 1
      return (1, [a, cond, result])
  ```
- `test_cond_gt_false`: a=0 (width=3), condition `a > 1` is False. Expected: 0.

**CQ Less-than conditional (lt):**
- `test_cond_lt_true`: a=1 (width=3), condition `a < 3` is True. Expected: 1.
- `test_cond_lt_false`: a=5 (width=3), condition `a < 3` is False. Expected: 0.

**QQ Greater-than conditional:**
- `test_cond_qq_gt_true`: a=3, b=1 (width=3), condition `a > b` is True. Expected: 1.
- `test_cond_qq_gt_false`: a=1, b=3 (width=3), condition `a > b` is False. Expected: 0.

**CQ Equality conditional (BUG-CMP-01 — xfail):**
- `test_cond_eq_true`: a=2 (width=3), condition `a == 2` is True. Expected: 1. xfail reason="BUG-CMP-01" (eq inverted).
- `test_cond_eq_false`: a=3 (width=3), condition `a == 2` is False. Expected: 0. xfail reason="BUG-CMP-01".

**CQ Not-equal conditional (BUG-CMP-01 — xfail):**
- `test_cond_ne_true`: a=3 (width=3), condition `a != 2` is True. Expected: 1. xfail reason="BUG-CMP-01" (ne inverted).
- `test_cond_ne_false`: a=2 (width=3), condition `a != 2` is False. Expected: 0. xfail reason="BUG-CMP-01".

**Conditional subtraction:**
- `test_cond_sub_true`: a=5 (width=3), condition `a > 2` is True. `with cond: result -= 1` (result starts at 3). Expected: 2.
- `test_cond_sub_false`: a=1 (width=3), condition `a > 2` is False. result starts at 3. Expected: 3 (unchanged).

**Conditional multiplication (untested cCQ_mul — may need xfail):**

**Important implementation details:**
- Always use `gc.collect()` in circuit builder or rely on conftest doing it
- Always return keepalive refs: `return (expected, [a, cond, result])`
- Width=3 for all tests (CONTEXT.md says 2-3 bits, 3 gives more interesting values)

**Value safety guide for width=3 (to avoid BUG-CMP-02):**
- Safe pairs for gt/lt: both values in [0,3] (MSB=0) or both in [4,7] (MSB=1)
- Risky pairs: one value < 4, other >= 4 (MSBs differ)
- For simplicity, keep all condition operands in [0,3] range for non-xfailed tests
