Create `tests/test_uncomputation.py` that verifies automatic uncomputation through the full pipeline.

**Architecture:** This test file CANNOT use the standard `verify_circuit` fixture because that fixture only extracts `bitstring[:width]` (the result register). Uncomputation verification requires inspecting ALL qubits — non-result qubits must be |0>. Build a custom pipeline function similar to `_run_comparison_pipeline` in `test_compare_preservation.py`.

**Custom pipeline function `_run_uncomputation_pipeline(circuit_builder, result_width)`:**
1. `gc.collect()` then `ql.circuit()`
2. `ql.option('qubit_saving', True)` to enable EAGER uncomputation
3. Call `circuit_builder()` which returns `(expected_value, [keepalive_refs])`
4. `qasm_str = ql.to_openqasm()`
5. `ql.option('qubit_saving', False)` to reset
6. Release keepalive refs
7. Load QASM, `measure_all()`, simulate with `AerSimulator(method="statevector")`, shots=1
8. Return `(bitstring, expected_value)` for analysis

**Checking function `_check_uncomputation(bitstring, result_width, expected)`:**
- Result bits: `bitstring[:result_width]` (highest qubit indices = leftmost in bitstring)
- Ancilla bits: `bitstring[result_width:]`
- Verify: `int(result_bits, 2) == expected` AND `all(b == '0' for b in ancilla_bits)`
- Return `(actual_result, ancilla_clean, ancilla_bits)` for assertion messages

**Calibration test (run FIRST):**
- `test_uncomp_calibration_add` — 2-bit: a=1, b=2, result=3. Prints full bitstring and qubit count. This empirically confirms the extraction positions work. Use a simple `assert` but print diagnostics.

**Arithmetic uncomputation tests (parametrized):**
- `test_uncomp_add` — parametrized with `(a, b, width)` pairs: width=2 (a=1,b=2), width=3 (a=3,b=4). Expected: `(a+b) % (1<<width)`. Check result correct AND ancilla clean.
- `test_uncomp_sub` — parametrized: width=2 (a=3,b=1), width=3 (a=5,b=2). Expected: `(a-b) % (1<<width)`. Check result AND ancilla.
- `test_uncomp_mul` — parametrized: width=2 (a=1,b=2), width=3 (a=2,b=3). Expected: `(a*b) % (1<<width)`. Check result AND ancilla. Multiplication circuits are large; limit to small values.

**Comparison uncomputation tests:**
- `test_uncomp_comparison` — parametrized with `(a, b, width, op_name)`. Operations: `gt` (a=3,b=1,w=2), `lt` (a=1,b=3,w=3). Use lambda dict like `test_compare_preservation.py`. Expected: Python `eval` of comparison. Result is 1-bit qbool. xfail tests that use `eq`/`ne` (BUG-CMP-01) or MSB-spanning values (BUG-CMP-02).

**Compound boolean uncomputation tests:**
- `test_uncomp_compound_and` — `(a > 1) & (b < 3)` with a=2, b=1, width=2. Both True -> result True. Multiple sub-expressions = multiple ancillae to uncompute.
- `test_uncomp_compound_or` — `(a > 1) | (b < 1)` with a=2, b=2, width=2. First True, second False -> result True.
- `test_uncomp_compound_eq` — `(a == 2) & (b < 3)` using eq. xfail with reason="BUG-CMP-01" (eq inverted).

**xfail strategy:**
- Tests using `eq`/`ne` comparisons: `@pytest.mark.xfail(reason="BUG-CMP-01", strict=False)`
- Tests where comparison operands span MSB: `@pytest.mark.xfail(reason="BUG-CMP-02", strict=False)`
- If ancilla are not clean but result is correct, that's a partial failure — assert both separately with clear messages.

**Important implementation details:**
- Always `import gc` and call `gc.collect()` before `ql.circuit()`
- Always return keepalive refs from circuit builders: `return (expected, [a, b, result])`
- Use `warnings.filterwarnings("ignore", message="Value .* exceeds")` at module level
- Keep bit widths to 2-3 as specified in CONTEXT.md
- Add module docstring explaining this tests VADV-01
