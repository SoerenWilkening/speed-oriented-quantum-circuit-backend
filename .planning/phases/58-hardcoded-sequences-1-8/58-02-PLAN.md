  <action>
Create `c_backend/src/sequences/add_seq_5_8.c` implementing hardcoded QQ_add and cQQ_add for widths 5-8.

**Follow exact same pattern as add_seq_1_4.c** but for larger widths.

**Gate counts for reference (QQ_add):**
- Width 5: 45 gates, 23 layers (5*5-2)
- Width 6: 63 gates, 28 layers (5*6-2)
- Width 7: 84 gates, 33 layers (5*7-2)
- Width 8: 108 gates, 38 layers (5*8-2)

**Gate counts for cQQ_add are much higher due to CNOT + half-rotation structure.**
- Refer to cQQ_add() layer formula: `bits * (bits + 1) / 2 * 4 + 4 * bits - 2 - bits / 4 * 4 + 3`

**At end of file, extend dispatch functions:**
```c
// Forward declarations from add_seq_1_4.c
extern const sequence_t* get_hardcoded_QQ_add(int bits);
extern const sequence_t* get_hardcoded_cQQ_add(int bits);

// Local dispatch for 5-8
static const sequence_t* get_hardcoded_QQ_add_5_8(int bits) {
    switch (bits) {
        case 5: return &HARDCODED_QQ_ADD_5;
        case 6: return &HARDCODED_QQ_ADD_6;
        case 7: return &HARDCODED_QQ_ADD_7;
        case 8: return &HARDCODED_QQ_ADD_8;
        default: return NULL;
    }
}
```

**IMPORTANT:** The dispatch functions in add_seq_1_4.c need to be modified to also check the 5-8 range. Consider having add_seq_1_4.c call into add_seq_5_8.c for widths 5-8, OR modify the dispatch to be a single unified function.

**Recommended approach:** Keep dispatch functions separate per file. The caller (IntegerAddition.c) will call get_hardcoded_QQ_add() which returns NULL for > 4, then caller checks 5-8 range with get_hardcoded_QQ_add_5_8().

**Alternative:** Have a single dispatch function that handles all 1-8, implemented in one file and declared in header. This is cleaner - implement in add_seq_5_8.c and update header.

**Go with alternative:** Implement the full dispatch in add_seq_5_8.c (referencing statics from add_seq_1_4.c via extern), and update sequences.h to only declare the unified dispatch.
