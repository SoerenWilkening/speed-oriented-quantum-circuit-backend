   - For each width, test EVERY value from 0 to `2^width - 1` using `generate_exhaustive_values(width)` from `verify_helpers`
   - For each value at each width:
     - Define `circuit_builder` that calls `ql.qint(value, width=width)` and returns `value`

     - Assert `actual == expected` with message from `format_failure_message('init', [value], width, expected, actual)`
   - Total test cases: 2 + 4 + 8 + 16 = 30 values across 4 widths
   - Implementation: Use a single parametrized test with width, then loop over all values inside. This avoids 30 separate test IDs while still testing exhaustively. 2. **`test_init_sampled`** - Parameterized test for widths 5-8 bits with sampled values.
   - Use `@pytest.mark.parametrize("width", [5, 6, 7, 8])`
   - For each width, test representative values using `generate_sampled_values(width)` from `verify_helpers`
   - Same pattern as exhaustive: build circuit, verify, assert with formatted message
   - Implementation: Same parametrize-with-tuples approach as exhaustive for consistent style.

3. **Module-level helpers** for generating parametrize data:
   - `_exhaustive_cases()` - Returns list of `(width, value)` tuples for widths 1-4
   - `_sampled_cases()` - Returns list of `(width, value)` tuples for widths 5-8

Import `quantum_language as ql`, `pytest`, and from `verify_helpers` import the generation and formatting functions.
