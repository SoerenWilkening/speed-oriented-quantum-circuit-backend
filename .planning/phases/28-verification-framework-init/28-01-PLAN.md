  <action>
Create `tests/verify_helpers.py` with these functions:

1. `generate_exhaustive_values(width)` - Returns list of all values 0 to 2^width - 1. Used for single-operand tests (init) at widths 1-4.

2. `generate_exhaustive_pairs(width)` - Returns list of all (a, b) tuples using `itertools.product(range(2**width), repeat=2)`. Used for binary operation tests at small widths.

3. `generate_sampled_values(width, sample_size=50)` - Returns list of representative values for larger widths (5+). Includes edge cases (0, 1, max_val, max_val-1) plus `sample_size` random values with `random.seed(42)` for determinism. Deduplicates via `set`.

4. `generate_sampled_pairs(width, sample_size=50)` - Returns list of representative (a, b) pairs for larger widths. Includes all edge-case pairs from `itertools.product([0, 1, max_val, max_val-1], repeat=2)` plus `sample_size` random pairs. Uses `random.seed(42)`. Deduplicates.

5. `format_failure_message(op_name, operands, width, expected, actual)` - Returns compact one-liner string: `"FAIL: {op_name}({operands_csv}) {width}-bit: expected={expected}, got={actual}"`. For single operand (init): `"FAIL: init(5) 4-bit: expected=5, got=3"`. For binary ops: `"FAIL: add(3,5) 4-bit: expected=8, got=6"`.

All functions should have clear docstrings. Use `import itertools` and `import random` from stdlib. No external dependencies needed.

<task type="auto">
  <name>Task 2: Create tests/conftest.py with verify_circuit fixture</name>
  <files>tests/conftest.py</files>
  <action>
Create `tests/conftest.py` (root-level conftest for verification tests, separate from `tests/python/conftest.py` which serves unit tests).

This file provides the core `verify_circuit` pytest fixture that encapsulates the full pipeline:

1. **`verify_circuit` fixture** (function scope) - Returns a callable `_verify(circuit_builder, width)` where:
   - `circuit_builder` is a callable that builds a quantum circuit using `ql` API and returns the expected integer result
   - `width` is the bit width of the result register to extract
   - The callable:
     a. Calls `ql.circuit()` to reset circuit state
     b. Calls `circuit_builder()` to build circuit and get expected value
     c. Calls `ql.to_openqasm()` to export OpenQASM 3.0
     d. Loads QASM into Qiskit via `qiskit.qasm3.loads()`
     e. Adds measurements if circuit has no classical registers (`if not circuit.cregs: circuit.measure_all()`)
     f. Simulates with `AerSimulator(method='statevector')`, `shots=1`
     g. Extracts result: gets bitstring from counts, takes first `width` chars (MSB-first), converts to int
     h. Returns `(actual, expected)` tuple
   - If simulation fails, raise a clear error with the QASM string included for debugging

2. **`clean_circuit` fixture** (function scope) - Simple fixture that calls `ql.circuit()` and yields. For tests that need circuit reset without full verification pipeline.

Import `quantum_language as ql`, `qiskit.qasm3`, and `from qiskit_aer import AerSimulator`.

IMPORTANT: The existing `tests/python/conftest.py` must NOT be modified. Pytest automatically uses conftest.py files at each directory level. The new `tests/conftest.py` serves the verification test files (verify_*.py) at the tests/ root level.

IMPORTANT: For NOT operations (in-place, bitstring length == width), the result extraction differs. Add a parameter `in_place=False` to `_verify()`. When `in_place=True`, use the full bitstring instead of first `width` chars. This handles NOT and similar in-place operations. Default is `False` for most operations.
  </action>

Verify the fixture function exists and is properly decorated. Also verify imports work:
