Add test functions to `tests/test_compile.py` for INV-07 scenarios. Follow existing test patterns in the file (use `ql.circuit()`, `ql.qint()`, `@ql.compile`, `ql.circuit_stats()`). Enable qubit-saving with `ql.option("qubit_saving", True)` at the start of each test (after `ql.circuit()`).

Required tests:

1. **test_auto_uncompute_basic**: Create a compiled function that allocates a temp ancilla and returns a new qint. Call it with qubit_saving on. Verify the return value is correct (measure or check qubit count). Verify circuit_stats shows fewer active qubits than would exist without deallocation.

2. **test_auto_uncompute_preserves_return_value**: Create a compiled function that does `result = ql.qint(0, width=x.width); result += x; return result` (allocates return qubits + possibly temp ancillas from addition). Call with qubit_saving on. Verify the return qint holds the expected value (use Qiskit simulation or measure).

3. **test_auto_uncompute_qubit_reuse**: Call a compiled function with qubit_saving on. The temp ancillas get deallocated. Then allocate a new qint -- verify it reuses the deallocated qubit indices (circuit_stats total_qubits should not grow, or the new qint's qubit indices overlap with freed ones).

4. **test_auto_uncompute_inverse_after**: Call a compiled function with qubit_saving on (auto-uncomputes temps). Then call `f.inverse(x)` -- should succeed and undo the return qubit effects. After inverse, return qint should be invalidated (_is_uncomputed=True).

5. **test_auto_uncompute_none_return**: Create a compiled function returning None that allocates temp ancillas internally. Call with qubit_saving on. All ancillas should be auto-uncomputed and deallocated. Verify no forward call record remains (f.inverse(x) should raise ValueError).

6. **test_auto_uncompute_inplace_skips**: Create a compiled function that modifies input qubits (e.g., `x += 1; return x`). This is an in-place function (return_is_param_index is not None). With qubit_saving on, auto-uncompute should NOT fire (no ancillas to uncompute anyway for in-place functions, but verify the path).

7. **test_auto_uncompute_off_no_effect**: Call a compiled function WITHOUT qubit_saving (default). Verify ancillas are NOT auto-uncomputed (forward call record has all ancillas, circuit_stats shows them as active).

8. **test_auto_uncompute_cache_key_includes_qubit_saving**: Call a compiled function without qubit_saving, then set qubit_saving on and call again with same args. Verify a new cache entry is created (cache miss, not reuse of the non-qubit-saving entry). Check via `len(f._cache)` or `f._total_misses` with debug=True.

9. **test_auto_uncompute_replay_path**: Call a compiled function twice with qubit_saving on (different qubits but same widths). Second call uses replay path. Verify auto-uncompute fires on both calls.

10. **test_auto_uncompute_controlled_context**: Call a compiled function inside a `with` controlled context with qubit_saving on. Verify auto-uncompute still fires and the uncompute gates are also controlled.

For tests needing value verification, use the pattern from existing tests: either `ql.measure()` on a fresh circuit with known initial values, or structural checks (circuit_stats, _is_uncomputed flag, _forward_calls dict length).

Key patterns from existing tests:
- `ql.circuit()` at start of each test
- `ql.option("qubit_saving", True)` right after circuit init
- Access `f._forward_calls` dict to check forward call records
- Access `f._cache` to check cache entries
- Use `ql.circuit_stats()` for qubit counts
- Check `result._is_uncomputed` for invalidation
