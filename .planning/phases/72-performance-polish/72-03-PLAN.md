**Understanding the technique:**

The CNOT-wrapping add-subtract replaces controlled adders (which use CCX for CX and MCX(3) for CCX in the CDKM chain) with uncontrolled adders (which use only CX and CCX) surrounded by CNOT wrappers.

For each multiplier bit `other[j]`:
1. CX-wrap: `CX(target=self[i], control=other[j])` for i=0..width-1
   - When other[j]=0: self unchanged
   - When other[j]=1: self becomes ~self (bitwise NOT)
2. Uncontrolled QQ_add: adds (possibly flipped) self into ret[j..j+width-1]
3. CX-unwrap: same CX gates restore self

Net effect per bit j:
- other[j]=0: ret += self (unwanted -- should be +0)
- other[j]=1: ret += ~self = ret + (2^width - 1 - self) (unwanted -- should be +self)

The desired multiplication is: ret = sum_{j: other[j]=1} self * 2^j

What we actually compute: ret = sum_{j=0}^{n-1} f(other[j]) * 2^j where:
- f(0) = +self
- f(1) = +(2^width_j - 1 - self) = -(self + 1) mod 2^width_j

Wait -- the width varies per iteration: width_j = n - j. And ~self for width_j bits is 2^width_j - 1 - self (considering only the lower width_j bits of self).

**Correction derivation:**

Let n = ret_bits. For iteration j (width = n-j):

When other[j]=0: adds self[0..width-1] into ret, contributing +self (but should contribute 0)
When other[j]=1: adds ~self[0..width-1] = (2^width - 1 - self_width), contributing +(2^width - 1 - self_width) (but should contribute +self_width)

Where self_width = self mod 2^width (lower width bits of self).

So the raw add-subtract computes:
  R = sum_{j=0}^{n-1} [ other[j] * (2^(n-j) - 1 - S_j) + (1-other[j]) * S_j ] * 2^j

where S_j = self mod 2^(n-j).

For the lower n-j bits of self, S_j = self (since self has n bits and we take n-j <= n bits). Actually S_j = self when n-j >= n (j=0), and S_j = self mod 2^(n-j) for j > 0.

This is getting complex. **Simpler approach: empirical correction.**

**Even simpler:** The correction for the add-subtract with the CDKM complement behavior is:

Each iteration j where other[j]=0 contributes +self (unwanted, should be 0).
Each iteration j where other[j]=1 contributes -(self+1) * 2^j instead of +self * 2^j.

For other[j]=0: actual = +self, desired = 0, error_j = +self
For other[j]=1: actual = -(self+1), desired = +self, error_j = -(2*self + 1)

from quantum_language import qint, circuit

class TestOptimizedMulCorrectness:
    """Verify optimized multiplication produces correct results."""

    @pytest.mark.parametrize("width", [1, 2, 3])
    def test_qq_mul_exhaustive(self, width):
        """Exhaustive QQ multiplication at small widths."""
        max_val = 1 << width
        for a_val in range(max_val):
            for b_val in range(max_val):
                c = circuit()
                a = qint(width, a_val)
                b = qint(width, b_val)
                result = a * b

                actual = result.measure()
                assert actual == expected, (
