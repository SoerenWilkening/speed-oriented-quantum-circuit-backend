<task type="auto">
  <name>Task 1: Create Toffoli hardcoded sequence generation script</name>
  <files>scripts/generate_toffoli_seq.py</files>
  <action>
Create `scripts/generate_toffoli_seq.py` that generates Toffoli hardcoded sequence C files for widths 1-8. Follow the pattern from `scripts/generate_seq_all.py` but much simpler because Toffoli sequences use only integer gates (CCX/CX/X) with no floating-point angles.

The script must:

1. **Generate the CDKM gate sequence algorithmically** for each width:
   - For width 1 QQ: single CX(target=0, control=1)
   - For width N >= 2 QQ: MAJ/UMA chain with ancilla at index 2*N
     - Forward MAJ sweep: MAJ(ancilla, N+0, 0), then MAJ(i-1, N+i, i) for i=1..N-1
     - Reverse UMA sweep: UMA(i-1, N+i, i) for i=N-1..1, then UMA(ancilla, N+0, 0)
     - MAJ(a,b,c) = CX(b,c), CX(a,c), CCX(c,a,b)
     - UMA(a,b,c) = CCX(c,a,b), CX(a,c), CX(b,a)
   - For width 1 cQQ: single CCX(target=0, ctrl1=1, ctrl2=2)
   - For width N >= 2 cQQ: cMAJ/cUMA chain with ancilla at 2*N, ext_ctrl at 2*N+1
     - cMAJ(a,b,c,ext) = CCX(b,c,ext), CCX(a,c,ext), MCX(c,[a,b,ext])
     - cUMA(a,b,c,ext) = MCX(c,[a,b,ext]), CCX(a,c,ext), CCX(b,a,ext)

2. **Output per-width C files** `toffoli_add_seq_N.c` (N=1..8):
   - Use `#include "toffoli_sequences.h"` at top
   - Guard with `#ifdef TOFFOLI_SEQ_WIDTH_N`
   - Static const gate_t arrays for each layer (1 gate per layer for CDKM)
   - Static const gate_t* layer pointer array
   - Static const num_t gates_per_layer array (all 1s for CDKM)
   - Static const sequence_t struct for QQ and cQQ
   - Public functions: `get_hardcoded_toffoli_QQ_add_N(int bits)` and `get_hardcoded_toffoli_cQQ_add_N(int bits)` that return pointers to the static structs

   Gate encoding in static const:
   - X gate: `.Gate=X, .Target=t, .NumControls=0, .GateValue=1`
   - CX gate: `.Gate=X, .Target=t, .NumControls=1, .Control={c}, .GateValue=1`
   - CCX gate: `.Gate=X, .Target=t, .NumControls=2, .Control={c1,c2}, .GateValue=1`
   - MCX (3 controls): `.Gate=X, .Target=t, .NumControls=3, .Control={c1,c2}, .large_control=NULL, .GateValue=1`
     IMPORTANT: For MCX with 3+ controls, the static const cannot hold a pointer to dynamically allocated large_control. Instead, store the first 2 controls in Control[2] and for the 3rd control use a separate static array. The dispatch function must handle copying large_control at lookup time OR use a pattern where the 3 controls are packed (check how the QFT cQQ sequences handle this -- if they avoid MCX entirely, the Toffoli cQQ approach needs a helper pattern).

   **Alternative for MCX in static const (simpler approach):** Since the cQQ CDKM uses MCX(3 controls) which needs large_control, and static const cannot initialize heap pointers, use a helper pattern: store 3 controls in a separate static const array alongside the gate, and have the dispatch function return a NON-const copy that sets up large_control properly. OR, more practically, since ToffoliAddition.c already dynamically generates cQQ sequences and caches them, the hardcoded cQQ sequences can use a one-time initialization function that allocates large_control and patches the static data. The simplest approach: for cQQ, generate the sequence dynamically in an init function called from the dispatch, rather than truly static const. This matches how the existing cQQ cache works.

   **Recommended approach for cQQ:** Generate a `toffoli_cQQ_add_init_N()` function that allocates and builds the sequence dynamically (same algorithm as ToffoliAddition.c but with fixed width), stores it in a file-static pointer, and returns it. This avoids the MCX/large_control static const problem entirely. The QQ variant (which uses only CX and CCX, max 2 controls) can be truly static const.

3. **Output dispatch file** `toffoli_add_seq_dispatch.c`:
   - `#include "toffoli_sequences.h"`
   - Forward declarations guarded by `#ifdef TOFFOLI_SEQ_WIDTH_N`
   - `get_hardcoded_toffoli_QQ_add(int bits)` with switch/case
   - `get_hardcoded_toffoli_cQQ_add(int bits)` with switch/case
   - Return NULL for widths outside 1-8

4. **Output header file** `toffoli_sequences.h`:
   - Include guard `QUANTUM_TOFFOLI_SEQUENCES_H`
   - `#include "types.h"`
   - Preprocessor guards: `TOFFOLI_SEQ_WIDTH_1` through `TOFFOLI_SEQ_WIDTH_8` (defined unless `TOFFOLI_SEQ_NO_WIDTH_N` is set)
   - `#define TOFFOLI_HARDCODED_MAX_WIDTH 8`
   - Public API: `const sequence_t *get_hardcoded_toffoli_QQ_add(int bits);`
   - Public API: `const sequence_t *get_hardcoded_toffoli_cQQ_add(int bits);`

5. **Run the script** to generate all files.

Reference `scripts/generate_seq_all.py` for code style and file structure patterns. The Toffoli version is much simpler (no angle computation, no CQ/cCQ variants, no template-init pattern).
