  <files>src/quantum_language/qint.pyx</files>
  <action>
**Bug:** In `__lt__` (line ~1834) and `__gt__` (line ~1924), `temp ^= self` calls `__ixor__` which builds a combined qubit_array of `[target_all_bits, source_all_bits]` and calls `Q_xor(min(w1,w2))`. When temp has `comp_width` (e.g. 4) bits and operand has fewer (e.g. 3) bits, `Q_xor(3)` reads source starting at array offset 3 — but offset 3 lands on temp's own LSB qubit (q63), NOT the operand's MSB qubit. The source block overlaps with temp's own qubits because the combined array is `[temp[60:64], operand[61:64]]` and index 3 is `temp[63]`, not `operand[61]`.

**Fix:** Replace each `temp ^= operand` with a CNOT loop that copies bits individually, LSB-aligned, ensuring zero-extension in the upper bits.

In `__lt__` QQ path (around lines 1831-1840), replace:
```python
temp_self ^= self   # BROKEN
temp_other ^= other # BROKEN
```

With a per-bit CNOT loop for each copy:
```python
# Copy operand bits to temp, LSB-aligned (upper bits stay 0 = zero-extension)
cdef int operand_bits, i_bit
operand_bits = self.bits
for i_bit in range(operand_bits):
    qubit_array[0] = temp_self.qubits[64 - operand_bits + i_bit]  # target
    qubit_array[1] = self.qubits[64 - operand_bits + i_bit]       # source
    seq = Q_xor(1)
    run_instruction(seq, &qubit_array[0], False, _circuit)

operand_bits = (<qint>other).bits
for i_bit in range(operand_bits):
    qubit_array[0] = temp_other.qubits[64 - operand_bits + i_bit]  # target
    qubit_array[1] = (<qint>other).qubits[64 - operand_bits + i_bit]  # source
    seq = Q_xor(1)
    run_instruction(seq, &qubit_array[0], False, _circuit)
```

**Why this produces unsigned comparison:** LSB-aligned copy with zero-extension means the upper bits of the (n+1)-bit temp remain 0. For example, `qint(5, width=3)` stores `101` in 3 bits. Zero-extending to 4 bits gives `0101` (value 5 in unsigned, NOT sign-extended `-3`). Similarly, `qint(2, width=3)` becomes `0010` (value 2). Subtraction `0101 - 0010 = 0011`, MSB=0, so `5 < 2` returns False. This is correct unsigned behavior. The previous XOR bug caused bits to land in wrong positions, corrupting the widened value and producing signed-like results for MSB-boundary values.

Apply the same CNOT loop fix to `__gt__` (around line ~1924), replacing its `temp_other ^= other` and `temp_self ^= self` lines.

**Important details:**
- Keep `comp_width = max(self.bits, (<qint>other).bits) + 1`
- Keep `temp_self = qint(0, width=comp_width)` and `temp_other = qint(0, width=comp_width)`

- Keep `msb = temp_self[63]` (MSB extraction is correct)
