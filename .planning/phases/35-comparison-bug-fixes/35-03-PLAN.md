  <name>Task 1: Fix XOR bit-copy alignment in __lt__ and __gt__</name>
  <files>src/quantum_language/qint.pyx</files>
  <action>
**Root cause:** In `__lt__` (line ~1834) and `__gt__` (line ~1924), `temp ^= self` calls `__ixor__` which uses `Q_xor(min(self.bits, other.bits))`. When temp has `comp_width` bits and operand has fewer bits, Q_xor operates on the FIRST `min` qubits in the array. Due to right-aligned storage, this means Q_xor targets the MSB side of the wider temp instead of the LSB side where the bits should go. Additionally, the source array includes temp's own LSB qubit mixed with the operand's qubits.

**Fix for both `__lt__` and `__gt__`:**

Replace the XOR-based bit copy with individual CNOT (controlled-NOT) gates that copy each bit from the operand to the correct LSB-aligned position in the widened temp.

In `__lt__` QQ path (around lines 1831-1840), replace:
```python
comp_width = max(self.bits, (<qint>other).bits) + 1
temp_self = qint(0, width=comp_width)
temp_self ^= self  # BROKEN: misaligned XOR
temp_other = qint(0, width=comp_width)
temp_other ^= other  # BROKEN: misaligned XOR
```

With code that copies bits individually using CNOT gates. For each bit i in [0, operand.bits), apply a CNOT from operand's qubit at position `(64 - operand.bits + i)` to temp's qubit at position `(64 - comp_width + (comp_width - operand.bits) + i)` = `(64 - operand.bits + i)`. Since both are right-aligned with LSB at index 63, the bit at position `64 - n + i` in an n-bit qint corresponds to bit value `2^(n-1-i)`.

For a correct LSB-aligned copy, we need to align the LSBs. The operand's LSB is at index 63, and the temp's LSB is also at index 63. So bit i of the operand (at index `64 - operand.bits + i`) should go to the same relative LSB position in the temp (at index `64 - operand.bits + i`). The upper bits of temp (indices `64 - comp_width` to `64 - operand.bits - 1`) remain 0.

This means the qubit indices actually match! The real issue is that `__ixor__` sets up qubit_array as `[target_all_bits, source_all_bits]` and Q_xor only processes `min(w1,w2)` pairs starting from index 0 of each block. Since target's qubits start from MSB (index 64-comp_width), the first `min` qubits of the target block are MSB qubits, but they should be paired with MSB qubits of the source. Q_xor pairs target[0] with source[0], target[1] with source[1], etc. — but target[0] is target's MSB while source[0] is source's MSB. So the MSBs are paired correctly!

Wait — re-examine. In __ixor__:
- qubit_array[0:4] = temp_self.qubits[60:64] = [q60, q61, q62, q63] (4-bit temp)
- qubit_array[4:7] = self.qubits[61:64] = [q61', q62', q63'] (3-bit self)
- Q_xor(3): target = array[0:3] = [q60, q61, q62], source = array[3:6] = [q63, q61', q62']

The source starts at array index 3, which is temp's q63 (LSB), NOT self's q61' (MSB)! This is the bug: the source block starts at index `min(w1,w2)` into the combined array, not at index `self.bits`. Since self.bits=4 (temp) but min=3, the source "starts" at offset 3 in the array, which overlaps with temp's own qubits.

**The actual fix:** Replace `temp ^= operand` with a loop that applies individual CNOT gates from each operand qubit to the corresponding LSB-aligned temp qubit. Use `Q_not(1)` as a controlled-NOT by placing [target_qubit, source_qubit] in qubit_array and calling the 2-qubit controlled version, OR more simply, use `Q_xor(1)` in a loop for each bit.

For each bit i from 0 to operand.bits-1:
```python
# Copy bit i from operand to temp (LSB-aligned)
# operand bit i: operand.qubits[64 - operand.bits + i]
# temp bit i (LSB-aligned): temp.qubits[64 - operand.bits + i]
# (temp has more bits, but LSBs align at index 63)
qubit_array[0] = temp.qubits[64 - operand_bits + i]  # target
qubit_array[1] = operand.qubits[64 - operand_bits + i]  # source
arr = qubit_array
seq = Q_xor(1)
run_instruction(seq, &arr[0], False, _circuit)
```

This copies each operand bit to the correct position in the wider temp, with zero-extension (upper bits of temp remain 0).

Apply this fix to BOTH `__lt__` (temp_self ^= self and temp_other ^= other) and `__gt__` (temp_other ^= other and temp_self ^= self).

**Important details:**
- Keep `comp_width = max(self.bits, (<qint>other).bits) + 1`
- Keep `temp_self = qint(0, width=comp_width)` and `temp_other = qint(0, width=comp_width)`
- Replace ONLY the `temp ^= operand` lines with the CNOT loop
- Keep `temp_self -= temp_other` (subtraction between same-width temps works correctly)
- Keep `msb = temp_self[63]` (MSB extraction is correct)
- Keep all other logic (result creation, dependency tracking, etc.)
- Declare `cdef int operand_bits` if needed for the loop variable
