Fix BUG-CMP-01: equality (eq) and inequality (ne) comparisons return inverted results for ALL inputs at ALL widths. The root cause is that `__eq__` still sets `_start_layer`/`_end_layer` on the result qbool, causing garbage-collection-triggered gate reversal that undoes the entire equality comparison circuit. This is the same bug that was fixed for `__lt__` and `__gt__` in Phase 29-16.

Purpose: 488 xfail tests depend on this fix. eq/ne are fundamental comparison operators used by other operations.
Output: Working eq/ne comparisons, Cython rebuild, validated against existing test suite.
</objective>

The root cause of BUG-CMP-01 is identical to the GC gate reversal bug fixed in Phase 29-16 for __lt__ and __gt__. The `__eq__` method still assigns `_start_layer`/`_end_layer` to the result qbool, which causes `_do_uncompute` to reverse the comparison circuit when the qbool is garbage collected.

**Fix 1: QQ path (qint == qint) - around line 1688-1690 of qint.pyx:**
Remove or comment out these two lines:
```python
result._start_layer = start_layer
result._end_layer = (<circuit_s*>_circuit).used_layer if _circuit_initialized else 0
```
Replace with a comment explaining why layer tracking is deliberately omitted:
```python
# Note: deliberately NOT setting _start_layer/_end_layer here.
# Comparison results must persist in the circuit even when GC'd,
# matching the pattern of arithmetic results (no auto-uncompute).
# See Phase 29-16 D29-16-2 for rationale.
```

**Fix 2: CQ path (qint == int) - around line 1748-1749 of qint.pyx:**
Remove or comment out these two lines:
```python
result._start_layer = start_layer
result._end_layer = (<circuit_s*>_circuit).used_layer if _circuit_initialized else 0
```
Replace with the same comment as above.

**Also remove the now-unused `start_layer` variable capture** at line 1663:
The `start_layer` variable and its capture at line 1663 are no longer needed. Remove:
```python
start_layer = (<circuit_s*>_circuit).used_layer if _circuit_initialized else 0
```
And the corresponding `cdef int start_layer` declaration at line 1651.

IMPORTANT: Do NOT remove `start_layer` from the QQ path lines 1688-1690 if it is also used in the recursive `self == 0` call's result. Check carefully: the QQ path calls `result = self == 0` which creates a NEW result via CQ path. The QQ path then overwrites `result._start_layer`.
