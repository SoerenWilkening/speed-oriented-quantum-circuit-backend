<objective>
Fix two bugs that together prevent branch() from accumulating rotations:

1. **gates_are_inverse() optimizer bug** (c_backend/src/gate.c:548): The function incorrectly treats two rotation gates (Ry/Rx/Rz) with the **same** angle as inverses, causing the optimizer's `add_gate()` to cancel both. The correct inverse of Ry(theta) is Ry(-theta), not Ry(theta). The P gate already has the correct negated-angle check; Ry/Rx/Rz need the same treatment.

2. **_start_layer overwrite in branch()** (src/quantum_language/qint.pyx:594-595): Each call to `branch()` overwrites `self._start_layer` / `self._end_layer`, so uncomputation after multiple branch() calls would only reverse the last rotation. Fix accumulates the range across calls.

Purpose: Without these fixes, `test_double_branch_accumulates` fails with P(1)=0.0 because both Ry(pi/2) gates are silently cancelled. After the fix, two Ry(pi/2) gates survive and accumulate to Ry(pi), giving P(1) > 0.95.

Output: Corrected gate.c and qint.pyx source files ready for package rebuild (plan 06).
</objective>

    The problem: the `else if` branch applies to ALL non-P gates including Ry, Rx, Rz. For these rotation gates, `same angle != inverse`. Two Ry(pi/2) gates are NOT inverses — Ry(theta)^{-1} = Ry(-theta).

    X, H, Z gates have GateValue=0, so "same GateValue" happens to work (0 != 0 is false → stays true, correctly self-inverse). But Ry/Rx/Rz with nonzero angles break this logic.

    Replace the block with:

    ```c
    if (G1->Gate == P || G1->Gate == Ry || G1->Gate == Rx || G1->Gate == Rz) {
        if (G1->GateValue != -G2->GateValue)
            return false;
    } else if (G1->GateValue != G2->GateValue)
        return false;
    ```

    This makes all rotation gates require negated angles to be considered inverses, matching the mathematical definition Ry(theta)^{-1} = Ry(-theta). The P gate behavior is unchanged.

        self._start_layer = start_layer
        self._end_layer = end_layer
    else:
        # Subsequent calls: expand tracked range
        self._start_layer = min(self._start_layer, start_layer)
