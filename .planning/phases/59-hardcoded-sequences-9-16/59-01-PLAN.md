<task type="auto">
  <name>Task 1: Create unified generation script</name>
  <files>scripts/generate_seq_all.py</files>
  <action>
Create `scripts/generate_seq_all.py` by refactoring and extending the existing generation scripts.

**Core gate generation functions** (port directly from `generate_seq_5_8.py` -- these are identical in both existing scripts):
- `Gate` dataclass with `to_c()` and `format_angle()` methods (exact copy from existing)
- `generate_qq_add(bits)` -- QFT + addition phase + IQFT (exact copy from existing)
- `generate_cqq_add(bits)` -- QFT + Block1/2/3 CCP decomposition + IQFT (exact copy from existing, includes the quick-015 fix: Block 2 uses `bits + bit` as control for negative half-rotations, NOT the external control qubit)
- `optimize_layers(layers)` -- merge non-overlapping layers (exact copy from existing)
- `generate_c_sequence(name, layers, width)` -- static const gate array generation (exact copy from existing)

**New CQ_add template-init generation:**
- `generate_cq_add_template_layers(bits)` -- returns the gate structure for CQ_add:
  - QFT on target register [0, bits-1] (same as QQ_add QFT section)
  - `bits` rotation layers: each has one `P` gate on qubit `i` with angle placeholder `0`
  - Inverse QFT on target register (same as QQ_add IQFT section)
  - Total layers: `5 * bits - 2`
- `generate_cq_add_template_c(bits)` -- generates C function `init_hardcoded_CQ_add_N()`:
  - Static cache variable `cached_CQ_add_N`
  - Returns cached if non-NULL
  - Allocates sequence with `malloc`/`calloc`
  - Populates all gates with exact positions and fixed angles from QFT/IQFT sections
  - Rotation layers get `GateValue = 0` (placeholder -- caller injects angles)
  - Uses `(gate_t)` compound literal initializer for each gate (same fields as existing `to_c()` output)
  - Caches and returns

**New cCQ_add template-init generation:**
- `generate_ccq_add_template_layers(bits)` -- returns the gate structure for cCQ_add:
  - QFT on target register [0, bits-1]
  - `bits` rotation layers: each has one `CP` gate on qubit `i` with control `bits` (the conditional qubit) and angle placeholder `0`
  - Inverse QFT on target register
  - Total layers: `5 * bits - 2`
- `generate_ccq_add_template_c(bits)` -- generates C function `init_hardcoded_cCQ_add_N()`:
  - Same pattern as CQ_add but rotation gates are `CP` (controlled phase) with `.NumControls = 1, .Control = {bits}`

**Per-width file generation:**
- `generate_width_file(width)` -- generates complete C source file for one width:
  - File header comment with "Generated by scripts/generate_seq_all.py - DO NOT EDIT MANUALLY"
  - `#include "sequences.h"`, SEQ_PI define
  - `#ifdef SEQ_WIDTH_N` preprocessor guard wrapping entire file content
  - QQ_ADD section: static const arrays via `generate_c_sequence()`
  - cQQ_ADD section: static const arrays via `generate_c_sequence()`
  - CQ_ADD section: template-init function via `generate_cq_add_template_c()`
  - cCQ_ADD section: template-init function via `generate_ccq_add_template_c()`
  - Per-width dispatch helpers: `get_hardcoded_QQ_add_N()`, `get_hardcoded_cQQ_add_N()`, `get_hardcoded_CQ_add_N()`, `get_hardcoded_cCQ_add_N()`
  - `#endif // SEQ_WIDTH_N`

**Dispatch file generation:**
- `generate_dispatch_file()` -- generates `add_seq_dispatch.c`:
  - Forward declarations for all 16 per-width helpers (4 functions * 16 widths = 64 declarations), each wrapped in `#ifdef SEQ_WIDTH_N`
  - 4 unified dispatch functions: `get_hardcoded_QQ_add(bits)`, `get_hardcoded_cQQ_add(bits)`, `get_hardcoded_CQ_add(bits)`, `get_hardcoded_cCQ_add(bits)`
  - Each uses `switch(bits)` with cases wrapped in `#ifdef SEQ_WIDTH_N`
  - QQ/cQQ dispatch returns `const sequence_t *`; CQ/cCQ dispatch returns `sequence_t *` (mutable for angle injection)

**CLI interface:**
- `--width N` to generate single width file (for debugging)
- `--dry-run` to print without writing
- `--output-dir DIR` to specify output directory (default: `c_backend/src/sequences/`)
- No arguments: generate all 16 per-width files + dispatch file
- Old scripts get a deprecation comment at top (do NOT modify them, just note in the new script's docstring)

**Critical: Preserve cQQ_add fix from quick-015.**
The `generate_cqq_add()` function MUST use `bits + bit` (b-register qubit) as control for negative half-rotations in Block 2, NOT the external control qubit. This is the fix from commit c891a32.
