  - Create a compiled function that does `x += 1; return x`
  - Call it OUTSIDE a `with` block -- inspect cache, verify uncontrolled + controlled entries exist (eager compilation)
  - Call it INSIDE a `with qbool:` block -- verify the cache hit (controlled variant)
  - Use `extract_gate_range()` or OpenQASM export to verify that gates inside the `with` block have controls

  **2. `test_compile_controlled_separate_cache_entries`:**
  - Create a compiled function
  - Call outside `with` block, note cache size
  - Call inside `with` block, verify no re-capture (cache hit on controlled variant)
  - Verify cache has entries for both control_count=0 and control_count=1

  **3. `test_compile_controlled_gates_have_extra_control`:**
  - Create a compiled function that does a simple operation (e.g., `x += 1`)
  - Call outside `with` block, extract the uncontrolled gate list from the CompiledBlock
  - Get the controlled CompiledBlock from cache
  - Verify every gate in the controlled variant has `num_controls == uncontrolled_gate.num_controls + 1`

  **4. `test_compile_controlled_nested_with`:**
  - Create a compiled function
  - Call inside nested `with qbool1: with qbool2:` blocks
  - Verify the function sees a single control (since `with` blocks AND the bools)
  - Verify controlled gates are replayed correctly

  **5. `test_compile_controlled_replay_correct_qubits`:**
  - Create a compiled function, call inside `with` block
  - Call again inside a DIFFERENT `with` block (different qbool)
  - Verify the replayed gates use the correct control qubit for each call

  **6. `test_compile_controlled_custom_key`:**
  - Create a compiled function with `key=lambda x: x.width`
  - Call outside `with` block, then inside `with` block
  - Verify separate cache entries exist (custom key wrapped with control count)

  **7. `test_compile_controlled_first_call_inside_with`:**

  - Call again inside `with` block -- verify the replayed gates ARE controlled (demonstrating subsequent calls work correctly)
