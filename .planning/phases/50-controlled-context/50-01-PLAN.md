  **3. Create `_derive_controlled_gates()` module-level function:**

  ```python
  def _derive_controlled_gates(gates, control_virtual_idx):
      """Add one control qubit to every gate in the list."""
      controlled = []
      for g in gates:
          cg = dict(g)
          cg['num_controls'] = g['num_controls'] + 1
          cg['controls'] = [control_virtual_idx] + list(g['controls'])
          controlled.append(cg)
      return controlled
  ```

  Place this after `_optimize_gate_list()` and before the global registry section.

  **4. Create `_derive_controlled_block()` method on `CompiledFunc`:**

  Creates a new `CompiledBlock` from an uncontrolled block by deriving controlled gates. The control qubit gets virtual index `uncontrolled_block.total_virtual_qubits` (one beyond all uncontrolled virtual qubits). The controlled block has `total_virtual_qubits = uncontrolled.total_virtual_qubits + 1` and `control_virtual_idx` set to the control placeholder index. All other fields (param_qubit_ranges, internal_qubit_count, return_qubit_range, return_is_param_index, original_gate_count) are copied from the uncontrolled block.

  **5. Modify `CompiledFunc.__call__()` to detect controlled context and extend cache key:**

  - At the start, call `_get_controlled()` to detect controlled context. Set `control_count = 1 if is_controlled else 0`.
  - Extend the default cache key from `(tuple(classical_args), tuple(widths))` to `(tuple(classical_args), tuple(widths), control_count)`.
  - When a custom `key` function is used, wrap the result: `cache_key = (self._key_func(*args, **kwargs), control_count)`.
  - On cache miss, call a new method `_capture_and_cache_both()` instead of `_capture()` directly.

  **6. Create `_capture_and_cache_both()` method on `CompiledFunc`:**

  This method handles cache miss logic:
  - If currently in controlled context (`is_controlled=True`), save the controlled state (`_get_controlled()`, `_get_control_bool()`, `_get_list_of_controls()`), temporarily set to uncontrolled (`_set_controlled(False)`, `_set_control_bool(None)`), call `_capture()`, then restore the saved state in a `finally` block.
  - If NOT in controlled context, just call `_capture()` normally.
  - After capturing the uncontrolled block, derive the controlled variant using `_derive_controlled_block()` (wrap in try/except -- on failure, fall back to re-capture in controlled mode by calling `_capture()` while controlled state is active).
  - Cache both uncontrolled and controlled blocks under their respective keys (uncontrolled key has `control_count=0`, controlled key has `control_count=1`).
  - Evict oldest if cache exceeds `_max_cache`.
  - **First-call trade-off:** When first called inside a `with` block, the capture runs in uncontrolled mode, so the first-call circuit contains uncontrolled gates. This is an accepted trade-off -- the circuit cannot be retroactively corrected since gates cannot be erased. On all subsequent calls inside `with` blocks, the controlled cached variant is replayed correctly. Document this in a docstring: "First call of a compiled function inside a `with` block executes the uncontrolled body; subsequent calls correctly replay controlled gates."
  - Return `block._first_call_result`.

  **7. Modify `CompiledFunc._replay()` to handle control qubit remapping:**

  After allocating ancillas for internal qubits, check if the block has `control_virtual_idx is not None`. If so, instead of allocating a fresh qubit for that virtual index, map it to `_get_control_bool().qubits[63]` (the MSB of the 1-bit qbool, which is its actual qubit index). The existing ancilla allocation loop (iterating from `vidx` to `block.total_virtual_qubits`) should check each virtual index against `block.control_virtual_idx`.
