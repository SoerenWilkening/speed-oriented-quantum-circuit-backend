<technical_approach>
The gate_t structure already has support for >2 controls via the `large_control` field:

```c
typedef struct {
    qubit_t Control[MAXCONTROLS];  // Static array for 0-2 controls
    qubit_t *large_control;        // Dynamic array for >2 controls
    num_t NumControls;
    ...
} gate_t;
```

Strategy:
1. Create `mcx()` helper function that:
   - Takes target qubit and array of control qubits with count
   - For NumControls <= 2: uses Control[0], Control[1]
   - For NumControls > 2: allocates and populates large_control array

2. Update CQ_equal_width Phase 2 (lines 155-179):
   - 1-bit: cx (unchanged)
   - 2-bit: ccx (unchanged)
   - 3+ bits: use mcx() with all operand qubits [1, bits] as controls, target = qubit[0]

3. Update cCQ_equal_width Phase 2 (lines 306-333):
   - 1-bit: ccx with control_qubit and qubit[1]
   - 2-bit: mcx with control_qubit, qubit[1], qubit[2]
   - 3+ bits: mcx with control_qubit and all operand qubits

Memory management:
- mcx() allocates large_control dynamically
- Caller must free large_control when freeing sequence (existing free_sequence handles this)
- Add free logic to test helper if not already present
</technical_approach>
