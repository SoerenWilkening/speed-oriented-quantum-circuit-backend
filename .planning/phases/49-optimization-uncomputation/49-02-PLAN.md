Add tests to test_compile.py verifying compiled function results integrate with the uncomputation system. Append after existing tests.

Tests to add:

1. **test_uncomputation_compiled_result_in_with_block**: Test that a qint returned from a compiled function is correctly uncomputed when used inside a `with` block:
   ```python
   ql.circuit()
   @ql.compile
   def make_result(x):
       y = x + ql.qint(1, width=x.width)
       return y

   a = ql.qint(3, width=4)
   ctrl = ql.qbool()
   with ctrl:
       result = make_result(a)
       # result should be auto-uncomputed at scope exit
   # Verify circuit has reverse gates (gate count after with > gate count before)
   ```
   Use `get_current_layer()` before and after the `with` block. The with block should produce both forward and reverse gates. Verify the total gate count is roughly double (forward + uncompute).

2. **test_uncomputation_in_place_return_no_double_uncompute**: Verify that when a compiled function returns its input parameter (in-place), the result is NOT double-uncomputed:
   ```python
   ql.circuit()
   @ql.compile
   def add_inplace(x):
       x += 1
       return x

   a = ql.qint(3, width=4)
   result = add_inplace(a)
   assert result is a  # In-place returns caller's qint
   # result.operation_type should be unchanged (not "COMPILED")
   ```

3. **test_uncomputation_replay_result_in_with_block**: Same as test 1 but trigger replay (second call) instead of first call. Verify the replay result is also correctly uncomputed:
   ```python
   ql.circuit()
   @ql.compile
   def make_val(x):
       y = x + ql.qint(2, width=x.width)
       return y

   # First call (capture)
   a = ql.qint(1, width=4)
   r1 = make_val(a)

   # Second call (replay) inside with block
   b = ql.qint(5, width=4)
   ctrl = ql.qbool()
   layer_before = get_current_layer()
   with ctrl:
       r2 = make_val(b)
   layer_after = get_current_layer()
   # Should have both forward replay + reverse uncomputation gates
   assert layer_after > layer_before
   ```

4. **test_compiled_result_has_operation_type**: Verify that both first-call and replay results have `operation_type` set:
   ```python
   ql.circuit()
   @ql.compile
   def make_new(x):
       return x + ql.qint(1, width=x.width)

   a = ql.qint(0, width=4)
   r1 = make_new(a)
   assert r1.operation_type is not None  # First call

   b = ql.qint(0, width=4)
   r2 = make_new(b)
   assert r2.operation_type is not None  # Replay
   ```

5. **test_uncomputation_uses_optimized_sequence**: Create a compiled function with cancellable gates. Inside a with block, call it. The uncomputation should reverse the OPTIMIZED gate count, not the original. Compare gate counts:
   ```python
   ql.circuit()
   @ql.compile
   def redundant_ops(x):
       x += 1
       x -= 1
       x += 3
       return x  # net effect: x += 3, but original has 3 operations

   a = ql.qint(0, width=4)
   r1 = redundant_ops(a)  # Capture + optimize

   optimized_count = redundant_ops.optimized_gates

   b = ql.qint(0, width=4)
   ctrl = ql.qbool()
   start = get_current_layer()
   with ctrl:
       r2 = redundant_ops(b)
   end = get_current_layer()

   replay_range_gates = extract_gate_range(start, end)
   # The with block should contain optimized forward + optimized reverse
   # NOT original forward + original reverse
   ```

Follow existing test patterns. Import `get_current_layer`, `extract_gate_range` from `quantum_language._core`. Use `ql.qbool()` for control qubits in with blocks.
