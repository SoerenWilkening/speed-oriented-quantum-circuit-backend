## Resolved Gaps

### setup.py package_data Fixed

**Issue:** `setup.py` was missing `*.py` in `package_data`, causing `__init__.py` to not be copied to the build directory.

**Resolution:** Fixed in commit 5052054 by adding `"*.py"` to `package_data` for both `quantum_language` and `quantum_language.state` packages.

**Verification:** Confirmed that `__init__.py` is now correctly copied to build directory and the public API (`ql.array(width=...)`, `ql.all/any/parity`) works in the compiled/installed package.
